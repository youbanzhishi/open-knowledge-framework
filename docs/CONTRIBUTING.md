# 贡献指南

感谢你对 OpenClaw Framework 的关注！

## 如何贡献

### Fork-Enhance-PR 流程

1. **Fork** 本仓库到你自己的账号
2. 在你的 Fork 中进行改进
3. 提交 **Pull Request** 回到本仓库

### 什么应该贡献

- ✅ 通用角色模板改进
- ✅ 脚本 bug 修复或功能增强
- ✅ 新的通用设计模式
- ✅ 规范文档改进
- ✅ 新的角色/技能/项目模板
- ✅ 架构改进建议
- ✅ 文档翻译

### 什么不应该贡献

- ❌ 你的私有项目内容
- ❌ 你的个人信息/凭据
- ❌ 与具体业务强相关的定制
- ❌ 含有硬编码 token/密钥的内容

### 提交规范

```
<type>: <描述>

type:
  feat     新功能/新模板
  fix      bug修复
  refactor 重构（不改变功能）
  docs     文档
  chore    杂项
```

### 代码风格

- Shell脚本：bash，`set -euo pipefail`，函数注释
- Markdown：中文，ATX标题，表格对齐
- 模板文件：`[占位符]` 标记需要定制的内容
