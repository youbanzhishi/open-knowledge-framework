# 架构设计

## 核心理念

OpenClaw Framework 遵循 **Extension Registry** 模式：

> 新功能 = 注册扩展，架构本身永远不需要改

这意味着：
- 添加新角色 → 不改框架，只创建新角色目录
- 添加新技能 → 不改框架，只创建新技能目录
- 添加新项目 → 不改框架，只创建新项目目录
- 添加新工具 → 不改框架，只注册新扩展

## 目录层次

```
宪法层（改一次全局生效）   共享知识/项目规范/
蓝图层（愿景+哲学）        共享知识/蓝图.md
角色层（执行者）           角色/[角色名]/
技能层（可复用能力）       技能/[技能名]/
项目层（产出物）           项目/[项目名]/
运行时（工单流转）         交接台/
自动化（引擎脚本）         scripts/
```

## 记忆体系

```
即时层（自动加载到上下文）
├── SOUL.md      身份定义
├── USER.md      用户画像
├── MEMORY.md    当前状态
├── TOOLS.md     工具经验
└── SECRET.md    敏感凭证

近中期层（按需加载）
├── recent_memory/index.json    记忆目录
├── recent_memory/project/      项目进度
├── recent_memory/decision/     重要决策
└── recent_memory/todo/         待办事项

长期层（语义检索）
└── memory_search    RAG语义搜索
```

## 自动化协作引擎

四种模式可组合：

| 模式 | 说明 |
|------|------|
| 匀速 | 日常维护，每4小时，3轮上限 |
| 加速 | 单项目冲刺，循环往复直到蓝图达标 |
| 并行 | 多项目同时推进，项目分组并行 |
| 聚焦 | 项目白名单，其他冻结，可叠加任何模式 |

## 安全机制

### 物理层（不可绕过）
- 未commit禁止pull：git alias + shell function双重拦截
- 禁止强推：pre-push hook
- 敏感文件检测：pre-commit hook

### 逻辑层（规则约束）
- 删除→回收站，覆盖=犯罪
- 先读后动
- 失败2次停止

### 体系层（自动维护）
- 五步门：读→做→验→反哺→汇报
- 热规则：踩坑自动升级成规则
- 工单流转：角色间自动交接
