# 定时任务恢复重建技能

## 功能描述
从备份中恢复并重建所有定时任务。支持多种配置来源，实现无缝迁移。

## 技能信息
- 加载模式：progressive
- 触发关键词：[定时任务, 恢复, calendar]
- 摘要：定时任务备份恢复与重建

## 配置来源

### 1. tasks.json（主配置）
路径：`./定时任务/tasks.json`

### 2. MEMORY.md（迁移配置）
路径：`./MEMORY.md` 中的"定时任务配置（迁移时恢复）"部分

**优先级：** MEMORY.md > tasks.json

## 适用场景
- 迁移到新机器后恢复定时任务
- 批量重建定时任务
- 定时任务配置备份与恢复

## 核心能力
1. **多源读取**：支持 tasks.json 和 MEMORY.md 两种配置来源
2. **心跳任务支持**：自动恢复 heartbeat.md 和心跳日程
3. **依赖检查**：重建前检查技能和文件是否存在
4. **增量恢复**：跳过已存在的任务
5. **批量创建**：一键重建所有定时任务
6. **恢复报告**：生成恢复结果报告

## 恢复流程

```
1. 检查 MEMORY.md 是否有"定时任务配置"部分
   ├─ 有 → 读取 MEMORY.md 配置
   └─ 无 → 读取 tasks.json 配置

2. 检查依赖文件和技能是否存在
   ├─ 缺失 → 记录到恢复日志，跳过该任务
   └─ 存在 → 继续创建

3. 遍历每个任务配置
   ├─ 心跳任务 → 创建心跳日程 + 确保 heartbeat.md 存在
   └─ 日程任务 → 创建日程

4. 记录恢复结果到 restore_log.md
```

## 使用方式

### 方式一：对话中直接恢复（推荐）
```
用户：帮我恢复所有定时任务
助手：加载技能 → 读取配置 → 检查依赖 → 逐个创建任务 → 返回恢复报告
```

**执行流程**（Agent 执行）：
1. 读取 MEMORY.md 中的"定时任务配置"部分
2. 如果没有，读取 `./定时任务/tasks.json`
3. 遍历每个任务，检查依赖
4. 调用 `calendar_create` 创建任务
5. 对于心跳任务，确保 heartbeat.md 文件存在
6. 记录结果并生成报告

### 方式二：脚本执行
```bash
python3 ./技能/定时任务恢复技能/scripts/restore_tasks.py
```

## 配置文件
- **主配置**：`./定时任务/tasks.json`
- **迁移配置**：`./MEMORY.md`（定时任务配置部分）
- **心跳配置**：`./心跳/heartbeat.md`
- **恢复日志**：`./定时任务/restore_log.md`

## 心跳任务特殊处理

心跳任务恢复时需要额外步骤：

1. **检查 ./心跳/heartbeat.md 是否存在**
   ```bash
   # 如果不存在，创建默认内容
   mkdir -p ./心跳
   echo "检查清单\n- 检查待处理事项\n- 检查定时任务执行状态" > ./心跳/heartbeat.md
   ```

2. **检查 ./定时任务/执行记录.md 是否存在**
   ```bash
   # 如果不存在，创建默认内容
   mkdir -p ./定时任务
   # 创建执行记录文件，用于心跳守护监控
   ```

3. **创建心跳日程**
   ```
   summary: 心跳检查-守护监控
   rrule: {"freq": "MINUTELY", "interval": 30}
   description: 从 MEMORY.md 读取
   ```

## 注意事项
- 恢复前确保依赖的技能和文件已存在
- 任务时间会自动调整为当前日期之后的下一个执行时间
- 已存在的同名任务会被跳过（避免重复创建）
- 心跳任务恢复需要确保 ./心跳/heartbeat.md 文件存在
