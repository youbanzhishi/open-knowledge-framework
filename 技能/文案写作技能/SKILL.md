# 文案写作技能

## 技能信息
- 加载模式：progressive
- 触发关键词：[文案, 写作, 标题]
- 摘要：多平台文案创作与优化

## 技能概述
专业的自媒体文案写手，负责网盘变现项目的文案创作，根据不同平台特点生成高质量、有吸引力的文案，并植入网盘分享链接。

## 功能特点
- ✍️ 多平台文案适配（小红书、微博、CSDN、博客园）⚠️ 知乎已由主技能直接编写，不再由本技能处理
- 🎨 多种写作风格（专业型、种草型、故事型、幽默型）
- 🤖 AI去机味处理（让文案像真人写的）
- 🔗 自动植入网盘链接（根据平台规范）

## 使用场景
- 网盘变现项目需要发布新资源时
- 需要为同一资源生成多平台文案
- 文案质量优化（去AI味、提升吸引力）

```json
{
  "resource_name": "资源名称，如：考研备考规划",
  "resource_type": "资源类型，如：考研资料、职场模板、学习资源",
  "share_link": "城通网盘分享链接，如：https://url98.ctfile.com/f/xxx-xxx-xxx",
  "password": "访问密码，默认：8069",
  "platforms": ["小红书", "微博", "CSDN"],  // 需要生成的平台列表（⚠️ 知乎由主技能直接编写，不在此列）
  "style": "可选，默认智能推荐。可选：专业型、种草型、故事型、幽默型"
}
```

## 输出格式

### 文件存放
- 生成路径：`./网盘引流项目/待发布文章/`
- 文件命名：`YYYY-MM-DD_平台_资源名称.md`
- 示例：`2026-04-28_知乎_考研备考规划.md`

### 文案内容结构
每个平台的文案文件包含：
```
# 标题

[正文内容]

标签/话题

下载链接（根据平台规范植入）
```

## 写作质量标准

### 三大要素
1. **有价值**：信息密度高、实用性强
2. **有吸引力**：标题抓人、开头留人、结尾钩子
3. **有真人感**：不端着、有细节、有情绪

### AI去机味原则
- ❌ 禁用：成语、书面词、套话、假大空
- ❌ 禁用：首先/其次/综上所述/值得注意的是
- ✅ 使用：口语化表达、具体细节、真实情绪
- ✅ 使用：第一人称、个人观点、不完美表达

### 真人真话测试
写完后必须通过测试：
> 这段话像"给所有人看的正确答案"，还是"一个真人在说自己的真话"？

## 执行流程

### 场景一：生成新文章
1. **接收参数** → 解析资源信息、链接、平台列表
2. **搜索素材** → 根据资源类型搜索相关数据和案例
3. **生成文案** → 按平台规范生成对应文案
4. **质量检测** → AI味检测、真人感测试
5. **保存文件** → 存入待发布目录

### 场景二：润色现有文章
1. **读取原文** → 获取文章内容和目标平台
2. **诊断分析** → 检测文章问题（AI味、结构、互动等）
3. **应用润色** → 根据润色方向逐项优化
4. **质量检测** → AI味检测、真人感测试、平台适配检查
5. **输出文件** → 覆盖原文件或另存为新文件

#### 润色诊断清单

**AI味检测**：
- [ ] 连接词密度：统计"首先、其次、此外、综上所述"等
- [ ] 书面词密度：统计"旨在、致力于、具有重要意义"等
- [ ] 结构完美度：检查是否总分总、段落长度相近
- [ ] 个人视角：检查是否有"我"、"我觉得"、"我认为"
- [ ] 具体细节：检查是否只有抽象概念
- [ ] 情绪表达：检查是否缺乏情绪词

**结构检测**：
- [ ] 标题吸引力：是否含数字、场景、情绪、利益
- [ ] 开头钩子：3秒内是否抓住注意力
- [ ] 段落长度：是否超过3行
- [ ] 钩子密度：是否每280-320字一个刺激点
- [ ] 表格使用：是否用表格整理信息

**互动检测**：
- [ ] 结尾互动：是否有具体问题或行动指令
- [ ] 利他性：是否让读者获益
- [ ] 低门槛：是否容易执行

**平台适配检测**：
- [ ] 知乎格式：⚠️ 知乎已由主技能直接编写，本技能不处理知乎
- [ ] 链接格式：是否符合平台规范
- [ ] 风格匹配：是否符合平台调性
- [ ] 字数范围：是否在平台推荐范围

## 爆款技巧速查（参考平台爆款特点分析.md）

### 标题通用公式
```
数字量化 + 具体场景 + 情绪触动 + 利益承诺
```

**示例对比**：
- ❌ "如何护肤"
- ✅ "为什么每天敷面膜，鼻翼还是疯狂爆皮？"（点击率高3.2倍）

### 开头三秒法则
- **今日头条**：强冲突/反常识/悬念钩子，第一段必须出现"你"或"我"
- **小红书**：故事/痛点引入，用人称代词"我""咱们"
- **知乎**：断言式陈述+真实场景+结果差异（不超过20字断言）
- **CSDN**：金句开场+emoji列出文章价值

### 内容结构要点
- **每段不超过3行**（手机阅读）
- **每280-320字设一个钩子**（知乎）
- **表格整理信息**（CSDN/知乎）
- **emoji引导视觉**（小红书/CSDN）

### 结尾互动设计
- ❌ "欢迎点赞收藏"（太笼统）
- ✅ "你是A还是B？" / "你有没有过这种经历？"（易回答）
- ✅ "在评论区留下你的行业+最卡壳的环节，我帮你改"（利他性）

## 注意事项

## ⚠️ 链接格式规范（重要！）

所有平台的链接**必须使用描述性链接格式**，禁止使用裸链接或错误扩展名：

### 统一格式
```
[点击下载{资源名称}](https://url98.ctfile.com/d/{user_id}-{folder_id}-xxx?p=8069)
```

### 各平台链接要求

| 平台 | 链接格式 | 错误示例 |
|-----|---------|---------|
| 知乎 | `[点击下载考证攻略汇总](链接?p=8069)` | ❌ 裸链接 |
| 微博 | `[点击下载考证攻略汇总](链接?p=8069)` | ❌ `https://...` 裸链接 |
| CSDN | `[点击下载考证攻略汇总](链接?p=8069)` | ❌ `[xxx.txt](链接)` 错误扩展名 |
| 小红书 | 评论区留言+私信获取（不放链接） | ❌ 直接放链接 |

### 常见错误
1. **裸链接**：`https://url98.ctfile.com/...` → 用户看起来不专业
2. **错误扩展名**：`[xxx.txt](链接)` → 实际是文件夹链接，不是txt文件
3. **缺少密码参数**：链接末尾必须有 `?p=8069`

### 检查清单
- [ ] 知乎格式：是否使用纯文本格式？禁止Markdown！
- [ ] 链接是否使用描述性格式？
- [ ] 是否包含密码参数 `?p=8069`？
- [ ] CSDN链接是否去掉了错误扩展名？
- [ ] 小红书是否用评论区引流而非直接放链接？
