# 文件清理技能知识提炼

> 来源：./自定义技能/文件清理技能/SKILL.md + references/

## 踩坑经验

| 坑 | 症状 | 解法 | 严重度 |
|----|------|------|--------|
| 误删重要文件 | 清理后系统/项目异常 | 回收站机制+白名单保护 | 🔴数据安全 |
| 文件被占用无法删除 | 清理中断/报错 | 跳过并记录，不中断流程 | 🟠容错处理 |
| 权限不足 | 部分文件无法清理 | 跳过并记录，提示用户 | 🟠权限处理 |
| 磁盘空间不足 | 清理本身失败 | 优先清理回收站最旧文件 | 🟠空间管理 |

## 最佳实践

### 回收站机制（核心安全保障）
1. **清理不直接删除**：先移入回收站
2. **保留期限**：默认7天，可配置
3. **原目录结构保留**：便于恢复
4. **过期才永久删除**：安全与空间平衡

### 清理类型识别
1. **临时文件**：匹配clean_rules.temp_files中的模式
2. **空文件夹**：文件夹内无任何内容
3. **重复文件**：计算文件hash识别
4. **过期用户上传**：处理完成超过N天

### 白名单保护
- 重要系统文件永不清理
- 项目关键文件受保护
- 支持自定义白名单规则

### 执行流程
1. 读取配置文件`config.json`
2. 确认目录存在
3. 清理过期回收站
4. 扫描垃圾文件
5. 白名单过滤
6. 移入当日回收站
7. 生成清理报告

## 平台规则/限制

### 目录结构
```
./回收站/
  └── YYYYMMDD/       # 按日期存放

./清理记录/
  └── 清理报告_YYYYMMDD.md
```

### 配置项
- 清理规则：哪些文件类型视为垃圾
- 白名单：永不清理的文件和目录
- 回收站保留天数（默认7天）
- 报告保存位置

## 设计决策

| 决策 | 选项 | 最终选择 | 理由 |
|------|------|----------|------|
| 删除方式 | 直接rm | mv到回收站 | 防止误删可恢复 |
| 保留期限 | 永久/临时 | 默认7天 | 安全与空间平衡 |
| 清理执行 | 手动/定时 | 推荐定时（每2天） | 自动化减负担 |
| 报告内容 | 简单/详细 | 详细报告含列表 | 便于审查追溯 |

## 技术架构

### 清理流程
```
执行前检查 → 清理过期回收站 → 扫描垃圾 → 白名单过滤 → 移入回收站 → 生成报告
```

### 报告格式
```markdown
# 清理报告 - YYYY年MM月DD日

## 执行时间
开始：HH:MM:SS
结束：HH:MM:SS

## 回收站清理
永久删除文件数：X
释放空间：XX MB

## 本次清理
移入回收站文件数：X
文件大小总计：XX MB

## 回收站当前状态
总文件数：XX
总大小：XX MB
最早日期：YYYYMMDD
```

## 可升级为热规则的警告

- ⚠️ **清理必须先移入回收站**：禁止直接删除，必须保留7天可恢复
- ⚠️ **白名单文件永不清理**：确保系统安全
- ⚠️ **清理前检查目录存在**：避免初始化问题
- ⚠️ **异常情况跳过不中断**：文件被占用/权限不足时跳过并记录
