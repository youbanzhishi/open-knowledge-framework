# 全流程动漫制作技能

## 任务目标
提供从"一个想法"到"一部动漫短片"的完整制作链路，覆盖创意到成片的全流程。

## 技能信息
- 加载模式：progressive
- 触发关键词：[动漫, 动画, 动画导演]
- 摘要：动漫短片全流程制作，从分镜到成片

## 制作流程（7大阶段）

```
想法 → ①剧本创作 → ②角色设计 → ③场景设计 → ④分镜脚本 → ⑤画面生成 → ⑥语音合成 → ⑦视频合成 → 成片
```

用户可以从任意阶段开始，也可以走完全流程。

---

## 阶段① 剧本/故事创作

### 触发条件
- 用户仅有想法/主题，无剧本
- 用户需要创作新故事

### 执行方式
使用 **novel-creation** 技能完成剧本创作。该技能位于 `.skills/skill_novel-creation/`，sub-agent 需通过 `skill_load` 加载该技能后使用。输出格式为动漫剧本（非小说格式），重点包含：
- 场景描述（便于后续场景设计）
- 角色对白和动作
- 镜头建议（可选）
- 情绪氛围标注

### 剧本格式要求
```
【场景1】侯府新房 - 新婚夜（夜/室内）
氛围：喜庆却压抑

沈晚瑜（怯懦地）：夫君，时辰不早了……
陆砚礼（转身，面露愧疚）：晚瑜，有件事我必须告诉你……

[动作] 陆砚礼双手微垂，眼神躲闪
[情绪] 沈晚瑜感到不安，手指攥紧裙摆
```

### 输出
- 完整剧本文件（Markdown格式）
- 角色列表（名称+简述）
- 场景列表（名称+简述）

## 阶段② 角色设计

### 触发条件
- 用户需要生成角色形象
- 用户需要角色AI绘画提示词

### 执行方式
使用 **anime-storyboard-generator** 技能的角色设计能力。该技能位于 `.skills/skill_anime-storyboard-generator/`，sub-agent 需通过 `skill_load` 加载该技能后使用。

### 生成内容

#### 角色形象
| 字段 | 说明 |
|------|------|
| 角色名称 | 故事中的名字 |
| 外貌 | 根据文本描述或合理推断 |
| 职业/身份 | 在故事中的角色定位 |
| 性格 | 基于行为、语言、心理活动分析 |
| 角色小传 | 背景、动机、关键事件 |

#### 角色提示词（AI绘画用）
必须包含：
- 年龄、性别
- 五官特征（眼睛、脸型等）
- 发型、发色
- 服饰（符合时代背景和角色身份）
- 配饰（饰品、道具等）
- 表情神态
- 画质要求（8k、高分辨率、大师级画作等）
- 风格细节（光影层次、精致的面部细节等）

### 角色一致性锚点（⚠️ 最关键步骤）

**问题**：AI生图每次独立生成，同一角色在不同画面中外观容易漂移。

**解决方案**：角色设计确认后，生成「角色参考图集」，后续所有画面生成都必须引用此图集作为一致性锚点。

#### 角色参考图集（每个角色生成4张）
使用 **image_generate** 生成以下参考图：

**⚠️ 链式生成规则（强制）**：
同一角色的4张参考图必须链式生成以保证一致性，具体步骤：
1. **第1张：全身正面立绘**（768x1024）— 标准站姿，中性表情。**先生成此张，等待用户确认后作为锚点**
2. **第2张：半身特写**（768x1024）— 清晰五官，用于面部一致性。**必须用第1张作为 ref_images 参数输入**
3. **第3张：侧面立绘**（768x1024）— 侧面轮廓，用于多角度画面。**必须用第1张作为 ref_images 参数输入**
4. **第4张：表情变化图**（1024x576）— 同一角色3-4种核心表情并排。**必须用第1张作为 ref_images 参数输入**

链式生成调用示例：
```
# 步骤1：生成锚点图（全身正面立绘）
image_generate(prompt="...", count=1, ...)

# 步骤2-4：使用锚点图作为参考，生成其余3张
image_generate(prompt="...", count=1, ref_images=["./角色/立绘/[角色1]_正面.png"], mode="生图")
image_generate(prompt="...", count=1, ref_images=["./角色/立绘/[角色1]_正面.png"], mode="生图")
image_generate(prompt="...", count=1, ref_images=["./角色/立绘/[角色1]_正面.png"], mode="生图")
```

**⚠️ 强制规则**：
- 第1张（全身正面立绘）必须先完成并由用户确认，才能生成后续3张
- 第2/3/4张生成时，ref_images 必须传入第1张的文件路径
- 如果第1张用户不满意，需重新生成第1张直到确认，再以此为基础生成其余

#### 角色一致性提示词模板
后续所有画面生成时，必须在提示词中包含完整的角色锚定描述：

```
[角色全名], [精确外貌描述: 发色/发型/瞳色/脸型], [精确服饰描述: 颜色/款式/配饰], [当前动作], [当前表情], [场景描述], anime style, [风格锁定关键词], 8k, high quality, cinematic lighting
```

**⚠️ 强制规则**：
- 角色提示词中的外貌描述必须从角色参考图集中提取，禁止凭空编造
- 每次生图时，角色发色、瞳色、服饰颜色必须与参考图集完全一致
- 如果生成结果与参考图集偏差过大，必须重新生成

## 阶段③ 场景设计

### 触发条件
- 用户需要场景描述和绘图提示词

### 执行方式
使用 **anime-storyboard-generator** 技能的场景设计能力。该技能位于 `.skills/skill_anime-storyboard-generator/`，sub-agent 需通过 `skill_load` 加载该技能后使用。

### 核心规则
⚠️ **绘图指令中绝对禁止出现任何人物**，必须是纯环境场景。

绘图指令必须包含关键词：`empty` / `scenery only` / `no people`

### 场景提示词格式

⚠️ **提示词语法说明**：本技能使用 image_generate 工具，画面比例通过提示词中的构图描述影响，提示词中**不包含** --ar/--v 等Midjourney语法。

```
场景 [序号]：[场景名称] - [副标题]
画面描述：[详细的空间、光线、氛围、细节描述]
绘图指令: [纯环境英文提示词，不含--ar/--v参数]
构图: 宽幅横向构图，16:9 画面比例（在提示词中描述'wide angle, panoramic view, 16:9 composition'影响构图比例）
```

### 画面生成
场景提示词生成后，使用 **image_generate** 工具生成场景图：
- 构图：宽幅横向构图，16:9比例（通过提示词中包含 'wide composition, 16:9 aspect ratio' 影响）
- 风格：与角色立绘保持一致
- 每个场景生成1张供用户确认

### 场景一致性锚点

**问题**：同一场景在不同分镜中可能出现色调、构图偏移。

**解决方案**：
1. 每个场景生成1张「场景主图」作为基准（1024x576，16:9）
2. 后续所有在同一场景发生的分镜，**必须使用场景主图作为 ref_images 参数输入**（同时提示词复用场景主图的关键描述），确保色调和构图一致：
   ```
   image_generate(
     prompt="[同场景分镜描述], [色调关键词], anime style, 8k, high quality",
     count=1,
     ref_images=["./场景/场景图/[场景1].png"],
     mode="生图"
   )
   ```
3. 场景主图确认后，提取色调关键词（如"暖黄色烛光"、"冷蓝色月光"），后续同场景分镜强制复用

### 输出
- 场景提示词文档
- 场景主图（1张/场景）
- 场景色调关键词表

## 阶段④ 分镜脚本

### 触发条件
- 用户需要将剧本转化为分镜
- 用户需要Sora视频分镜提示词
- 用户需要表格化分镜脚本

### 执行方式
使用 **anime-storyboard-generator** 技能的分镜能力，包含两种格式。该技能位于 `.skills/skill_anime-storyboard-generator/`，sub-agent 需通过 `skill_load` 加载该技能后使用。

#### 格式A：Sora分镜提示词（JSON格式）
每个分镜包含3个时间段：0-3秒、3-7秒、7-10秒

```json
[
  {
    "分镜序号": 1,
    "sora提示词": "角色:[角色描述]\n【0-3秒】\n镜头:【景别+运镜】[画面描述]\n音效/音乐:[音效描述]\n情绪/氛围:[情绪描述]\n旁白/对话:[台词]\n\n【3-7秒】\n...\n\n【7-10秒】\n..."
  }
]
```

**强制要求**：
- 每个分镜必须包含完整的3个时间段
- 景别必须多样化（远景/全景/中景/近景/特写）
- 运镜必须多样化（固定/推进/拉远/摇镜/晃动/旋转）

#### 格式B：表格化分镜脚本

| 镜头编号 | 场景 | 景别 | 镜头内容 | 预估时长 | 运镜 | 音效/台词 | BGM音量 | 转场 | 衔接 | 备注 |
|:---|:---|:---|:---|:---|:---|:---|:---|:---|:---|

**强制要求**：
- 相邻镜头景别必须变化
- 运镜方式多样化
- 至少包含5种景别和5种运镜方式

### 景别选择规则
| 场景类型 | 推荐景别 |
|---------|---------|
| 开场/转场 | 远景、全景 |
| 对话场景 | 中景、近景 |
| 情绪高潮 | 特写 |
| 动作场景 | 中景+快速运镜 |
| 内心独白 | 特写、大特写 |

### 运镜选择规则
| 运镜方式 | 适用场景 |
|---------|---------|
| 推进 | 强调细节、情绪递进 |
| 拉远 | 展示环境、释放紧张 |
| 摇镜 | 跟随人物、扫描场景 |
| 晃动 | 混乱场景、主观视角 |
| 固定 | 冷静观察、客观陈述 |
| 旋转 | 戏剧化效果、环视环境 |

### 转场设计

镜头之间的过渡方式影响叙事节奏和情绪传达：

| 转场方式 | 适用场景 | ffmpeg实现 |
|---------|---------|-----------|
| 硬切（直切） | 对话场景、快速节奏 | concat直接拼接（默认） |
| 淡入淡出 | 时间流逝、回忆、梦境 | xfade滤镜：`xfade=transition=fade:duration=0.5:offset=[时间]` |
| 溶解（交叉淡化） | 情绪过渡、场景关联 | xfade滤镜：`xfade=transition=dissolve:duration=0.8:offset=[时间]` |
| 黑场过渡 | 章节分隔、重大转折 | 在两片段间插入0.5-1秒黑帧 |
| 白场过渡 | 回忆结束、震撼瞬间 | 在两片段间插入0.3秒白帧 |

转场选择规则：
- 连续对话 → 硬切（快节奏，沉浸感强）
- 场景转换 → 淡入淡出或溶解（给观众适应时间）
- 情绪转折 → 黑场过渡（制造停顿感）
- 紧张→平静 → 溶解（缓和过渡）
- 同场景不同角度 → 硬切（维持紧张感）

在分镜脚本中标注每两个镜头之间的转场方式，合成时按标注执行。

### 镜头衔接设计（⚠️ 声画同步关键）

**衔接列**标注当前镜头与下一镜头的连接方式，决定声音和画面的切换时机：

| 衔接标记 | 含义 | 使用场景 |
|---------|------|---------|
| 直切 | 画面切完再出下一镜声音 | 默认方式，缓慢叙述 |
| J-Cut 0.2s | 声音提前0.2秒切入 | 对话接话、脚步声 |
| J-Cut 0.3s | 声音提前0.3秒切入 | 敲门、拍墙、呼喊 |
| J-Cut 0.5s | 声音提前0.5秒切入 | 惊叫、爆炸、巨响 |
| L-Cut | 画面先切，声音延续 | 回忆、余韵 |

⚠️ **J-Cut是策略B声画同步的核心机制**，在分镜阶段就必须标注，否则合成时声画节奏会对不上。

### 音效设计

⚠️ 只有BGM和配音没有音效，画面会显得"空洞"。音效是让画面"活"起来的关键。

#### 必要音效清单

| 音效类别 | 示例 | 来源 |
|---------|------|------|
| 环境音 | 风声、雨声、虫鸣、城市嘈杂 | freesound.org（免费） |
| 动作音效 | 脚步声、门开合、物体碰撞 | freesound.org |
| 特效音 | 挥剑、魔法释放、爆炸 | freesound.org |
| 过场音 | 嗖声、whoosh、转场音效 | freesound.org |

#### 音效在分镜脚本中的标注

在分镜脚本的"音效/台词"列中，按以下格式标注：
```
[环境音] 淅沥雨声
[动作音] 门推开声
[特效音] 剑出鞘
[台词] 夫君，时辰不早了
```

#### 音效叠加方式

音效文件单独下载/生成，在步骤3（配音处理）之后、步骤4（BGM叠加）之前叠加：
```bash
# 叠加单个音效到指定时间点
ffmpeg -i with_voice.mp4 -i sfx_rain.mp3 \
  -filter_complex "[1:a]adelay=[开始时间ms]|[开始时间ms],volume=0.4[sfx];[0:a][sfx]amix=inputs=2:duration=first[aout]" \
  -map 0:v -map "[aout]" -c:v copy with_sfx.mp4
```

音效音量参考：
- 环境音：0.2-0.3（背景层）
- 动作音效：0.4-0.6（突出层）
- 特效音：0.5-0.7（强调层）

### 镜头衔接逻辑

相邻镜头之间不是简单拼在一起，需要逻辑衔接：

| 衔接方式 | 说明 | 示例 |
|---------|------|------|
| 动作顺接 | 上一镜的动作在下一镜继续 | A镜抬手 → B镜手落下 |
| 视线引导 | 角色看向某处，下一镜展示该处 | 角色看向窗外 → 窗外风景 |
| 声音先入 | 下一镜的声音在上一镜末尾开始 | 画外音先响起 → 切到说话人 |
| 图形匹配 | 上下镜头构图相似 | 圆形月亮 → 圆形眼睛特写 |
| 对比切换 | 强对比制造冲击 | 笑脸特写 → 哭脸特写 |

在分镜脚本中，相邻镜头必须在"备注"列标注衔接方式。如果没有明确衔接逻辑，默认使用硬切。

分镜脚本检查项新增：
- [ ] 相邻镜头是否有衔接逻辑标注
- [ ] 转场方式是否标注
- [ ] 音效是否标注

### 镜头时长计算规则（⚠️ 关键）

**两步走策略**：由于配音音频在阶段⑥才生成，分镜阶段无法获取实际音频时长，因此采用「先估算→后修正」的方式。

#### 第一步：分镜阶段 — 按语速估算

镜头时长基于内容估算：

| 镜头类型 | 时长计算公式 | 示例 |
|---------|------------|------|
| 有台词的镜头 | **字数 ÷ 3.5 + 0.5s**（中文3.5字/秒）| 台词10字 → 10÷3.5+0.5 ≈ 3.4s |
| 有英文台词的镜头 | **词数 ÷ 2.5 + 0.5s**（英文2.5词/秒）| 8个词 → 8÷2.5+0.5 ≈ 3.7s |
| 纯动作镜头 | 动作复杂度 × 1.5s | 简单动作1.5s，复杂动作3-4.5s |
| 环境展示镜头 | 2-3s | 远景/全景 |
| 特写反应镜头 | 1-2s | 表情特写 |
| 转场镜头 | 1-1.5s | 场景切换 |

**节奏控制原则**：
- 总时长目标：1分钟短片 = 约8-12个分镜
- 避免：连续超过3个同景别镜头（节奏单调）
- 避免：单个镜头超过6秒（拖沓）
- 推荐：快慢交替，对话镜头短+动作镜头长

#### 第二步：配音完成后 — 回填修正

阶段⑥配音完成后，用实际音频时长替换预估值：
1. 读取 voice_mapping.md 中每句台词的实际音频时长
2. 更新分镜脚本中的镜头时长：实际时长 = 音频时长 + 0.5s缓冲
3. 如实际时长与预估偏差超过1秒，需调整相邻镜头时长以保持节奏
4. 更新 SRT 字幕文件的时间轴

### 输出
- Sora分镜提示词（JSON文件）
- 表格化分镜脚本（Markdown文件，时长为预估值）
- 镜头时长估算表（含每镜预估时长和依据）
- 配音完成后输出：镜头时长修正表（含实际时长与偏差）

4. **多角色画面**：传入所有出场角色的参考图（但注意：单镜角色不超过2个）

### 6.1 台词配音（TTS语音合成）

#### 执行方式

**首选方案：edge-tts（微软TTS，免费，支持多语言多音色）**

安装命令：
```bash
pip install edge-tts
```

逐句合成命令模板：
```bash
edge-tts --voice "[voice_name]" --text "[台词文本]" --rate="[语速]%" --write-media "[输出路径.mp3]"
```

使用示例：
```bash
# 基础合成
edge-tts --voice "zh-CN-XiaoxiaoNeural" --text "夫君，时辰不早了" --rate="+0%" --write-media "./配音/voice_001_沈晚瑜.mp3"

# 加速合成（语速+20%）
edge-tts --voice "zh-CN-XiaoxiaoNeural" --text "快走！" --rate="+20%" --write-media "./配音/voice_002_沈晚瑜.mp3"

# 减速合成（语速-10%，适合悲伤场景）
edge-tts --voice "zh-CN-YunjianNeural" --text "这件事……我必须告诉你" --rate="-10%" --write-media "./配音/voice_003_陆砚礼.mp3"
```

**情感参数**（通过SSML实现，edge-tts 部分音色支持）：
```bash
# 使用SSML控制情感（仅部分音色支持，如 zh-CN-XiaoxiaoNeural）
edge-tts --voice "zh-CN-XiaoxiaoNeural" --rate="+0%" --write-media "./配音/voice_001.mp3" < ssml_input.txt

# ssml_input.txt 内容示例：
<speak version='1.0' xmlns='http://www.w3.org/2001/10/synthesis' xml:lang='zh-CN'>
  <voice name='zh-CN-XiaoxiaoNeural'>
    <mstts:express-as style='sad' styledegree='3' xmlns:mstts='http://www.w3.org/2001/mstts'>
      夫君，时辰不早了……
    </mstts:express-as>
  </voice>
</speak>
```

**备选方案：create_podcast（仅当 edge-tts 不可用时）**
> ⚠️ create_podcast 生成的是播客节目格式，不是逐句角色台词，仅作为降级备选方案。如需使用，需将所有台词组织为一段完整文本一次性生成，再手动切分。

**备选方案：pyttsx3 或其他 Python TTS 库**
```bash
pip install pyttsx3
```
pyttsx3 为离线方案，音质不如 edge-tts，建议仅在无网络环境下使用。

|---------|------------|------|
| 年轻女性（温柔） | zh-CN-XiaoxiaoNeural | 温柔、标准女声 |
| 年轻女性（活泼） | zh-CN-XiaoyiNeural | 活泼、可爱 |
| 年轻女性（知性） | zh-CN-XiaomoNeural | 知性、成熟 |
| 少女/童声 | zh-CN-XiaoshuangNeural | 少女、清脆 |
| 年轻男性（爽朗） | zh-CN-YunxiNeural | 爽朗、阳光 |
| 年轻男性（热血） | zh-CN-YunxiaNeural | 少年、热血 |
| 中年男性（沉稳） | zh-CN-YunjianNeural | 沉稳、成熟 |
| 中年男性（正式） | zh-CN-YunyangNeural | 新闻播报、正式 |

#### 逐句合成规则
1. 按分镜脚本拆分每句台词
2. 为每句台词标注说话角色和情感
3. 逐句生成音频文件，命名格式：`voice_[镜号]_[角色].mp3`
4. 语速调整建议：
   - 平静对话：+0%
   - 激动场景：+5%~+15%

   - 紧急场景：+10%~+25%

格式示例：
```markdown
| 镜号 | 角色 | 台词 | 音频文件 | 音频时长(s) | 音色 | 语速 |
|------|------|------|---------|-----------|------|------|
| 001 | 沈晚瑜 | 夫君，时辰不早了 | voice_001_沈晚瑜.mp3 | 2.8 | zh-CN-XiaoxiaoNeural | -10% |
| 002 | 陆砚礼 | 晚瑜，有件事我必须告诉你 | voice_002_陆砚礼.mp3 | 3.5 | zh-CN-YunjianNeural | +0% |
```

**⚠️ 配音完成后必须执行**：回填修正阶段④分镜脚本中的镜头时长（详见阶段④"第二步：配音完成后 — 回填修正"）。

### 6.2 声画对齐策略（⚠️ 成片可看性的关键）

**核心原则**：声音必须精确对齐画面，口型同步尽力而为但不是硬性要求。

#### 策略A：声音驱动生成（推荐，口型同步概率最高）

不是先生成画面再配音，而是**用配音驱动画面生成**，让 run_video_task 从音频中推断口型：

1. 先在阶段⑥完成所有台词的TTS配音
2. 使用 run_video_task 生成说话片段时，同时提供：
   - 关键帧图片（角色形象锚点）
   - 配音音频文件（驱动口型）
   - 任务描述模板：
   ```
   基于以下图片和音频生成动漫风格角色说话动画：
   - 参考图片：[关键帧图片路径]
   - 音频文件：[配音文件路径]
   - 要求：画面中的角色嘴巴动作必须与音频对齐，角色保持动漫风格，表情与台词情绪一致
   - 时长：与音频时长一致
   ```

3. 生成后立即检查：播放片段，观察口型是否与声音基本同步
   - 如果同步效果好 → 采用此片段
   - 如果口型明显错位 → 降级到策略B

#### 策略B：静态画面 + 精确配音（稳定保底）

当口型同步效果不理想时，采用此方案：
- 静态关键帧图片 + 配音直接合成视频
- **优势**：声音和画面的时间关系可精确控制
- **观感**：类似高端幻灯片+配音，配以字幕辅助阅读，可观看性足够

合成命令：
```bash
# 静态图+配音合成，时长由音频决定
ffmpeg -loop 1 -i frame_001.png -i voice_001_沈晚瑜.mp3 \
  -c:v libx264 -tune stillimage -c:a aac -b:a 192k \
  -pix_fmt yuv420p -shortest clip_001.mp4
```

⚠️ **策略B的声画同步坑：画面切换和声音到达必须同时发生**

**问题场景**：
- 镜1：小龙瘫坐喃喃自语 → 镜2：隔壁老张拍墙喊"小龙"
- 如果"小龙"的配音编在镜2的clip里，拼接后效果是：
  - 镜1画面播完 → 切到镜2画面 → "小龙"声音才出来
  - **观众感知**：画面先切了，声音慢半拍，或者声音出来了画面才切换，节奏不匹配
- **正确效果**：喊声出现的同时画面切换到老张（声音和画面同时到达）

**解决方案：声音先行（J-Cut效果）**

当下一镜的台词是"突发性"的（喊叫、敲门、惊呼等），应该让声音在当前镜末尾提前切入，制造声音驱动画面切换的效果：

```bash
# 方法：把下一条配音提前叠加到当前clip的末尾
# 例：clip_001（小龙喃喃自语，3秒配音），clip_002（老张喊"小龙"，1.3秒配音）
# 想让"小龙"的喊声在clip_001最后0.5秒就开始出声

# Step1: clip_001不变（自带喃喃自语配音）
# Step2: 在拼接后的merged.mp4上，提前叠加clip_002的配音
ffmpeg -i merged.mp4 -i voice_002_老张.mp3 \
  -filter_complex "[1:a]adelay=[clip001时长ms-提前量ms]|[clip001时长ms-提前量ms],volume=1.0[v2];[0:a][v2]amix=inputs=2:duration=first[aout]" \
  -map 0:v -map "[aout]" -c:v copy with_jcut.mp4
```

**J-Cut参数参考**：
| 声音类型 | 提前量 | 效果 |
|---------|--------|------|
| 喊叫/惊呼 | 0.3-0.5秒 | 声音先到，画面紧随，制造惊吓感 |
| 敲门/拍墙 | 0.2-0.3秒 | 声音先到，画面切到声源 |
| 对话接话 | 0.1-0.2秒 | 自然接话节奏 |
| 缓慢叙述 | 0秒（不需要J-Cut） | 画面切完再出声即可 |

⚠️ J-Cut叠加后，clip_002中的原配音会重复出现，需要在clip_002合成时**去掉该句配音**：
```bash
# clip_002不包含这句配音（纯静态画面，无配音音轨）
ffmpeg -loop 1 -i frame_002.png -c:v libx264 -tune stillimage \
  -pix_fmt yuv420p -t [时长] -an clip_002_silent.mp4
```
然后拼接列表用 `clip_001.mp4` + `clip_002_silent.mp4`，配音通过J-Cut方式叠加。

#### ⚠️ 对用户的诚实说明

制作开始前必须告知用户：
- AI生成的口型同步效果不稳定，可能完美也可能完全错位
- 建议首次制作以策略B为主（稳定可控），积累经验后再尝试策略A
- 无论哪种策略，字幕都是必要的（辅助观众理解台词，也弥补口型不准的问题）
- 真正的口型同步需要专业工具（MuseTalk/AniTalker），不在本技能范围内

#### 声音时间对齐保障（两种策略共用）

无论用哪种策略，配音在最终视频中的时间位置必须精确：

1. 每个动画片段（clip_*.mp4）的时长 = 该分镜的回填后实际时长
2. 如果用策略A：片段内声音和画面已经绑定，拼接即可
3. 如果用策略B：片段内声音和画面已经绑定（ffmpeg -shortest），拼接即可
4. 拼接后的视频配音是天然对齐的，不需要再用adelay重新定位
5. **adelay方案仅在"先拼接无声视频→后叠加独立配音"时才需要**，而我们的流程是"每个片段自带配音→拼接"，天然避免了声画错位

⚠️ **关键改动**：步骤3（叠加配音）的逻辑需要同步修改——
- 如果每个clip已经自带配音音轨（策略A或B都是），步骤3从"用adelay叠加独立配音"改为"确认拼接后视频的配音音轨完整性"
- 只有无台词的纯动作/环境镜头才是无声的，这些片段拼接后自然形成间隔
- adelay方案保留作为"后期补录修改某句台词"的备用手段，不再作为主流程

### 6.3 动作与表情控制

## 阶段⑦ 视频合成

### 触发条件
- 画面生成完成，用户需要合成视频
- 配音和画面都需要最终合成

### 工具分工（⚠️ 关键）

| 工具 | 职责 | 说明 |
|------|------|------|
| run_video_task | 生成动画片段 | 把关键帧图片生成3-8秒的动画片段（每个分镜一个片段），见阶段6.2 |
| ffmpeg | 最终合成 | 拼接片段 + 叠加音频 + 叠加BGM + 烧录字幕 |

**⚠️ run_video_task 不能拼接图片序列+音频+BGM+字幕，最终合成必须用 ffmpeg 完成。**

### 合成流程（6步）

#### 步骤1：生成动画片段（run_video_task）
已在阶段6.2完成，每个分镜对应一个 `clip_[镜号].mp4` 片段。
如未使用口型同步，则用 ffmpeg 将静态关键帧转为视频片段：
```bash
# 静态图片转视频片段（按分镜时长）
ffmpeg -loop 1 -i frame_001.png -c:v libx264 -t [时长] -pix_fmt yuv420p -r 24 clip_001.mp4
```

⚠️ 注意：如果使用策略A或策略B，每个clip_*.mp4 已包含配音音轨，拼接后配音自然对齐。只有纯动作/环境镜头的clip是无声的。

#### 步骤2：拼接视频片段（ffmpeg）
```bash
# 创建拼接文件列表
echo "file 'clip_001.mp4'
file 'clip_002.mp4'
file 'clip_003.mp4'" > concat.txt

# 拼接所有片段
ffmpeg -f concat -safe 0 -i concat.txt -c copy merged.mp4
```

#### 步骤3：检查并完善配音音轨

⚠️ 由于每个动画片段（clip_*.mp4）在阶段⑥已自带配音音轨，拼接后的视频配音已自然对齐，无需额外叠加。

1. 检查拼接后视频的配音音轨是否完整：
```bash
# 检查视频音轨信息
ffprobe -v error -select_streams a -show_entries stream=codec_type,duration -of csv=p=0 merged.mp4
```

2. **⚠️ J-Cut处理（声画同步关键步骤）**

检查分镜脚本中标注的J-Cut标记，对需要"声音先行"的镜头执行J-Cut叠加：

```bash
# 检查分镜脚本的"衔接"列，如有"J-Cut 0.3s"标记，则执行：
# 计算adelay时间 = 前面所有clip累计时长 - J-Cut提前量
ffmpeg -i merged.mp4 -i voice_[镜号]_[角色].mp3 \
  -filter_complex "[1:a]adelay=[累计时长ms-提前量ms]|[累计时长ms-提前量ms],volume=1.0[v2];[0:a][v2]amix=inputs=2:duration=first[aout]" \
  -map 0:v -map "[aout]" -c:v copy with_jcut.mp4
```

3. 如果发现某句配音需要替换或补录（后期修改），使用 adelay 精确定位：
```bash
# 替换/补录单句台词（先移除原音轨中的该段，再叠加新配音）
# 此步骤仅用于后期修改，不是主流程
ffmpeg -i merged.mp4 \
  -i new_voice_003.mp3 \
  -filter_complex "[1:a]adelay=[该句开始时间ms]|[该句开始时间ms][v1];[0:a][v1]amix=inputs=2:duration=first[aout]" \
  -map 0:v -map "[aout]" -c:v copy with_voice.mp4
```

4. 对于纯动作/环境镜头（无配音），拼接后自然形成静音间隔，无需处理

#### 步骤3.5：叠加音效（ffmpeg）

在配音和BGM之间叠加音效层：

1. 从分镜脚本中提取所有音效标注及其时间位置
2. 音效文件来源：freesound.org（免费）或用户提供的音效文件
3. 逐个叠加音效到指定时间点（方法同配音的adelay方式）
4. 如音效较多，用Python脚本批量生成ffmpeg命令

```bash
# 单个音效叠加示例
ffmpeg -i with_voice.mp4 -i sfx_door.mp3 \
  -filter_complex "[1:a]adelay=5000|5000,volume=0.5[sfx];[0:a][sfx]amix=inputs=2:duration=first[aout]" \
  -map 0:v -map "[aout]" -c:v copy with_sfx.mp4
```

音效文件保存在 `音效/` 目录下。

#### 步骤4：叠加BGM（ffmpeg，分段动态控制）

⚠️ BGM不能一条铺到底固定音量，必须根据分镜脚本中标注的BGM音量级别分段控制。

1. 从分镜脚本中提取每镜的BGM音量标注
2. 根据方法选择：
   - 单条BGM + 分段音量控制（1-2分钟短片推荐）→ 详见BGM章节"方法1"
   - 多条BGM交叉渐变（3分钟以上推荐）→ 详见BGM章节"方法2"
3. 生成ffmpeg命令并执行
4. 播放检查BGM音量是否合理（对话时BGM不应盖过人声）

## 背景音乐（BGM）

### 选曲原则
根据剧本整体氛围选择BGM风格：

| 剧本类型 | 推荐BGM风格 |
|---------|-----------|
| 古风/仙侠 | 古筝、笛子、弦乐，悠扬/悲壮 |
| 现代/都市 | 钢琴、吉他，轻快/忧伤 |
| 热血/战斗 | 鼓点、电吉他，激烈/紧张 |
| 恐怖/悬疑 | 低频弦乐、环境音，压抑/不安 |
| 日常/治愈 | 尤克里里、口琴，温馨/轻松 |

### BGM使用规则
- 开场3-5秒：渐入（fade in）
- 对话场景：BGM音量降至0.2-0.3，不盖过人声
- 动作场景：BGM音量0.4-0.6，增强氛围
- 结尾3-5秒：渐出（fade out）
- 场景转换时：可切换BGM或短暂静音

### BGM分段动态控制（⚠️ 影响观感的关键）

一条BGM从头铺到尾音量不变，听感会很平。需要根据剧情节点动态调整BGM音量和风格。

#### 方法1：单条BGM + 分段音量控制（推荐，适合1-2分钟短片）

在分镜脚本中标注每个镜头的BGM音量级别，然后用 ffmpeg 的 volume 滤镜分段控制：

```python
# 根据 分镜脚本.md 中每镜的BGM音量标注，生成ffmpeg命令
# bgm_segments = [(start_sec, end_sec, volume), ...]
# 示例：
# (0, 3, 0.0),     # 开场静音
# (3, 5, 0.5),     # BGM渐入
# (5, 20, 0.3),    # 对话场景，低音量
# (20, 30, 0.6),   # 动作场景，高音量
# (30, 35, 0.3),   # 回归对话
# (35, 38, 0.0),   # 结尾静音

# ffmpeg 命令：用 volume 滤镜 + enable 表达式分段控制
# 对于简单场景，可以用 afade + volume 组合：
ffmpeg -i with_voice.mp4 -i bgm.mp3 \
  -filter_complex "\
    [1:a]afade=t=in:st=0:d=3,\
    volume=0.3:enable='between(t,5,20)',\
    volume=0.6:enable='between(t,20,30)',\
    volume=0.3:enable='between(t,30,35)',\
    afade=t=out:st=35:d=3[bgm];\
    [0:a][bgm]amix=inputs=2:duration=first[aout]" \
  -map 0:v -map "[aout]" -c:v copy with_bgm.mp4
```

⚠️ 注意：volume 的 enable 参数中，多个条件之间可能存在间隙，需确保覆盖全时长。建议用Python脚本生成完整的分段volume滤镜。

#### 方法2：多条BGM交叉渐变（适合3分钟以上或有明显场景切换的短片）

不同场景使用不同风格的BGM，场景之间交叉渐变：

```bash
# 场景1用bgm_01.mp3，场景2用bgm_02.mp3，交叉渐变2秒
# 假设场景1结束于30秒，场景2从28秒开始渐入

ffmpeg -i with_voice.mp4 \
  -i bgm_01.mp3 \
  -i bgm_02.mp3 \
  -filter_complex "\
    [1:a]afade=t=in:st=0:d=3,volume=0.3,afade=t=out:st=28:d=2[b1];\
    [2:a]afade=t=in:st=28:d=2,volume=0.3,afade=t=out:st=[结束时间-3]:d=3[b2];\
    [b1][b2]concat=n=2:v=0:a=1[bgm];\
    [0:a][bgm]amix=inputs=2:duration=first[aout]" \
  -map 0:v -map "[aout]" -c:v copy with_bgm.mp4
```

#### 分镜脚本中的BGM标注

在阶段④分镜脚本的表格中，增加"BGM音量"列：

| 镜头编号 | 场景 | 景别 | 镜头内容 | 预估时长 | 运镜 | 音效/台词 | BGM音量 | 备注 |
|:---|:---|:---|:---|:---|:---|:---|:---|:---|

BGM音量取值：
- `静音`：开场/结尾留白、极度安静的场景
- `0.2`：安静对话、内心独白
- `0.3`：一般对话、日常场景
- `0.4-0.5`：紧张场景、情绪递进
- `0.5-0.6`：动作场景、高潮段落

## 字幕生成

### 字幕内容
从分镜脚本中提取所有台词和旁白，生成SRT格式字幕文件。

### SRT格式
```
1
00:00:01,000 --> 00:00:03,500
夫君，时辰不早了……

2
00:00:04,000 --> 00:00:07,200
晚瑜，有件事我必须告诉你……
```

### 字幕时间轴对齐

字幕时间轴必须基于回填修正后的分镜脚本精确计算：

1. **构建时间轴**：从回填后的分镜脚本中，按镜头顺序累计每镜时长，得到每镜的开始时间
   ```
   镜头1: 0.0s - 3.4s
   镜头2: 3.4s - 6.8s
   镜头3: 6.8s - 9.2s（此镜有台词）
   ...
   ```

2. **定位台词时间**：对于有台词的镜头，台词开始时间 = 镜头开始时间 + 0.3s（短暂留白），台词结束时间 = 台词开始时间 + 实际音频时长

3. **生成 SRT 时间戳**：
   - 字幕出现时间 = 台词开始时间 - 0.2s（提前显示，方便阅读）
   - 字幕消失时间 = 台词结束时间 + 0.3s（延迟消失）
   
4. **合并规则**：相邻字幕间隔不足0.5s时合并为一条显示

5. **自动生成脚本**（推荐用 Python 脚本自动计算）：
   ```python
   # 输入：回填后的分镜脚本（含每镜实际时长和台词）+ voice_mapping.md（含音频时长）
   # 输出：subtitles.srt
   
   def generate_srt(shots, voice_mapping):
       srt_lines = []
       current_time = 0.0
       subtitle_index = 1
       
       for shot in shots:
           shot_start = current_time
           shot_end = current_time + shot['duration']
           
           if shot.get('dialogue'):
               for line in shot['dialogue']:
                   voice = voice_mapping[line['id']]
                   text_start = shot_start + 0.3  # 镜头内留白
                   text_end = text_start + voice['duration']
                   
                   # SRT时间戳格式
                   srt_start = format_srt_time(text_start - 0.2)
                   srt_end = format_srt_time(text_end + 0.3)
                   
                   srt_lines.append(f"{subtitle_index}")
                   srt_lines.append(f"{srt_start} --> {srt_end}")
                   srt_lines.append(line['text'])
                   srt_lines.append("")
                   subtitle_index += 1
           
           current_time = shot_end
       
       return "\n".join(srt_lines)
   
   def format_srt_time(seconds):
       h = int(seconds // 3600)
       m = int((seconds % 3600) // 60)
       s = int(seconds % 60)
       ms = int((seconds % 1) * 1000)
       return f"{h:02d}:{m:02d}:{s:02d},{ms:03d}"
   ```

6. 配音完成回填时长后，需同步更新字幕时间轴

### 字幕样式（建议）
- 位置：画面底部居中
- 字体：思源黑体/黑体，36-42px
- 颜色：白色文字 + 黑色描边（2px）+ 半透明黑色背景条
- 特殊标注：内心独白用斜体，系统音用【】包裹

### 字幕烧录方式（ffmpeg）
使用 ffmpeg 烧录字幕，命令模板：
```bash
# 基础烧录
ffmpeg -i with_bgm.mp4 -vf "subtitles=subtitles.srt:force_style='FontName=SimHei,FontSize=24,PrimaryColour=&HFFFFFF,OutlineColour=&H000000,Outline=2'" final_output.mp4

# 带半透明背景条
ffmpeg -i with_bgm.mp4 -vf "subtitles=subtitles.srt:force_style='FontName=SimHei,FontSize=24,PrimaryColour=&HFFFFFF,OutlineColour=&H000000,Outline=2,BackColour=&H80000000,BorderStyle=4'" final_output.mp4
```

## 风格一致性保障（⚠️ 核心难点）

### 全局风格锁定
阶段②角色设计确认后，提取风格关键词，后续所有阶段复用：

```
风格锁定：[动漫风格/写实风格/水墨风格/...]
色彩基调：[暖色/冷色/...]
线条特征：[粗线条/细线条/...]
画面质感：[赛璐璐/水彩/厚涂/...]
```

### 角色一致性保障机制

**这是整个技能最核心的难点。** AI生图无法保证角色在不同画面中完全一致，必须通过以下策略最大化一致性：

#### 策略1：提示词锚定（基础）
每次生成都必须包含角色的「精确特征字符串」：
```
[角色名], [发色+发型], [瞳色], [肤色], [服装颜色+款式+配饰], ...
```
- 禁止使用模糊描述如"穿着漂亮的衣服"
- 必须精确到"身穿淡青色立领对襟刺绣长袄，下着月白色百迭裙"

#### 策略2：角色参考图集约束（链式生成）
- 生成画面时，在提示词中引用角色参考图集的文件路径
- 使用 image_generate 的 ref_images 参数，以角色参考图作为参考输入
- 优先使用「半身特写」作为面部参考，「全身正面」作为服饰参考
- 参考图集通过链式生成保证内部一致性（详见阶段②链式生成规则）

#### 策略3：同场景批量生成
- 同一场景的多个分镜，尽量在同一次任务中批量生成
- 减少参数波动导致的风格偏移

#### 策略4：用户人工校验兜底
- 每个阶段生成后，**必须**用户确认再进入下一阶段
- 角色一致性偏差大的画面，提供"重新生成"选项
- 用户可手动指定某张图作为标准，后续以该图为参考重新生成

### 场景一致性保障
- 每个场景提取色调关键词，同场景所有分镜强制复用
- 光线方向必须一致（如"左侧暖色烛光"不能下一镜变成"右侧冷色月光"）
- 同场景分镜生成时必须使用场景主图作为 ref_images 参数（不能仅靠文字描述）

### ⚠️ 现实约束说明（对用户透明）

需要让用户知道：
1. **AI生图角色一致性有限**：目前无法保证100%一致，尤其是多角度、多表情场景
2. **建议短片起步**：首次制作建议控制在1-2分钟，约8-15个分镜
3. **减少同镜角色数**：单镜内角色不超过2个，越多越容易出错
4. **特写+中景为主**：远景角色细节少，一致性更容易维持
5. **预留修改预算**：大约30%的画面可能需要重生成

### ⚠️ 积分消耗警告

**最大消耗点：run_video_task（seedance2.0_pro）**

- 这是整个流程中最消耗积分的环节
- 每个分镜生成3-8秒视频片段，消耗量与片段数成正比
- 1分钟短片（8-12个分镜）= 8-12次 run_video_task 调用
- **建议**：
  1. 首次制作建议用策略B（静态画面+配音），零视频生成消耗
  2. 只对关键镜头（如情绪高潮）使用策略A生成动态片段
  3. 先确认所有关键帧满意后再生成视频，避免重试浪费
  4. 考虑使用 seedance1.5_pro 模型（可能消耗更低，需测试对比）

**第二消耗点：image_generate**

- 关键帧生成消耗中等，但数量多（8-15张/分钟短片）
- 加上角色参考图集（4张/角色）和场景主图（1张/场景）
- **建议**：先确认角色立绘再批量生成关键帧，减少重试

### 实测踩坑记录

1. **image_generate 429限流**：连续调用时可能触发，需等待后重试
2. **run_video_task 会自动生成分镜方案**：如果只提供参考图+任务描述，它会自行拆分镜头，不一定按你的意图来。需要在任务描述中尽量精确地描述每个镜头的内容
3. **ffmpeg 相关工具均零消耗**：配音叠加、BGM叠加、字幕烧录、片段拼接全部是本地操作，不消耗积分
4. **🔴 云端写二进制大文件(>100KB)超时**：ffmpeg在/tmp生成MP4后，无法直接cp/write_file到云端目录（超时失败）。解决方案：base64编码→分段复制到云端→cat合并→base64 -d解码还原。实测8秒MP4(196KB)通过base64中转成功写入
5. **🔴 声音和角色形象不适配**：测试中用YunxiaNeural（热血少年）配慵懒小龙，违和感强。修复：新增音色选型原则表（按角色气质匹配，必须试听），不再只按性别选
6. **🔴 画面切换与声音到达不同步**：下一镜的台词编在下一镜clip里，拼接后"声音出来画面才切"。修复：引入J-Cut机制——突发性声音提前0.2-0.5秒切入当前镜末尾，制造声音驱动画面切换的效果。需在分镜阶段标注衔接类型（J-Cut/L-Cut/直切）

### 省积分的推荐流程

如果积分有限，按以下优先级操作：

1. ✅ 先做：剧本→角色设计→场景设计→分镜脚本（全文本，几乎零消耗）
2. ✅ 再做：角色立绘+场景主图（少量图片，必要消耗）
3. ⚠️ 谨慎：关键帧画面（数量多，中等消耗）
4. ⚠️ 谨慎：edge-tts配音（免费但需逐句生成）
5. 🔴 最谨慎：run_video_task视频片段（最高消耗，策略B可完全避开）
6. ✅ 最后：ffmpeg合成（零消耗）
