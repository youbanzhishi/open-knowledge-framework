# 元宝收藏导出技能

> ⚠️ **隐私声明**：本技能包含私人博客链接和签名模板，仅限个人使用，请勿公开分享。

## 技能信息
- 加载模式：progressive
- 触发关键词：[元宝, 收藏, 导出]
- 摘要：元宝App收藏内容批量导出

## 技能概述

从腾讯元宝 App/网页版导出收藏文章，按指定格式整理为 Markdown 文件，支持自动分类和定时任务。

## 适用场景

- 导出腾讯元宝收藏的文章内容
- 将 AI 对话整理为博客文章
- 定期增量导出新收藏内容
- 批量转换收藏为 Hexo 博客格式

## 功能特性

1. **自动登录**：支持手机号验证码登录
2. **导出前去重**：通过索引文件检查，避免重复导出
3. **批量导出**：支持批量或定时增量导出
4. **智能分类**：根据内容自动判断文章分类
5. **格式规范**：统一 Front Matter 格式，适配 Hexo
6. **进度追踪**：记录已导出文章，支持断点续传

## 导出前去重机制（v1.2.0新增）

### 索引文件
- 路径：`./元宝收藏导出/索引.json`
- 格式：`{"已导出标题": ["标题1", "标题2", ...]}`
- 用途：导出前查询，快速判断是否已存在

### 执行流程
```
1. 打开元宝 App → 进入收藏列表
2. 读取索引.json，加载已导出标题列表
3. 滚动浏览收藏，提取待导出文章标题
4. 对比索引，跳过已导出，仅处理新文章
5. 导出完成后，更新索引.json
```

### 优势
- 减少重复工作量
- 避免导出后再去重
- 索引查询速度快

## 导出后自检流程（v1.3.0 必须）

### 1. 文件大小检查
- 对于内容明显过短的文件（<1KB），必须特别检查
- 对比同类文章大小，判断是否缺失内容

### 2. 内容完整性核对
- **开头检查**：有标题、有用户问题或正文开头
- **结尾检查**：有收尾段落，不是突然中断
- **截图对照**：对比截图确认原文长度与导出长度匹配
- **格式检查**：
  - 不应包含 `## 用户问题`、`## AI回复` 标题
  - 不应包含 `**内容由AI生成**` 标记
  - 正文应直接开始，用户问题用引用格式 `> 问题` 标注

### 2.1 发现问题及时修复
自检过程中如果发现格式问题，立即修正：
- 去掉多余的标题行（用户问题、AI回复、内容由AI生成）
- 调整开头格式，确保正文直接开始
- 记录修正情况到处理记录

### 3. 分类归档
导出完成后必须：
1. 判断文章分类（IT技术类/中医养生类/其他类）
2. 移动文件到对应分类子目录：`./元宝收藏导出/{分类}/`
3. 文件命名去掉序号前缀，直接使用标题

### 4. 多版本处理
- 如果存在同名文件，对比大小保留更完整版本
- 更新版本的文件直接覆盖旧版本

### 5. 更新记录
- 更新 `索引.json` 添加新导出标题
- 更新 `处理记录.md` 记录本轮导出情况

### 6. 清理缓存截图（v1.5.0 更新）
自检确认内容完整后，删除导出过程中产生的所有临时文件：
- 删除 `./元宝收藏导出/screenshots_日期/` 目录
- 删除 `./元宝收藏导出/web_screenshots/` 目录
- 删除 `./元宝收藏导出/screenshot_*.png` 等根目录截图
- 删除 `./mobile_use/` 目录下本次任务产生的截图
- 避免占用过多存储空间

**⚠️ 删除旧文件必须移至回收站**：
- 删除旧版本文章时，必须移至 `./回收站/YYYYMMDD/`
- **禁止直接rm删除**，防止误删无法恢复

**⚠️ 注意**：只有确认文章内容完整后才可删除截图，如发现问题需要截图对照则保留

## 常见问题与解决方案

### 问题1：导出内容不完整
- **原因**：未核对原文完整性，只看文件生成就认为完成
- **解决**：执行自检流程，对比截图确认内容完整

### 问题2：文件未分类
- **原因**：导出后直接放在根目录，未按分类归档
- **解决**：导出完成后必须移动到分类子目录

### 问题3：文件名带序号
- **原因**：导出时使用了序号前缀
- **解决**：命名规范为 `{标题}.md`，不带序号

### 问题4：重复导出同名文章
- **原因**：未检查索引文件
- **解决**：导出前检查索引，导出后更新索引

### 问题5：云手机无法复制文字
- **原因**：部分App限制复制功能
- **解决**：改用截图方式，通过OCR识别内容

### 问题5.1：截图导出内容不完整（2026-04-29 严重问题）
- **原因**：截图+OCR方式容易遗漏内容，且删除截图后无法核对
- **案例**：创建空文件与模板脚本修正.md 内容严重缺失
- **影响范围**：2026-04-27和2026-04-28两次导出的所有文章（共10篇）都可能有问题
- **解决**：
  1. **优先使用元宝App的"继续对话+复制"功能**
  2. 在文章详情页点击"继续对话"进入对话页面
  3. 长按对话内容 → 全选 → 复制
  4. 截图仅作为备用方案
  5. 如果用截图导出，必须在确认内容完整后才可删除截图
  6. **待重新导出文章列表**：
     - 创建空文件与模板脚本修正
     - Docker容器间网络通信配置
     - Nginx_RTMP推流Docker部署方案
     - 过度耗气的症状与原因
     - 过度练功导致耗气症状解析
     - 经典功法分类与功效解析
     - 配置通用模板步骤说明
     气功练习的核心原理
     - 中医养生通经络与补气血差异
     - 气血充足的身体表现

### 问题6：文章格式修改后仍有问题
- **原因**：批量sed处理没考虑代码块未闭合，自检流于形式
- **案例**：创建空文件与模板脚本修正.md 多次修改仍有：
  1. AI回复开头没删干净
  2. 代码块未闭合导致签名被包裹
- **解决**：
  1. 修改文章格式时必须逐个检查完整内容
  2. 不能只看开头结尾就认为完成
  3. 特别注意代码块的 ``` 是否正确闭合


### 问题7：复制内容不完整（2026-04-29 严重问题）
- **原因**：长对话只复制了最后部分AI回复，漏掉前面的用户问题和多轮对话
- **案例**：创建空文件与模板脚本修正.md 从139行→470行，缺失用户纠正和优化建议
- **解决**：
  1. 导出长对话时，必须确认复制了完整内容
  2. 对比元宝原始内容确认完整性
  3. 检查是否包含用户纠正消息、优化建议等关键内容

### 问题8：擅自修改原文内容（2026-04-29 严重问题）
- **原因**：自作主张添加原文没有的内容，或修改原文结构
- **案例**：原文只有"极简版本"，却添加了"优化版本"，导致内容与原文不符
- **正确理解**：
  - **导出任务 = 复制，不是创作或优化**
  - 用户要的是**原文原样**，不是我认为"更好"的版本
- **解决**：
  1. **严格复制原文**：原文有什么就导出什么
  2. **不添加内容**：原文没有的，不要自己加
  3. **不删除内容**：原文有的，不要删
  4. **先确认再操作**：不确定原文结构时，先截图查看
  5. **只做格式包装**：Front Matter和版权签名是规范要求，正文内容不改

### 问题9：AI对话痕迹未清理干净（2026-04-29 补充）
- **现象**：文章开头仍有"明白了，我理解错了..."等AI回复开场白
- **原因**：误以为这是正文开头，实际是对话痕迹
- **正确理解**：
  - AI回复的开场白（如"明白了..."、"好的..."）是对用户问题的回应，不是正文主体
  - 正文应从**主体内容**开始（如脚本、说明、方法等）
- **解决**：导出时去掉所有AI对话痕迹，只保留主体内容
