# 小说创作生态 Skill

## 技能信息
- 加载模式：progressive
- 触发关键词：[网文, 小说, 写作]
- 摘要：网文创作全流程，从大纲到成稿

## 能力说明
起点中文网男频玄幻小说半自动创作全流程辅助技能。覆盖从世界观搭建、人设卡构建、百万字大纲生成、章节细纲设计、正文初稿生成、自检扫描、修改迭代到最终润色的完整链路。

**适用场景**：用户需要创作、打磨、迭代起点中文网男频玄幻小说时使用本技能。

**核心模式**：半自动创作——AI负责大纲/人设/细纲/关键场景/自检/润色，用户主导正文修改与定稿。此模式符合起点严禁AI代笔正文的平台政策。

## 技能信息
- 加载模式：progressive
- 触发关键词：[网文, 小说, 写作]
- 摘要：网文创作全流程，从大纲到成稿

## 任务目标
- 本 Skill 用于：网文创作全流程辅助，从书名生成、文案撰写、人设设计，到大纲规划、开篇创作、章节写作、节奏优化、百万字长篇规划
- 能力包含：书名生成、平台文案适配、人设卡构建、大纲生成、黄金开篇、爽点生成、单章写作、对话场景、冲突制造、反转生成、节奏诊断优化、百万字长篇规划
- 触发条件：网文创作各阶段的辅助需求，包括构思、开篇、发展、高潮、结尾、优化等

## 工作流程（严格按顺序执行）

### 第一阶段：世界观与基础设定
1. 与用户讨论核心创意（金手指、世界观、战力体系、核心矛盾）
2. 生成完整人设卡（主角+核心配角，每个角色含：基础信息、外貌特征、性格特质、背景故事、社会关系、成长方向）
3. 生成百万字大纲（卷级结构+每卷核心冲突+爽点分布）
4. 生成起点平台简介文案（书名+一句话简介+长简介+标签）

### 第二阶段：黄金开篇（前三章）
1. 生成前三章详细细纲（每章含：本章目标、节奏设计表、场景拆解、关键对话要点、爽点/钩子、伏笔清单）
2. 逐章写作正文初稿
3. 每章初稿完成后**必须执行自检扫描**（见自检流程）
4. 将自检问题清单发给用户，每个问题提供A/B/C修改选项
5. 用户确认后统一修改
6. 用户审阅修改后版本，可能要求进一步调整
7. 前三章全部完成后，执行一次全局自检（三章联动检查连贯性）

### 第三阶段：后续章节（循环执行）
1. 生成章节细纲 → 用户确认
2. 写正文初稿
3. 执行自检扫描 → 输出问题清单+修改选项
4. 用户确认修改 → 执行修改
5. 用户审阅 → 可能迭代修改
6. 进入下一章

## 操作步骤

### 1. 爆款书名生成器

**输入信息**：
- 题材类型（玄幻/都市/言情/科幻/悬疑/历史等）
- 核心情绪标签（热血/甜蜜/悬疑/复仇/逆袭/救赎等）
- 可选：故事核心梗或设定

**执行流程**：
1. 阅读书名生成策略参考：[references/title-generation-strategy.md](references/title-generation-strategy.md)
2. 基于题材和情绪标签，应用至少10种不同的命名技巧
3. 确保生成的30个书名具有多样性，包含不同风格和长度
4. 每个书名需简短说明其亮点和适用场景

**输出格式**：
```
=== 爆款书名清单（30个）===
【书名1】亮点说明
【书名2】亮点说明
...
```

**质量评估**：
- 是否符合目标平台用户的命名习惯
- 是否准确传达核心情绪
- 是否具有记忆点和传播性

### 2. 小说简介/文案生成器

**输入信息**：
- 故事梗概（核心剧情、主角设定、世界观）
- 目标平台（番茄/晋江/起点/POPO）
- 题材类型
- 可选：字数要求、特殊风格要求

**执行流程**：
1. 阅读平台文案规范：[references/platform-guidelines.md](references/platform-guidelines.md)
2. 根据目标平台的用户画像和文案风格进行调整
3. 确保文案结构符合平台要求（开篇、核心冲突、看点、期待点）
4. 加入平台偏用的关键词和表达方式
5. 提供文案的优化建议（可测试的标题、钩子位置）

**输出格式**：
```
=== [平台名称] 文案 ===

【推荐标题】3-5个备选标题

【正文文案】
（根据平台规范生成的完整文案）

【优化建议】
- 标题优化：...
- 钩子强化：...
- 关键词调整：...
```

**质量评估**：
- 是否符合平台用户阅读习惯
- 是否在3秒内抓住读者注意力
- 是否准确传达故事核心看点
- 是否符合平台的内容规范

### 3. 人设卡生成器

**输入信息**：
- 角色定位（主角/男配/女配/反派/龙套等）
- 题材背景
- 角色核心功能（推动剧情/制造冲突/情感支撑等）
- 可选：参考角色原型、性格关键词

**执行流程**：
1. 阅读人设卡模板：[references/character-template.md](references/character-template.md)
2. 根据角色定位填充完整人设结构
3. 确保人设服务于剧情，具有成长性或冲突性
4. 提供角色与主线剧情的联动点
5. 补充角色的经典台词和标志性动作

**输出格式**：
```
=== 人设卡 ===

【基础信息】
姓名：...
性别：...
年龄：...
身份：...

【外貌特征】
（3-5个最具辨识度的外貌特点）

【性格特质】
核心性格：...
矛盾点：...
成长方向：...

【背景故事】
（与角色功能匹配的背景设定）

【能力设定】
（根据题材补充能力或技能）

【剧情联动】
核心冲突：...
与主角关系：...
关键剧情点：...

【经典台词】
（3-5句体现人设的台词）
```

**质量评估**：
- 人设是否具有记忆点和辨识度
- 是否有合理的矛盾和成长空间
- 是否服务于整体剧情
- 是否符合题材的世界观设定

### 4. 大纲生成器

**输入信息**：
- 题材类型（玄幻/都市/言情/悬疑等）
- 核心设定（世界观、金手指、主角身份等）
- 故事主线（核心目标、核心冲突）
- 预期字数（如：100万字、200万字）
- 可选：参考大纲模板

**执行流程**：
1. 阅读大纲生成指南：[references/outline-guide.md](references/outline-guide.md)
2. 选择合适的大纲结构（起承转合/三幕式/英雄之旅）
3. 规划分卷和每卷的核心冲突
4. 设计爽点分布和节奏安排
5. 提供大纲完整度检查清单

**输出格式**：
```
=== 大纲概览 ===
书名：...
题材：...
字数：...
核心爽点：...

=== 分卷规划 ===
第一卷：[卷名]（第X-Y章）
- 核心冲突：...
- 爽点分布：...
- 关键剧情：...

第二卷：[卷名]（第Y-Z章）
...
```

**质量评估**：
- 主线是否清晰贯穿
- 结构是否完整合理
- 爽点分布是否合理
- 节奏是否张弛有度

### 5. 黄金开篇生成器

**输入信息**：
- 题材类型
- 核心设定（金手指、主角身份等）
- 开篇类型（冲突式/悬念式/反差式等）
- 开篇目标（吸引读者/建立期待等）

**执行流程**：
1. 阅读黄金开篇指南：[references/golden-opening-guide.md](references/golden-opening-guide.md)
2. 应用黄金三章法则
3. 设计开篇冲突或悬念
4. 确保开篇即爽或建立强烈期待
5. 提供开篇优化建议

**输出格式**：
```
=== 黄金开篇（前3章） ===

第一章：[章节名]
- 核心冲突：...
- 爽点/悬念：...
- 建立期待：...

第二章：[章节名]
...

第三章：[章节名]
...

=== 开篇优化建议 ===
- 冲突强化：...
- 期待建立：...
- 节奏调整：...
```

**质量评估**：
- 是否在3秒内抓住读者
- 是否开篇即爽或建立强烈期待
- 是否符合黄金三章法则
- 是否有明显的记忆点

### 6. 爽点生成器

**输入信息**：
- 题材类型
- 当前剧情阶段（开篇/发展/高潮/结尾）
- 爽点类型需求（身份爽/实力爽/金钱爽/情感爽等）
- 可选：爽点强度（小爽/中爽/大爽）

**执行流程**：
1. 阅读爽点生成指南：[references/cool-point-guide.md](references/cool-point-guide.md)
2. 根据剧情阶段选择合适的爽点类型
3. 设计爽点的铺垫和爆发
4. 确保爽点与剧情结合
5. 控制爽点的节奏和频率

**输出格式**：
```
=== 爽点设计方案 ===

爽点类型：...
爽点强度：...
铺垫方式：...
爆发方式：...

爽点情节：
1. [情节描述]
2. [情节描述]
3. [情节描述]

=== 爽点优化建议 ===
- 节奏调整：...
- 强度提升：...
- 效果强化：...
```

**质量评估**：
- 爽点是否有冲击力
- 是否与剧情自然结合
- 铺垫是否充分
- 爆发是否有力

### 7. 单章写作

**输入信息**：
- 章节编号和名称
- 章节目标（推进剧情/揭示真相/制造冲突等）
- 前文剧情摘要
- 可选：参考文风、特殊要求

**执行流程**：
1. 阅读前文已有章节，确保连贯性
2. 根据章节目标设计情节和爽点
3. 按照写作规范执行
4. 初稿完成后执行自检流程
5. 根据自检反馈修改
6. 输出最终版本

**输出格式**：
```
=== 章节内容 ===

（完整章节内容）

=== 章节信息 ===
- 核心爽点：...
- 节奏评价：...
- 遗留问题：...
```

**质量评估**：
- 情节是否推进
- 爽点是否明确
- 节奏是否合理
- 语言是否流畅

### 8. 对话场景生成

**输入信息**：
- 场景类型（冲突/谈判/情感/日常等）
- 参与角色及性格定位
- 场景目标（说服/对抗/揭露等）
- 可选：参考风格

**执行流程**：
1. 分析角色性格和关系
2. 设计对话的核心目的和底线
3. 构建对话节奏（起承转合）
4. 加入动作和表情描写
5. 检验对话是否符合角色

**输出格式**：
```
=== 对话场景 ===

角色A：[动作/表情] "台词..."
角色B：[动作/表情] "台词..."

=== 场景分析 ===
- 核心目的：...
- 角色关系变化：...
- 亮点设计：...
```

**质量评估**：
- 对话是否口语化
- 是否符合角色性格
- 是否有推进剧情
- 动作表情是否自然

### 9. 冲突制造

**输入信息**：
- 冲突类型（人物冲突/内心冲突/外部冲突）
- 参与角色
- 冲突导火索
- 可选：预期结果

**执行流程**：
1. 阅读冲突制造指南：[references/conflict-creation-guide.md](references/conflict-creation-guide.md)
2. 确定冲突的核心矛盾
3. 设计冲突的升级路径
4. 准备冲突的爆发点
5. 规划冲突的解决或延续

**输出格式**：
```
=== 冲突设计 ===

冲突类型：...
核心矛盾：...
导火索：...

冲突升级路径：
1. [阶段1]
2. [阶段2]
3. [阶段3]

爆发点：...
结果/延续：...
```

**质量评估**：
- 冲突是否合理
- 升级是否自然
- 爆发是否有力
- 是否符合角色

### 10. 反转生成

**输入信息**：
- 反转类型（身份反转/命运反转/真相反转等）
- 预期反转力度（小/中/大）
- 当前剧情阶段
- 可选：反转方向建议

**执行流程**：
1. 阅读反转生成指南：[references/twist-generation-guide.md](references/twist-generation-guide.md)
2. 确定反转的伏笔位置
3. 设计反转的触发条件
4. 构建反转的冲击力
5. 安排反转后的剧情走向

**输出格式**：
```
=== 反转设计 ===

反转类型：...
反转力度：...

伏笔设计：
- 伏笔位置：...
- 伏笔内容：...

触发条件：...
反转内容：...

后续影响：...
```

**质量评估**：
- 伏笔是否隐蔽
- 反转是否意外
- 是否符合逻辑
- 后续影响是否深远

### 11. 节奏诊断与优化

**输入信息**：
- 待诊断章节或全文
- 预期节奏类型（快节奏/慢热/张弛有度）
- 问题描述（如有）

**执行流程**：
1. 阅读节奏诊断指南：[references/rhythm-diagnosis-guide.md](references/rhythm-diagnosis-guide.md)
2. 分析当前节奏状态
3. 识别节奏问题（过快/过慢/平淡）
4. 提供优化方案
5. 预测优化后的效果

**输出格式**：
```
=== 节奏诊断报告 ===

当前节奏评估：...
主要问题：...

优化建议：
1. [建议1]
2. [建议2]
3. [建议3]

预期效果：...
```

**质量评估**：
- 问题识别是否准确
- 建议是否可行
- 优化后是否平衡

### 12. 百万字长篇规划

**输入信息**：
- 题材类型

**执行流程**：

**输出格式**：
```

- [角色A]：阶段性任务...
- [角色B]：阶段性任务...

爽点分布：
- 第X卷：[爽点类型]
- ...

高潮设计：
- [高潮点1]
- [高潮点2]
...
```

**质量评估**：
- 结构是否完整
- 成长线是否清晰
- 节奏是否合理

## 半自动创作模式说明

### 分工表
| 环节 | AI负责 | 用户负责 |
|------|--------|----------|
| 世界观/人设/大纲 | 生成+迭代 | 确认+补充细节 |
| 章节细纲 | 生成+调整 | 确认+提要求 |

## 写作规范（铁律，必须遵守）

### 1. 排版规则（⚠️ 曾经严重违反，绝不许再犯！）
- **不用`---`分隔线**（闪回可用`——`），场景切换用省略号`……`独占一行或空行
- **单段不超过3句**（约60字），超过就拆
- **同一动作链不能拆段**，必须合并在同一行——拆开会割裂画面感+丢失主语指代
- **主语指代必须清晰**，拆段后尤其要注意"他"指的是谁
- **段落之间用空行分隔**
- **标题行必须独占一行**，标题后必须空一行再接正文

### 1.5 段落合并铁律（⚠️ 2026-05-01新增，因为3-72章全部犯了这个错）
- **禁止"断行诗"写法**——一句一段只在以下情况使用：
  - 拟声词独占一行（如"轰。""砰。""滚烫。""脚步声。"）
  - 关键转折点（如"陆渊动了。""他死了。"）
  - 场景过渡标记（如"……""夜幕降临。"）
- **日常场景2-3句合一段**，参考万族之劫的段落节奏：
  - ❌ 断行诗（每句独占一行）：
    ```
    他转过身，看着陆渊。

    那双眼睛里有什么东西在翻涌。

    "我还有一件事要告诉你。"

    萧无痕开口，声音比之前更沉。

    "关于万契宗。"
    ```
  - ✅ 正确合并：
    ```
    他转过身，看着陆渊。那双眼睛里有什么东西在翻涌。

    "我还有一件事要告诉我。"萧无痕开口，声音比之前更沉："关于万契宗。"
    ```
- **对话+动作/表情必须合并为一段**：
  - ❌：`"不给。"` 独占一行 + `两个字，沙哑，却很清晰。` 独占一行
  - ✅：`"不给。"两个字，沙哑，却很清晰。`
- **短描写连续动作必须合并**：
  - ❌：`陆渊的嘴角扯出一个弧度。` 一行 + `血从唇角淌下来。` 一行
  - ✅：`陆渊的嘴角扯出一个弧度，血从唇角淌下来。`
- **合并时的标点处理**：
  - 两个独立句子用句号+空格连接：`他转过身。那双眼睛里有什么东西在翻涌。`
  - 对话+动作用冒号或直接拼接：`他开口，声音更沉："关于万契宗。"`
  - 同一动作的连续描写用逗号连接：`陆渊的嘴角扯出一个弧度，血从唇角淌下来。`
- **合并=把多行变成一行，不是删空行！**

### 2. 爆发点单独成行
- 暴力动作/冲击爆发点独占一行：踢、踹、打、震飞、金光冲天等
- 后果跟下一行
- 示例：
  ```
  又是一脚。
  陆渊摔倒在地，蜷缩着，像一只被踩烂的虫子。
  ```

### 3. 画面与表情
- **情绪=表情细节+身体反应**，禁止直接写情绪词（如"痛苦""愤怒""绝望"）
- 用画面传达情绪，如"龇牙笑的苦涩"代替"感到苦涩"
- 万族之劫风格用于日常/情感场景，牧神记风格用于战斗/紧张场景

### 4. 表情必须符合角色身份和场景
- 重伤者姿态是警惕防备，不是审讯
- 上位者蔑视用"不识抬举"而非脏话

### 5. 对话语气（核心经验）
- 对话语气必须联系角色定位和世界观，不能脱离人物性格硬加语气
- **陆渊的硬是压出来的不是吼出来的，安静比咆哮更有力**
- 微调对话语气时：补几个关键字带画面即可，不需要大改，不能脱离上下文
- **动作已经表达了情绪时，台词反而要收**——越平静越吓人

### 6. 对话内容避坑
- **严禁"你爹的种"这类逻辑不通的废话式表达**——谁不是爹妈的种？既没信息量也没压迫感
- 知道对方来历的角色，用身份点破（如"陆苍的儿子"）——点破身份本身就是威慑
- 上位者蔑视用"不识抬举""不知死活"这类居高临下的口吻，不是骂街
- **每句对话都要问：这个角色在这种身份下，真的会这样说吗？**

### 7. 力度词规则
- "收紧""攥紧"这类动作词不需要再加力度修饰（猛地、用力）
- 结果已经说明力度（如"骨头碎裂的声音"已经说明"手指收紧"有多紧），加力度词是多余的

### 8. 结尾经验
- 章末用画面/动作收尾，不要用总结性句子
- **严禁**"故事才刚刚开始""一切才刚刚开始"这类AI味结尾
- 好的章末是留白——画面停在那里，读者自己会往下想
- **卷末收束**：卷的最后一章用反复出现的画面意象呼应开头，形成闭环
- 类似禁用："新的篇章""命运的齿轮开始转动"

### 9. 去AI味（最高优先级）
- **禁止词表**：仿佛、宛如、似乎、不禁、竟然、不由得、居然
- **"似乎"和"像是"同类**，不能用"似乎"替换"像是"来绕过
- **"像是"三种用法处理**：
  - 角色/读者已知的事实→删掉"像是"，直接写
  - 角色不确定的判断→可以用"像是"，但优先用更具体的感知词
  - 比喻修辞→可以用，但要精准，不要堆砌
- **禁止排比渲染情绪**（如"多疑。冷静。算计。"这种三词并列→合并为一句）
- **禁止大段内心独白**
- **禁止"没有X没有Y只有Z"的否定+结论式旁白**
- **禁止"像是"弱化表述**——已知的事不用"像是"：
  - 已知代价是折寿，就不用"像是有什么东西在蚕食他的生命"
  - 已知敌人是敌人，就不用"像是带着恶意"
- **禁止旁白评价词**（如"惊人地相似"→"如出一辙"）
- **禁止"不知过了多久"时间模糊套路**→用环境变化暗示时间流逝
- **禁止文艺词堆砌**→用精准词替换（如"蚕食"→"啃"更有画面）
- **同一段落内禁止重复词汇**：
  - 如"蔓延"出现3次→替换后两次
  - 如"他的眼神"出现2次→合并或替换
- **禁止两句说同一件事**：
  - 如"不是现在。现在不是崩溃的时候。"→留一句
  - 如"他很害怕。他害怕得发抖。"→合并为一句
- **抽象解释要转化为画面**：
  - ❌："不是生理上的疼痛，是更深层的警告"
  - ✅："纹路底下有什么东西在动，在往骨头里钻"

### 9.5 高级幽默标准（⚠️ 2026-05-04修订，从"互相内涵"升级）
- **幽默从人物性格和处境里自然"长"出来的，不是"写"进去的**
- 互相内涵可以是表现形式，但不能硬加
- **判断标准：删掉这段对话故事仍成立，那这段就是硬塞的，该删**
- 前三章的幽默为什么好？因为沈炼越认真越较劲，读者越觉得好笑——幽默是人物自己活出来的
- 后面的幽默为什么死？因为把他写成了苦情主角，笑点自然就死了
- **沈炼式幽默三合一**：外憨式（装傻充愣占便宜）、怕死式（自我调侃）、智谋式（吐槽拆招）

### 10. 禁止假煽情
- 不用"承诺""还债""一条命"等套路，动机用实际理由驱动
- 角色的情感表达必须符合身份和处境，不拔高不鸡汤

### 10.5 人物形象活出来（⚠️ 2026-05-04新增）
- **人物形象是活出来的，不是写出来的**
- 不用情绪标签告诉读者角色感受（如"五味杂陈""若有所思""心头一震"），用动作和选择让读者自己体会
- **不要让沈炼自己说出来"我想苟""我想低调"**——越不说读者越觉得他苟
- 用行为侧面展现：缩脖子、躲阳光、往墙角靠……这些动作比说"我想苟"有力一百倍
- **沈炼核心驱动力**：被动卷入、只想脱身但越陷越深。不要让他自己说出来，用行为侧面展现

### 11. 动作描写不能短期重复（⚠️ 2026-05-04新增）
- **同一动作5章内不超过1次**
- "揉太阳穴""捏鼻梁""按眉心"这些动作要轮换使用，不能反复出现同一个
- 发现重复时立即替换为不同但等价的动作

### 12. 角色身份和关系必须前后一致（⚠️ 2026-05-04复原）
- 已确立的身份、地位、师承、辈分关系不能前后矛盾
- 配角的身份信息（如外门弟子/内门弟子/长老）必须跨章节一致
- 发现矛盾时以首次出现为准，后续章节修正

### 13. 人物关系不能凭空产生（⚠️ 2026-05-04复原）
- 两个角色不能突然表现出亲密/敌对关系，除非之前有铺垫
- 关系的建立需要过程：相识→互动→建立关系
- 禁止"因为剧情需要"就让两个角色突然成为朋友/敌人

### 14. 写前必须通读所有已有章节（⚠️ 2026-05-04复原）
- 每次写新章节前，必须通读全部已有章节
- 目的：确保不与已写内容矛盾、不重复已有信息、延续人物性格和语气
- 不通读就写=制造逻辑错误

### 15. 幽默不能断档（⚠️ 2026-05-04复原）
- **每章至少1个笑点，每3章至少1个让人笑出声的桥段**
- 老前辈的吐槽不能断，这是最大的幽默来源之一
- 外憨装傻坑人的戏份得持续有
- 炼化/炸炉场景要继续有，这是书名核心笑点
- 重情节后必须有轻段落回弹，紧张完了来一段搞笑，不能一路紧绷
- 幽默断了=读者觉得主角变苦情，节奏就死了

### 16. 禁止信息重复（⚠️ 2026-05-04复原）
- 已交代过的对话/事实，新章节不能重新走一遍流程，写前必须通读已有章节
- 跨章节也不能重复——读者记得前面写过，再来一遍只会觉得水
- 如需唤起读者记忆，用一笔带过（如"上次那事"），不能重新展开
- 发现重复时：追问解决方案而非重复提问；提醒而非重新告知

## 七猫发布流程

### 平台信息
- 七猫作家助手APP，账号验证码登录
- 要求独家首发，不能在其他平台发过
- 投稿门槛：开篇2万字正文+大纲
- 签约需2万字后在线申请

### 手机上传方案（网页剪贴板法）
手机直接输入长文本会截断（type模拟打字方式），解决方案：
1. **生成HTML**：将章节正文生成带"一键复制"按钮的HTML文件，保存到 `./web/chXX.html`
   - 跳过md文件第一行`#`标题，正文放入`<div id="content">`
   - `white-space: pre-wrap` 保留空行格式
   - 橙色"一键复制"按钮，使用`navigator.clipboard.writeText()`
2. **生成公网URL**：用 `file_to_url` 工具将HTML转为 `https://www.coze.cn/s/xxx` 公网链接
3. **手机操作**：QQ浏览器打开URL → 长按正文全选复制 → Home键 → 七猫APP → 新建章节 → 粘贴 → 发布
4. "一键复制"按钮在QQ浏览器中**不可靠**（静默失败，按钮变绿但剪贴板未更新），优先使用手动长按全选复制

### ⚠️ 上传铁律（2026-05-04血泪教训，必须遵守！）

**1. 顺序不可跳跃**
- 必须按章节号顺序上传！前面章节没发完，绝对不能发后面的！
- 跳章会导致读者看到内容断片，严重违反平台规则
- 踩坑记录：2026-05-04，22-31章还没上传就跑去发32-41章，被主人痛骂

**2. 粘贴后必须检查正文开头**
- 粘贴完成后，先看正文开头几句话是否与当前章节匹配
- 之前出现过剪贴板残留上一章内容的情况
- 不匹配则删除重新复制粘贴

**3. 标题只输名称，不加"第X章"**
- 七猫新建章节时自动生成编号前缀
- 只需输入章节名称部分，如"莫老现身"而非"第22章 莫老现身"
- 如果自动编号与实际不符，清空标题框只输名称

**4. 标题必须与内容一致**
- 七猫APP的标题输入框有时难以删除自动填充的文字
- 方法：长按标题文字→全选→删除键；或双击→全选→删除；或连续按退格键
- **标题和内容必须匹配**，绝不能带着错误标题发布
- 如果标题实在改不对，宁可暂不发布，等确认标题正确后再提交

**5. 剪贴板可能丢失**
- QQ浏览器复制后切到七猫APP，剪贴板内容可能丢失
- 如果粘贴时发现为空，需要重新回QQ浏览器复制
- 可以尝试切换浏览器（手机自带浏览器/Chrome）看是否更稳定

**6. 上传完成必须验证内容（开头+结尾双重检查！）**
- 每章上传后检查：**正文开头30字 + 正文结尾30字**
- 开头确认内容正确（与原文匹配，不是其他章节的内容）
- **结尾确认没被截断**（与原文结尾匹配，不能只到一半就没了）
- 不能只看标题就认为上传成功
- **批量上传后也要抽查明细**：不能只验证最后一章，中间章节可能被截断或内容错位
- 踩坑记录：2026-05-05发现之前上传的章节只验了开头没验结尾，无法确认是否截断

**7. 上传前必须验证前置章节**
- 每轮新上传前，先检查验证记录表（`./玄幻小说-异界大陆/上传验证记录.md`）
- 确认之前所有章节的**开头+结尾+标题与内容匹配**都已验证通过
- 如果有⏳待验证的章节，必须先补验再继续上传
- 验证项包括：章节序号、标题、开头内容、结尾内容是否与原文一致

### ⚠️ 七猫章节标题规则（关键踩坑，2026-05-01）
七猫作家助手新建章节时**自动生成"第X章 "编号前缀**！
- 标题输入框已有自动编号，**只需输入章节名称部分**
- 例如：自动显示"第12章 "，只需输入"代价与适应"，最终显示"第12章 代价与适应"
- **不要输入完整的"第十二章 代价与适应"，否则标题变成"第12章 第十二章 代价与适应"重复**
- 如果自动编号与实际章节不符，清空整个标题框，只输入章节名称
- 踩坑记录：2026-05-01上传第11章时输入完整标题导致重复，产生错误草稿需手动删除

### 格式自检（上传前必做）
上传前对md文件做格式自检，检查项：
1. **行首多余句号**：行首不能有`。`
2. **标题行与正文混行**：`#`标题行不能超过30字
3. **断行诗检测**：连续4行以上每行只有一句短句（<30字、句号结尾、不含对话），排除拟声词行、对话行、场景过渡行

### 格式修复流程（发现问题时）
1. **先备份**：将原文件复制到 `./备份/` 目录，文件名加日期后缀（如 `第X章-章节名_20260501.md`）
2. **修改源文件**：在原文件上修改
3. **同步HTML**：重新生成对应的 `./web/chXX.html`
4. **告知主人**：说明修改了哪些文件、改了什么内容

### 每日上传节奏
- 新书期：每日更新2章（早8点+晚8点）
- 公众期：每日更新1章
- 推荐期：根据推荐节奏调整
- 完结期：每日1章直到完结

## 自检流程（每章初稿完成后必须执行）

### 7维度扫描
1. **去AI味**：仿佛/宛如/似乎/不禁/竟然、排比渲染、旁白评价词、弱化表述、重复表达
2. **对话辨识度**：不同角色说话是否有区分度、语气是否符合身份
3. **场景画面**：画面是否具体、情绪是否用画面传达而非直接说、抽象解释是否有画面替代
4. **情绪推进**：每段情绪是否递进、是否有原地踏步、高潮是否充分释放
5. **节奏问题**：段落是否过长需要拆分、爆发点是否单独成行、场景分隔是否合理
6. **角色一致性**：行为/语气是否符合人设、表情是否符合身份和场景
7. **前后连贯**：主语指代是否清晰、动作链是否连贯、伏笔是否埋好

### ⚠️ 去重扫描（必须执行，写在7维度之后）
通读不能防止重复。每章写完后，必须用grep搜索以下类型的词，与已有章节对比：

```bash
# 1. 场景描写去重：搜本章出现的环境描写关键词
grep -n "老槐\|槐树\|竹林\|月光\|夕阳" ./网文创作/*.md

# 2. 动作描写去重：搜本章出现的关键动作词
grep -n "揉太阳穴\|深吸一口气\|嘴角上扬\|眼珠转了转" ./网文创作/*.md

# 3. 信息重复去重：搜本章涉及的关键事实/解释
grep -n "邪气引\|续命\|观万界\|万界熔炉" ./网文创作/*.md

# 4. 对话去重：搜本章角色的关键台词
grep -n "被动卷入\|只想苟\|前辈说得对" ./网文创作/*.md
```

**操作步骤**：
1. 从新章节中提取5-10个关键描写词/事实词
2. 用grep搜索所有已有章节文件
3. 如果搜到完全相同或高度相似的句子，标记为重复，必须修改
4. 重点检查：同一环境描写、同一动作、同一事实解释出现两次以上

**铁律**：已交代过的信息不能重新走一遍流程，只能用"提起"或"暗示"的方式简要带过

### 输出格式
- 问题按严重程度排序：高→中→低
- 每个问题必须给出A/B/C三个修改选项
- 高优先级问题单独列出，中低优先级另列

### 修改确认
- 用户确认后统一修改
- 修改完成后输出**修改对比表**

### 批量创作自动修改策略
- 批量创作多章时，对高频共性问题（如"揉太阳穴"重复、断行诗、"像是"弱化）自动修改，不必每处都等用户确认
- 自动修改范围仅限于铁律中明确禁止的问题，不确定的仍需用户确认
- 批量修改后输出完整修改对比表

### 新问题更新机制
- 发现新的写作问题（新AI味特征、新重复动作、新逻辑坑）时，必须立即更新SKILL.md对应章节
- 同时更新references下的对应文件（去AI味检查清单/写作经验案例库）
- 不等用户提醒，不等话题结束

```

├── 第X章-章节名.md        # 各章正文（最终版）

├── 备份/                  # 修改前的备份文件
├── web/                   # 七猫上传用的HTML文件
│   └── chXX.html          # 每章带一键复制的网页
├── 上传进度.md            # 七猫上传进度跟踪
└── 章节URL映射.md         # 章节HTML公网URL映射
```

### 命名规则（⚠️ 2026-05-01新增）
- **细纲**：`第X章细纲.md`（如`第12章细纲.md`）
- **正文**：`第X章-章节名.md`（章节名2-4个字，体现本章核心事件，如`第12章-代价与适应.md`）
- **自检**：`前X章自检清单.md`（如`前12章自检清单.md`）
- **HTML**：`chXX.html`（XX为2位数字章节号，如`ch12.html`）
- **备份**：`第X章-章节名_YYYYMMDD.md`（如`第12章-代价与适应_20260501.md`）

### 备份规范
- 每次修改章节前，先备份原版本
- 备份命名格式：`章节名_v1.md`、`章节名_v2.md`
- 重大修改后立即备份
- 保留最近5个版本

## 注意事项

- **智能体主导**：所有创作内容由智能体基于用户需求和参考文档生成，无需外部工具
- **质量优先**：生成的30个书名、文案、人设卡应具备实际可用性，而非简单堆砌
- **平台适配**：文案生成时必须严格按照目标平台的用户画像和风格规范调整
- **创作自由度**：在参考文档基础上，鼓励创新和差异化
- **迭代优化**：用户反馈后可快速调整，提供多个版本供选择
- **合规性**：确保生成内容符合各平台的内容审核规范

### 写作质量要求（重要）

在生成文章、对话、场景时，必须遵循以下高级写作技巧：

**1. 避免模板化词汇**
- ❌ 少用：充满了、坚定、不甘、难以置信、震惊、愤怒、绝望、悲伤、冷冷地说、愤怒地说、嘲讽地说
- ✅ 多用：具体动作、场景细节、对话表现

**2. 动作/场景代替解释性叙述**
- ❌ 错误：林风眼中充满了愤怒，紧紧握住拳头，不甘地看着赵峰。
- ✅ 正确：林风盯着赵峰，拳头慢慢收紧，指节发白。他呼吸变得急促，但没有说话。

**3. 对话更口语化，带停顿**
- ❌ 错误："我不会放过你的。"
- ✅ 正确："我不会，放过你的。" 或 "我不会放过你的..."

**4. 情绪不直接喊出来，用细节表现**
- ❌ 错误：苏晴很悲伤，眼泪流了下来。
- ✅ 正确：苏晴坐在床边，看着手里的照片。她的肩膀微微颤抖，但没有发出声音。一滴眼泪掉在照片上，晕开了苏晴的笑容。

**5. 主角的狠劲"藏在平静里"，更高级**
- ❌ 错误：林风愤怒地说："我要杀了你！"他狠狠地握紧拳头，眼神凶狠。
- ✅ 正确：林风看着赵峰，没有说话。他慢慢走到赵峰面前，蹲下。"赵峰。""林风，你...你不敢..."死吧。

### 📋 问题修复手册（2026-05-05实战经验）

#### 问题1：章节缺失导致整体错位
**症状**：第N章没传上去，但第N+1及之后的章节都上传了，导致所有后续章节的位置偏移一位（标题是N+1章的标题，内容是N章的内容）
**案例**：第22章"莫老现身"上传失败被跳过，第23-26章全部向前错位一位
**修复步骤**：
1. 先用手机查实七猫上的实际章节列表，确认缺失的是哪一章
2. 对比本地原文，确认错位范围（从缺失章到最后一章都偏了）
3. **不能简单插入新章节**——七猫新建章节只能加到末尾，无法插入中间位置
4. **正确方案**：逐章编辑错位的章节，把每章的标题和内容改回正确的
   - 第22章（原内容是第23章的）→ 编辑标题和内容为第22章正确的
   - 第23章（原内容是第24章的）→ 编辑标题和内容为第23章正确的
   - 以此类推...
   - 最后一章新建补上
5. 编辑时从HTML公网URL复制正确内容（用file_to_url转URL，浏览器打开复制）
6. 修复后逐章验证开头+结尾

#### 问题2：正文被截断（只传了一半）
**症状**：上传/修改后正文不完整，结尾不是原文的最后一句
**案例**：修复错位时粘贴的内容被截断，第22-26章结尾都不对
**根因**：从HTML页面复制时，页面没有完全加载/滚动到底，选择区域没覆盖全文
**修复步骤**：
1. 在浏览器打开HTML公网URL后，**先滚动到页面底部**，确保所有内容加载
2. 再回到顶部，长按拖选到底全选，复制
3. 粘贴后**必须滚到正文底部验证结尾**是否完整
4. 如果结尾不对，删掉重粘，不要提交不完整的内容
5. 七猫APP编辑器删除正文困难时的替代方案：
   - 尝试长按→全选→删除
   - 或用输入法自带全选功能
   - 或直接在旧内容前面粘贴新内容，然后删掉后面多余的
   - **禁止尝试七猫网页版**——网页版登录验证过不了，浪费时间

#### 问题3：验证记录表的使用
**作用**：防止遗漏验证、避免带着问题继续上传
**文件位置**：`./玄幻小说-异界大陆/上传验证记录.md`
**使用时机**：
- 每章上传/修改后立即记录验证结果
- 每轮新上传前先检查此表，确认所有前置章节已✅
- 发现⏳待验证的章节必须先补验

#### 问题4：手机查章节不全
**症状**：手机查七猫只看到10章，实际有26章
**根因**：章节列表需要下拉滚动才能看到更多，手机任务只看了第一屏
**修复**：查章节时必须切换为正序排列，然后持续向下滚动直到列表底部
