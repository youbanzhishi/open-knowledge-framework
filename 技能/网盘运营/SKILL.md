# 网盘运营技能

> 由「网盘变现主理技能」「网盘自媒体运营技能」「网盘资源发布技能」合并而来，统一管理网盘项目的资源规划、内容生产、平台运营、发布调度及团队协作。

## 技能信息
- 加载模式：progressive
- 触发关键词：[网盘, 运营, ctfile]
- 摘要：城通网盘运营与资源管理

## 技能概述
网盘变现项目的核心枢纽，负责资源规划、内容生产、发布调度及团队协作。既能统筹调度，又能亲自执行，是项目的"主理人"角色。

## 功能特点
- 🔍 自动搜索合规热门资源
- 📤 城通网盘API上传（优化流程：先建文件夹再上传）
- ✍️ 多平台文案适配（知乎、微博、小红书、CSDN、博客园）
- 📱 自动发布到自媒体平台
- 📊 每周数据统计与策略调整
- 🎯 平台引流机会发现与运营（知乎/小红书）

## 核心分工

| 角色 | 职责 | 动作 |
|-----|------|------|
| **运营职能** | 发现需求、发布回答、记录复盘、**质量监督** | 检查平台 → 提需求 → 质量检测 → 发回答 → 更新记录 |
| **主理职能** | 收集资源、准备内容、上传分享 | 准备资源 → 补充公开链接 → 创建文件夹分享 |

> ⚠️ **质量监督责任**：主理职能偷懒时（占位符链接、敷衍内容），必须监督其改正，不合格的资源不能发布！

## 使用场景
- 每日定时发布新资源
- 批量上传资源并生成分享链接
- 多平台内容分发
- 数据分析与运营优化
- 平台引流：发现知乎/小红书引流机会，匹配资源发布回答

## 执行流程

### 完整运营流程（严格按此顺序执行！）
1. **补齐未来3天内容** → 检查待发布目录，确保未来3天都有资源和文章，缺失的先补上（必须第一步！不能先发当天再补）
2. **先收资源再写文章** → 必须先收集/创建资源文件，再根据资源实际内容编写文章
3. **发布当日** → 补齐完成后，再发布当天文章 → 根据当前发布模式执行：
   - **手动发布模式（当前）**：将各平台文案内容推送给用户（知乎纯文本版/微博/小红书标题+正文/CSDN Markdown版），用户自行发布后告知分享链接
   - **自动发布模式**：云手机App发布各平台 → 收集各平台文章分享链接发给用户确认 → 验证发布是否真的成功
4. **发布后更新** → 移动文章到已发布目录 → 更新发布记录

⚠️ **严禁先发当天再补未来3天！** 顺序不能颠倒，补齐是第一优先级。

### 平台引流运营流程

#### 第一步：发现需求

检查知乎/小红书的引流机会：

```
检查流程：
1. 知乎邀请回答 → 筛选可回答问题
2. 知乎私信 → 记录求资源用户
3. 知乎主动评论机会 → 搜索相关高热度文章
4. 小红书私信 → 记录求资源消息
```

**识别用户真实需求**，不能直接发链接敷衍！

**⚠️ 不检查抖音**

#### 第二步：资源匹配（运营核心责任！）

**⚠️ 重要：运营职能负责资源匹配，不是主理职能！**

**匹配流程**：
1. **先检查所有已有资源类型**
   - 职场Excel模板
   - PPT资源包
   - 前端学习路线图
   - 免费商用字体合集
   - 考证攻略汇总
   - 一建法规备考资料
   - 考研资料等

2. **根据问题关键词匹配**
   - "批量命名表格" → Excel模板（不是PPT！）
   - "PPT制作" → PPT资源
   - "编程学习" → 前端学习路线图
   - "考证备考" → 考证攻略/一建法规
   - "字体下载" → 免费商用字体

3. **不要只盯着一个资源类型**
   - ❌ 错误：所有问题都用PPT资源回答
   - ✅ 正确：根据问题关键词选择最匹配的资源

**参考**：`./网盘资源收集/` 目录下的所有文件夹

#### 第三步：协调资源

向主理职能发送需求：

```
**需求**：为「[问题标题]」准备引流资源

**资源类型**：[匹配后的资源类型]

**要求**：
1. 创建文件夹（/d/ 格式链接，不是 /f/）
2. 补充公开网盘资源链接（百度/夸克/阿里云盘）
3. 资源真实有价值，不能是占位符 `xxxxx`
4. 资源放到 ./网盘资源收集/ 对应目录下
```

#### 第四步：质量检测（必须执行！）

**拿到资源后必须检查**：

| 检查项 | 要求 | 禁止 |
|-------|------|------|
| 公开网盘链接 | 真实可用的链接 | ❌ 占位符 `xxxxx` |
| 分享链接格式 | `/d/` 文件夹链接 | ❌ `/f/` 文件链接 |
| 资源内容 | 匹配用户需求、有价值 | ❌ 货不对板、敷衍内容 |

**检测方法**：
```bash
# 检查是否有占位符
grep -l "xxxxx\|提取码：xxxx" ./网盘资源收集/**/*.md

# 检查链接格式
grep "url98.ctfile.com/f/" ./网盘资源收集/**/*.md
```

**如果检测不通过**：
1. 要求主理职能重新补充真实链接
2. 不合格的资源不能发布
3. **监督主理职能改正**，禁止偷懒敷衍

#### 第五步：发布回答

质量检测通过后发布：

1. **写干货内容**（不能直接甩链接）
2. 针对问题给出解决方案
3. 最后自然引出资源链接
4. 云手机自动发布或手动发布

**回答模板**：
```
[干货内容...]

📁 **下载地址**：[资源名称](分享链接)
访问密码：8069

直接下载即可使用！
```

#### 第六步：记录复盘与存档

**⚠️ 必须存档回答内容，不能只记录标题！**

更新三个记录文件：

| 文件 | 内容 |
|-----|------|
| `./网盘资源收集/已回答问题记录.md` | 新增已回答问题 |
| `./网盘资源收集/待处理引流机会.md` | 移除已处理项 |
| `./定时任务/执行记录.md` | 更新执行状态 |

**回答内容存档**：

每次发布后，将完整回答内容保存到：
```
./网盘资源收集/回答存档/YYYY-MM-DD_问题关键词.md
```

存档格式：
```markdown
# 问题：[问题标题]

**平台**：知乎/小红书
**发布时间**：YYYY-MM-DD HH:MM
**问题链接**：[链接]

**匹配资源**：[资源名称]
**分享链接**：[城通网盘链接]

---

## 回答内容

[完整的回答文本内容]
```

**关联检查（防止发错资源！）**：

发布前必须二次确认：
```
✅ 问题关键词 ↔ 资源类型 是否匹配？

示例：
- "批量命名表格" + Excel模板 ✅
- "批量命名表格" + PPT资源 ❌ 货不对板！
- "PPT制作" + PPT资源 ✅
- "建造师法规" + 一建法规资料 ✅
- "建造师法规" + 考研资料 ❌ 货不对板！
```

**⚠️ 禁止**：人家问PPT，你发建造师考研链接！

#### 4. 小红书私信检查

### 发布模式（重要！）

**支持两种发布模式，可随时切换：**

| 模式 | 说明 | 当前状态 |
|------|------|---------|
| **自动发布** | 云手机App自动发布到各平台，发布后获取分享链接 | 保留备用 |
| **手动发布** | 将各平台文案内容推送给用户，由用户自己手动发布（与CSDN相同方式） | ✅ **当前启用** |

**切换方式**：用户随时告知切换到自动/手动模式，立即生效

**手动发布模式规则**：
- 所有平台（知乎/微博/小红书/CSDN）统一将文案内容推送给用户
- 推送格式：每个平台单独一段，标注平台名，内容用纯文本格式包裹方便复制
- 知乎：推送纯文本版文案（`知乎_{资源名}_纯文本.md`内容）
- 微博：直接推送微博文案内容
- 小红书：推送标题和正文（标题和正文分开标注）
- CSDN：推送Markdown版文案内容
- **所有平台文案中的链接格式按平台适配**：
  - 知乎纯文本：用描述性链接 `[点击下载xxx](链接?p=8069)`
  - CSDN（Markdown）：用描述性链接 `[点击下载xxx](链接?p=8069)`
  - 微博：直接放URL `https://url98.ctfile.com/d/xxx?p=8069`（微博会自动识别，禁止Markdown语法）
  - 小红书：不放链接不放密码
  - 所有链接下方必须跟"访问密码：8069"（小红书除外）
- **所有平台文案不放标签/关键词行**：容易被识别为AI。CSDN标签放正文最末尾（下载链接之后），格式 `标签：xxx、xxx、xxx`（用顿号分隔，不用#号）
- **推送文案时需附带建议标签**：在文章内容之外，单独列出该文章建议使用的标签（如"CSDN标签：公务员考试、行测、申论..."），方便用户复制粘贴到平台的标签栏
- 用户发布后自行获取分享链接，告知我后更新发布记录
- 依然遵守流程顺序：先补未来3天 → 再发布当天 → 最后更新记录

### 云手机发布规范（自动发布模式时使用）

**知乎发布规范（重要！）**：
- 必须发布**文章**（写文章/专栏文章），不是圈子/想法！
- 必须使用知乎专属纯文本版文案（`知乎_{资源名}_纯文本.md`），不用Markdown格式
- 编辑完成后直接点击发布按钮，不需要用户手动确认

**微博发布规范**：
- 编辑完成后直接点击发布按钮，不需要用户手动确认

**发布后必须验证**：云手机显示"已发布"不等于真的发布成功！必须验证：
- 微博：进入"我"的页面，确认微博数量不是0
- 知乎：进入"我的创作"，确认文章存在
- 小红书：进入"我"的页面，确认笔记存在

**获取分享链接方式（发布后必须执行！）**：
- 知乎：文章详情页 → 点击分享按钮 → 菜单第一排第二个"复制链接"（蓝色链接图标）
- 微博：微博详情页 → 点击"..." → 菜单第三排向右滑动找"复制链接"（回形针图标，在"设置分享范围"右边，需要多滑几次！）
- 小红书：笔记详情页 → 右上角分享图标 → 复制链接
- CSDN：用户手动发布后自行获取

**分享链接必须做的事**：
1. 获取后立即发给用户确认（让用户验证链接能否正常打开）
2. 将分享链接记录到发布记录 `./网盘引流项目/已发布文章/发布记录.md` 中对应条目
3. 如果获取失败（如小红书账号限制），需告知用户并标注待获取

**云手机常见问题**：
- 任务卡住超10分钟 → 取消重试
- 多平台发布要排队，不能同时占用云手机
- 任务描述中直接写明发布内容，避免再去读文件导致卡死
- 小红书账号安全限制 → 需用户手动解除

**本地资源更新后必须同步城通网盘**：
- 本地资源文件修改后（如补充免费链接、修复内容），**必须重新上传到城通网盘覆盖旧版本！不能只改本地不管网盘！**
- 上传同名文件会创建新版本，旧版本仍保留
- 流程：本地修改 → 重新上传城通网盘 → 确认网盘版本已更新

### 找新资源流程（重要！）

**第一阶段：收集资源（必须先完成！）**
1. **读取资源清单** → 查看 `./网盘引流项目/资源清单.md` 获取所有可用资源类型
2. **读取已发布记录** → 检查 `./网盘引流项目/已发布文章/发布记录.md` 获取已发布资源类型
3. **读取历史资源目录** → 检查 `./网盘资源收集/` 下所有子目录，确认已有资源
4. **去重检查（强制！）** → 新资源不能与历史已有资源重复（包括已发布和已收集未发布的）
5. **选择新资源方向** → 从清单中选择未收集或热度高的类型
6. **创建资源文件** → 生成有价值的资源内容（拒绝空白模板），存放到 `./网盘资源收集/{资源名}/` 目录下（⚠️ 不是 `./网盘引流项目/资源文件/`！）
7. **补充免费网盘链接（强制！）** → 整合免费网盘下载链接（肖四/肖八/腿姐/徐涛等），绝不能让用户自行购买！⚠️ **严禁编造假链接！** 所有网盘链接必须通过搜索互联网获取真实可用的链接，不能自己捏造。编造的链接用户打不开，直接损害信誉！
8. **自检（强制！）** → 检查内容质量、文件大小、格式正确，**自检通过才能上传**
9. **上传到网盘** → 创建规范命名的文件夹 → 上传文件 → 获取分享链接

**第二阶段：编写文章（基于资源实际内容！）**
10. **阅读资源内容** → 通读已创建的资源文件，理解核心内容和亮点
11. **生成初始文案** → 根据资源实际内容为各平台生成适配文案，存入待发布目录
12. **生成知乎专属版** → ⚠️ **知乎文章由主技能直接编写，不交给写作技能！** 直接写纯文本版本，命名格式：`知乎_{资源名}_纯文本.md`，放在同级目录。知乎文章必须严格遵守知乎格式规范（见下方"知乎格式要求"）
13. **非知乎文案润色** → 调用文案写作技能润色除知乎外的其他平台文章（微博/小红书/CSDN）。知乎文章由主技能直接写好，不需要额外润色
14. **生成对比报告** → 以对比形式展示润色前后改动（原文 vs 润色后），便于用户直观查看（仅限写作技能润色的文章）
15. **检查监督** → 检查链接格式、内容质量、去AI味效果，确认无误后再覆盖原文件

**第三阶段：记录归档**
16. **更新发布记录（强制！）** → 在 `./网盘引流项目/已发布文章/发布记录.md` 中预记录
17. **更新清单状态** → 在资源清单中更新该资源的收集状态

### 资源去重规则
- **检查范围**：历史所有已有资源（不仅限于最近7天！）
- **去重依据**：资源类型 + 资源内容（如职场模板、考研资料、PPT模板等）
- **去重来源**：
  1. 已发布文章记录 `./网盘引流项目/已发布文章/发布记录.md`
  2. 历史资源目录 `./网盘资源收集/` 下所有子目录
- **示例**：
  - 已收集：考研资料/、前端学习路线/ → 不能再收集同类资源
  - 应选择：未收集过的类型，如公务员备考、教师资格等

### 资源质量标准（重要！）
**核心原则**：资源必须有价值+有需求+有免费链接，拒绝简陋空白模板！

**有价值资源定义**：
- ✅ 信息型：整理后的干货信息（如考研规划表、书单推荐、避坑指南）
- ✅ 稀缺型：市面上不易找到的资料（如真题汇总、独家整理）
- ✅ 实用型：能直接使用的内容（如完整方案、可直接套用的模板）

- [ ] **小红书无链接无密码**：小红书文章是否没有链接和密码提醒？小红书只靠评论区引流
- [ ] **知乎格式**：是否使用纯文本格式？禁止Markdown格式！
- [ ] **链接格式**：是否使用描述性链接 `[点击下载xxx](链接?p=8069)`？
- [ ] **链接正确**：链接是否有效？密码参数是否完整？
- [ ] **小红书规范**：是否用评论区引流而非直接放链接？
- [ ] **CSDN扩展名**：是否去掉了错误扩展名（如.txt）？
- [ ] **去AI味**：是否有"XXX是提升XX的重要途径"等套话？
- [ ] **真人感**：是否有第一人称、真实案例？
- [ ] **避坑指南**：是否有实用避坑内容？

### 常见润色错误
| 错误类型 | 表现 | 正确做法 |
|---------|-----|---------|
| 文案用非城通链接 | 夸克/百度链接放在文案中 | 文案只能用城通网盘链接，其他网盘放资源文档 |
| 城通链接缺密码 | 链接末尾没有 `?p=8069` | 必须带上密码参数，否则用户无法下载 |
| 链接缺密码提醒 | 链接下方没有"访问密码：8069" | 链接和密码提醒是整体，必须成对出现 |
| 小红书泄露链接/密码 | 放了链接或密码 | 小红书只靠评论区引流，不放链接也不放密码 |
| 小红书正文填到标题栏 | 标题超64字被截断 | 标题栏只填短标题（20字以内），正文填到正文输入框 |
| 知乎Markdown | #标题、**加粗**直接显示 | 用【】括起来，纯文本格式 |
| 缺少免费链接 | 让用户自行购买资源 | 补充百度网盘/夸克等免费下载链接 |
| 裸链接 | `https://...` 直接显示 | `[点击下载xxx](链接)` |
| 错误扩展名 | `[xxx.txt](链接)` | 去掉扩展名 |
| AI套话 | "XX是提升XX的重要途径" | 用真实故事引入 |
| 缺少避坑 | 只有正面描述 | 增加避坑指南 |
| CSDN未推送内容 | 用户需要自己去找文件 | 发布时直接把CSDN文章内容发给用户 |
| 未发分享链接 | 用户无法确认发布是否成功 | 发布后获取各平台文章分享链接，发给用户确认 |

### 发布前最终确认
文章润色完成后，发布前必须确认：
1. **免费网盘链接完整**：资源包含免费下载链接，用户无需自行购买
2. **知乎格式正确**：使用纯文本格式，禁止Markdown
3. **链接格式正确且可访问**
4. **链接密码完整**：所有城通网盘链接末尾必须包含 `?p=8069`
5. 内容无AI味、有真人感
6. 小红书不放链接
7. 资源文件已上传到网盘
8. **CSDN文章内容已推送给用户**：方便用户直接复制粘贴发布
9. **各平台分享链接已收集并发送给用户**：发布后获取文章分享链接，发给用户确认发布成功

## 协作机制（重要！）

### 与网盘运营项目组协作
本项目支持与其他项目组协作，运营组可通过以下方式反馈需求：

**协作场景**：
- 运营中发现用户高频需求 → 反馈补充相应资源
- 用户反馈某类资源稀缺 → 优先准备该类内容
- 某篇文章数据异常（高互动/低下载）→ 调整策略

**协作流程**：
1. 运营组发起需求 → 说明资源类型、用户痛点、预期效果
2. 本技能接收需求 → 评估可行性、排期
3. 执行资源准备 → 按正常流程：收集→上传→文案→润色
4. 反馈执行结果 → 告知运营组资源已就绪

**协作注意事项**：
- 协作需求优先级高于日常计划
- 需求要具体：资源类型、目标用户、预期效果
- 执行完成后主动通知运营组

## 历史教训（必读！）

### 教训1：敷衍资源 = 自毁信誉
**错误**：上传362字节的空白txt作为"真题"
**教训**：资源必须有价值（信息型/稀缺型/实用型），用户不是傻子
**标准**：所有资源必须经过质量检查，拒绝空白模板

### 教训2：链接格式错误
**错误**：用文件链接 `/f/` 而非文件夹链接 `/d/`
**教训**：必须先建文件夹→上传文件到文件夹→分享文件夹链接
**标准**：链接格式 `[点击下载{名称}](https://url98.ctfile.com/d/...)`

### 教训3：重复发布同类资源
**错误**：职场模板连续发布3天
**教训**：每次发布前检查最近7天已发布内容
**标准**：去重规则见上方"资源去重规则"章节

### 教训4：API认证方式错误
**错误**：用 `Authorization: Bearer` header认证
**正确**：在请求体中传 `session` 参数
**详情**：见 `references/ctfile_api.md`

### 教训5：不调用已有技能
**错误**：绕过技能文档自己摸索API
**教训**：任务涉及某领域 → 先skill_load对应技能 → 看文档 → 再动手

### 教训6：去重检查形同虚设
**错误**：2026-04-27发考研资料，04-28又发考研政治真题，连续两天同类资源
**原因**：发布记录.md 文件不存在，去重检查无依据
**教训**：每次发布后必须更新发布记录，否则去重规则无法执行
**标准**：发布完成后强制更新 `./网盘引流项目/已发布文章/发布记录.md`
