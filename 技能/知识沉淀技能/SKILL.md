# 知识沉淀技能

## 技能说明

从**聊天记录文件和项目文档**中识别有价值的知识，整理成结构化的 md 文件存入 `./知识沉淀/` 目录。

> ⚠️ 2026-04-30 重要更新：数据源从"扫描session历史"改为"扫描文件"。
> 原因：sessions_history 无法拉取太久之前的对话，事后复盘不可靠。
> 新方案依赖主对话中实时写入的 `./聊天记录/` 和 `./项目文档/`，数据不会丢失。

---

## 技能信息
- 加载模式：progressive
- 触发关键词：[知识, 沉淀, 记忆]
- 摘要：知识自动沉淀与整理

## 触发方式

1. **自动触发（优先）**：对话过程中检测到高价值知识时，**自动启动 sub-agent 沉淀**，不等用户提醒
2. **手动触发**：用户说"沉淀一下"、"整理一下知识"、"这个值得记录"
3. **定时扫描**：每2天执行一次补充扫描，查漏补缺

### ⚠️ 自动触发规则（2026-05-07 新增）

**核心原则：对话中出现的知识，主动沉淀，不要等用户说"记录一下"**

**自动触发信号**（出现任一即触发）：

| 信号 | 说明 | 动作 |
|------|------|------|
| 技术概念解释 | 解释清楚某个概念/原理 | 自动沉淀 |
| 多方案对比 | 列出2个以上方案并给出选型 | 自动沉淀 |
| 踩坑教训 | 失败案例+原因+解决方案 | 自动沉淀（双写SKILL.md+文档） |
| 方法论/流程 | 可复用的最佳实践 | 自动沉淀 |
| 系统机制理解 | 对平台/框架机制的深入理解 | 自动沉淀 |
| 决策记录 | 选型理由、架构决策 | 自动沉淀 |

**自动触发执行方式**：
1. 主对话中检测到高价值知识 → 简短告知主人"这段值得沉淀" → 启动 sub-agent 执行沉淀
2. sub-agent 加载本技能（`./技能/知识沉淀技能/`），按完整流程执行
3. 沉淀完成后告知主人文件位置

**不触发自动沉淀的**：
- 简单问答、纯操作步骤、临时信息、流水账
- 已在 `./知识沉淀/.已沉淀清单.json` 中去重命中的内容

## 判断标准

### ⚠️ 核心原则（最高优先级）

**不能只依赖搜索，不能用户说什么才去沉淀什么！**

好的内容就在对话过程中我给用户的回答上，必须主动深入挖掘和总结。

### 扫描深度要求

1. **不能只看 summary**：很多技术细节藏在对话过程中，而不是 summary 里
2. **必须阅读完整对话历史**：使用 sessions_history 获取详细内容
3. **主动识别高价值回答**：而不是等用户指出

### 高价值信号识别

| 信号类型 | 说明 | 优先级 |
|----------|------|--------|
| **表格回答** | 使用表格整理对比、方案、优缺点 | ⭐⭐⭐ 必沉淀 |
| **多方案对比** | 列出多个方案并给出选型建议 | ⭐⭐⭐ 必沉淀 |
| **技术演进路径** | 记录尝试多种方案最终找到最优解 | ⭐⭐⭐ 必沉淀 |
| **踩坑教训** | "之前XX里已经记录过，但执行时没看" | ⭐⭐⭐ 必沉淀 |
| **概念解释** | 解释清楚某个技术概念或原理 | ⭐⭐ 建议沉淀 |
| **方法论** | 可复用的流程、经验、最佳实践 | ⭐⭐ 建议沉淀 |
| **系统理解** | 对平台/系统机制的深入理解 | ⭐⭐ 建议沉淀 |
| **问题解决** | 有价值的故障排查和解决方案 | ⭐⭐ 建议沉淀 |

### 值得沉淀的内容类型

| 类型 | 特征 | 示例 |
|------|------|------|
| **概念解释** | 解释清楚某个技术概念或原理 | "xdotool和Playwright的区别" |
| **工具对比** | 多个工具/方案的对比分析 | "一键分发平台对比" |
| **方法论** | 可复用的流程、经验、最佳实践 | "Agent架构设计思路" |
| **系统理解** | 对平台/系统机制的深入理解 | "tasks.json和calendar的区别" |
| **问题解决** | 有价值的故障排查和解决方案 | "Chrome启动失败的处理方法" |

**不值得沉淀的：**
- 简单的问答（"帮我查个东西"）
- 纯操作步骤（"点这里然后点那里"）
- 临时性的信息（"今天天气怎么样"）
- 流水账式的对话

## ⚠️ 过滤标准：流程性内容 vs 有价值知识

### 必须过滤的流程性内容

**定义**：由技能或定时任务自动生成的、缺乏复用价值的表格或汇总内容。

| 过滤类型 | 判断依据 | 示例 |
|----------|----------|------|
| **测试报告** | 技能测试任务的元数据输出 | `测试报告-2026-04-27.md` |
| **运行日志** | 技能执行过程中的状态记录 | "本次扫描了X个Session，耗时Y秒" |
| **任务汇总表** | 定时任务的状态汇总 | "今日任务执行清单" |
| **进度报告** | 项目推进的状态汇报 | "本周完成了XX，下周计划YY" |

### 必须保留的有价值知识

**定义**：有复用价值的技术规范、踩坑经验、解决方案，即使来源是技能或定时任务。

| 保留类型 | 判断依据 | 示例 |
|----------|----------|------|
| **技术规范** | 有复用价值的规格、标准 | 微信表情包素材尺寸规范 |
| **踩坑经验** | 失败案例和解决方案 | 云手机App核对铁律 |
| **规则说明** | 平台机制、时间规则 | 掘金福利兑换开放时间规则 |
| **技术演进** | 尝试多种方案的最终结论 | 表情包透明背景处理最终方案 |

### 核心判断原则

```
过滤标准 ≠ "是否由技能生成"
过滤标准 = "是否有复用价值"
```

**正确示例**：
- ✅ "掘金福利兑换物品开放规则" → 虽然和定时任务相关，但规则本身有复用价值，保留
- ✅ "云手机任务执行规范" → 踩坑经验，保留
- ❌ "知识沉淀技能测试报告" → 纯元数据，删除

## 执行流程

### 手动触发模式（完整版）

```
1. 扫描 ./聊天记录/ 目录
   - 读取所有 .md 文件
   - 按 ./知识沉淀技能/references/价值判断规则.md 评估每段内容的价值

2. 扫描 ./项目文档/ 目录
   - 读取所有项目文档
   - 重点关注「进行中」项目中的经验教训和决策

3. 使用 memory_search 补充搜索
   memory_search(query="概念解释 工具对比 方法论 原理 架构", max_results=50)

4. 合并结果，按时间排序，去重

5. 检查是否与 ./知识沉淀/.已沉淀清单.json 中的已有内容重复

6. 不重复的内容整理成 md 文件，存入 ./知识沉淀/ 目录

7. 更新 .已沉淀清单.json，追加新记录
```

## 去重机制

在 `./知识沉淀/.已沉淀清单.json` 中记录已处理的对话特征：

```json
{
  "records": [
    {
      "id": "20260425-001",
      "date": "2026-04-25",
      "topic": "xdotool vs Playwright vs RPA 区别",
      "keywords": ["xdotool", "playwright", "rpa", "自动化"],
      "source_session": "主对话",
      "file": "xdotool-vs-Playwright-vs-RPA区别.md"
    }
  ],
  "last_scan_date": "2026-04-27",
  "scanned_sessions": ["主对话", "cron-xxx", "sub-xxx"]  // 记录已扫描的session
}
```

判断是否重复：
- 关键词重合度 > 70% → 视为重复，跳过
- 同一话题的补充信息 → 追加到现有文件，不新建

## 输出格式

沉淀文件使用**分类签名模板**，根据内容主题自动选择对应分类的标题格式和签名。

> 模板文件：`./技能/知识沉淀技能/references/分类签名模板.md`

### 使用步骤

1. **判断分类**：根据内容主题，对照模板中的分类关键词判断属于哪一类（IT技术/中医养生/金融/音乐音频/动漫游戏建模绘画/其他）
2. **提取标签**：按分类规则提取单个核心关键词作为 tags
3. **选择模板**：使用对应分类的签名模板生成文件
4. **填充内容**：替换模板中的 `{{title}}`、`{{date}}`、`{{tag}}`、`{{content}}`

### 通用格式说明

```markdown
---
title: {文章标题}
date: {YYYY-MM-DD HH:mm:ss}
tags: {单个核心关键词，无中括号}
---

{正文内容，直接开始，不加"用户提问"等标题}

{正文与签名之间空三行}

{对应分类的版权签名}
```

### 各分类签名

| 分类 | 签名 |
|------|------|
| IT技术类 | 乱码三千（含图片链接） |
| 中医养生类 | 万物之道 |
| 金融类 | 投资与财富 |
| 音乐音频类 | 有瓣音频 |
| 动漫游戏建模绘画类 | ACG作坊 |
| 其他 | 仅版权声明，无链接签名 |

详细签名内容见 `references/分类签名模板.md`
