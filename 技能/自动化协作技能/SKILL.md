# 自动化协作引擎技能 v4 (AutoForge Collab)

> 一句话：拉代码→扫描工单→切角色执行→清空后PM蓝图审查→未达标继续提工单→达标才终止
> 版本：v4.0（三模式+聚焦过滤器：匀速+加速+并行，均可叠加聚焦）

## 技能说明

本技能实现"无人值守"的自动化协作循环，核心能力：
1. **三模式**：匀速（日常维护）+ 加速（单项目冲刺）+ 并行（多项目同时推进）
2. **聚焦过滤器**：可叠加在任何模式上，只推指定项目，其他项目冻结
3. **蓝图驱动**：工单清空不是终点，PM审查蓝图目标是否实现才是终点
4. **省Token**：机械操作脚本化，子agent task模板化，工单传路径不传内容
5. **Merge Only**：拉取使用merge方式，与push.sh一致，防止多平台推拉丢失内容
6. **项目分组并行**：独立项目同时spawn不同角色，同角色排队，集中推送

> ⚠️ 本引擎是体系级基础设施，不属于任何特定项目。文档归属 `技能/自动化协作技能/`。

**触发方式**：Calendar定时任务（主）+ Heartbeat轻量检查（辅）

- 辅助脚本已就绪：`scripts/auto-collab.sh`

## 模式说明

### 匀速模式（默认）

- 每4小时触发一次Calendar
- 每次最多3轮循环
- 工单清空→PM检查停滞项目→产出报告→终止
- 适合：日常维护、主人在线

### 加速模式

- 触发后持续循环，不设轮次上限
- 工单清空→PM蓝图审查→有差距就提工单→继续执行→再审查→直到达标
- 终止条件：PM确认所有活跃项目达到蓝图预期 且 交接台无待领工单
- 安全约束：单轮执行≤30min，总时长≤2小时，超时强制终止
- 自动降速：连续3轮无新工单+PM无新工单→降速到匀速
- 适合：主人说"全力推进"/"冲刺"/离线无人值守

### 并行模式

- 以项目为中心，独立项目组同时spawn不同角色并行处理
- **组间并行**：不同项目的工单同时执行（如OpenForge和表情包同时推进）
- **组内可并行**：同一项目内不同角色的工单可同时执行（如系统开发者+前端开发）
- **同角色串行**：同一角色被多个项目需要时排队处理
- **集中推送**：子agent只commit不push，主agent收集结果后统一push（避免git冲突）
- Push锁机制：`auto-collab.sh lock/unlock` 防止并发push
- 终止条件：同加速模式（蓝图达标才停）
- 适合：多项目同时推进/主人离线时的全速冲刺

### 聚焦过滤器（可叠加在任何模式上）

- 指定某个/某几个项目重点推进，其他项目工单冻结
- **白名单机制**：只处理白名单项目的工单 + 体系维护工单
- **冻结而非删除**：非白名单项目的⏳待领取工单标记为⏸️聚焦暂停，不处理但保留
- **自动解冻**：解除聚焦或切换模式时，所有⏸️工单恢复为⏳
- **体系维护不冻结**：BUG单和体系优化项始终处理（体系健康是所有项目基础）
- **PM只审聚焦项目**：蓝图审查只看白名单项目的进度
- 用法示例：
  - "聚焦网文" → 聚焦[网文创作] + 当前模式
  - "聚焦网文和OpenForge，并行模式" → 聚焦[网文创作,OpenForge] + 并行
  - "解除聚焦" → 取消聚焦，恢复所有工单
- 适合：某项目紧急需集中资源/主人只想推特定项目

### 模式切换

- Calendar事件summary含标记：`自动化协作引擎[加速]` / `自动化协作引擎[匀速]` / `自动化协作引擎[并行]` / `自动化协作引擎[聚焦:项目名+模式]`
- 主人手动：告诉小龙"切加速模式"/"切匀速模式"/"切并行模式"/"聚焦网文"/"解除聚焦"
- 自动降速：加速/并行模式3轮空转→降速到匀速
- 聚焦达标：聚焦项目全部达标→通知主人，等待指令（自动解冻或切换聚焦）

## 完整流程

### Step 0: 初始化

```bash
# 拉取最新仓库（merge方式，禁止rebase）
cd open-knowledge-system && bash scripts/auto-collab.sh pull

# 如果pull失败（冲突），跳过本轮，通知主人

# 弹性调度（可选，避整点高峰）
JITTER=$(bash scripts/auto-collab.sh jitter | grep SLEEP_SECONDS | cut -d= -f2)
echo "⏱️ 本轮延迟: ${JITTER}秒后开始"
# 主agent按JITTER秒数延迟后再spawn子agent
```

读取模式：
- 检查Calendar事件summary判断当前模式
- 默认匀速模式
- 检查聚焦白名单（从事件summary或主人指令提取项目名）
- 如有聚焦白名单：非白名单项目的⏳工单标记为⏸️聚焦暂停

扫描工单：
```bash
cd open-knowledge-system && bash scripts/auto-collab.sh scan
```
- 输出：`WO-029|P0|系统开发者|OpenForge Sprint 1 Rust核心开发`
- 或：`NO_WORKORDERS`
- 聚焦模式下：白名单外工单标注⏸️不处理，体系维护工单仍处理

### Step 1: 处理工单（有待领工单时循环）

**匀速/加速模式** → 串行处理，按P0→P1→P2排序

**并行模式** → 项目分组并行处理，见下方"并行模式流程"

**聚焦模式** → 只处理白名单项目工单+体系维护工单

---

#### 匀速/加速模式：串行处理

对每个待领工单，按P0→P1→P2排序：

**1.1 合并同角色工单**

同一角色有多个工单→合并为一次spawn，task中列出所有工单路径。

**1.2 更新工单状态**
```bash
bash scripts/auto-collab.sh status WO-029 "🔧处理中"
```

**1.3 Spawn子agent（使用模板，省Token）**

```
你是{角色名}，请处理以下工单。

## 执行步骤
1. cd open-knowledge-system && bash scripts/act.sh "处理工单" . "角色/{角色名}"
2. 按五步门执行
3. 工单路径：交接台/工单/{WO-文件名}.md（自行读取完整内容）
4. 文件安全铁律：删除→回收站，覆盖=犯罪，先读后动
5. 重试熔断：失败2次→返回失败信息，不第3次自作主张
6. image_generate: count=1逐张生成
7. 完成后必须反哺：更新INDEX/knowledge/hot-rules
8. 汇报：完成/失败+原因
```

**1.4 检查执行结果（含轻量状态同步）**

等待子agent返回时，如果超过5分钟未响应：
1. 通过 sessions_status 检查子session状态（running/idle/error）
2. 如果是 running → 继续等待（可能还在执行）
3. 如果是 idle/error → 查 sessions_history 最后几条消息，判断卡点
4. 卡点可恢复 → sessions_send 继续推进
5. 卡点不可恢复 → 更新工单状态为❌失败，提bug单

正常完成的子agent：
- ✅ 成功 → `bash scripts/auto-collab.sh status WO-029 "✅已完成"`
- ❌ 失败 → `bash scripts/auto-collab.sh bug "{角色}" "{原因}"`
- ⚠️ 部分完成 → 更新工单备注卡点

**1.5 推送中间变更**
```bash
bash scripts/auto-collab.sh push "auto-collab: 处理{WO编号}完成"
```

**1.6 重新扫描**
```bash
bash scripts/auto-collab.sh scan
```
- 有新待领工单 → 回到Step 1
- 无待领工单 → Step 2

**循环控制：**
- 匀速模式：最多3轮
- 加速模式：无上限，但每轮结束检查是否空转

### Step 2: 产品经理蓝图审查（工单清空后）

**匀速/加速/并行模式** → PM审查所有活跃项目

**聚焦模式** → PM只审查白名单项目的蓝图进度

---

#### 聚焦模式下的PM审查

- 只读取白名单项目的INDEX.md和蓝图目标
- 白名单项目全达标 → 通知主人"聚焦项目已达标，是否解除聚焦？"
- 主人确认解除 → 恢复所有⏸️工单为⏳ → 继续常规流程
- 主人不解除 → 保持聚焦，等待下次触发

---

#### 并行模式：项目分组并行处理

**1.1 项目分组**
```bash
bash scripts/auto-collab.sh group
```
输出：
```
GROUP:OpenForge
  WO-029|P0|系统开发者|WO-029-OpenForge-Sprint1-Rust核心开发.md
  WO-030|P1|前端开发|WO-030-OpenForge-Sprint1-Web-Studio前端初始化.md
GROUP:体系维护
  BUG-028|P1|AI调教师|BUG-028-SECRET.md已提交到Git仓库.md
```

**1.2 生成执行计划**
```bash
bash scripts/auto-collab.sh plan
```
- 不同项目的不同角色 → 可同时spawn
- 同一角色跨项目 → 串行排队（先P0后P1）
- 同一项目内不同角色 → 可并行

**1.3 Spawn并行波次**

每个波次中，所有可并行的工单同时spawn：

```
波次1（3个agent同时启动）：
  agent-A [OpenForge + 系统开发者] → WO-029
  agent-B [OpenForge + 前端开发]   → WO-030  ← 与agent-A同项目不同角色，可并行
  agent-C [体系维护 + AI调教师]    → BUG-028  ← 独立项目，可并行

波次2（agent-A完成后）：
  agent-D [OpenDAW + 系统开发者]   → WO-002  ← 系统开发者排队等agent-A完成
```

**1.4 并行子agent task模板**
```
你是{角色}，处理{项目名}项目的工单。

## 执行步骤
1. cd open-knowledge-system && bash scripts/act.sh "处理工单" . "角色/{角色}"
2. 读取工单：交接台/工单/{文件名}.md
3. 按五步门执行
4. ⚠️ 并行模式约束：
   - 只修改 {项目} 相关文件，不碰其他项目
   - 完成后 git commit，不要 git push（主agent统一推送）
   - 如需修改共享文件（交接台/），只改工单状态，不改README
5. 铁律：删除→回收站|覆盖=犯罪|先读后动|失败2次停
6. 反哺：更新项目INDEX/knowledge/hot-rules
7. 汇报：完成/失败+原因+修改的文件列表
```

**1.5 等待所有并行子agent完成**

**1.6 收集结果 + 更新工单状态 + 更新交接台README**

主agent统一处理：
- 各子agent回报结果 → 更新工单状态
- 交接台README更新（只有主agent改，避免冲突）

**1.7 统一推送**
```bash
bash scripts/auto-collab.sh push "auto-collab[并行]: 波次{N}完成"
```

**1.8 重新扫描+分组**
- 有新工单 → 新的执行计划 → 回到1.3
- 无新工单 → Step 2

**并行模式安全约束：**
- 子agent绝不push，只commit
- 主agent统一push
- 交接台README只有主agent修改
- 共享知识/目录：子agent只读不改（如需改→提工单给AI调教师）
- Push锁：`auto-collab.sh lock/unlock` 兜底

---

**2.1 预处理**
```bash
bash scripts/auto-collab.sh pm-check
```
输出各项目进度概览。

**2.2 PM审查（spawn子agent）**

```
你是产品经理角色，请执行蓝图审查。

## 执行步骤
1. cd open-knowledge-system && bash scripts/act.sh "蓝图审查" . "角色/产品经理"
2. 读取 共享知识/蓝图.md，提取每个项目的终极目标
3. 读取项目进度概览（已在上方输出）
4. 对每个活跃项目判断：
   - 项目INDEX进度 vs 蓝图目标 → 是否有差距？
   - 有差距 → 明确下一步需要什么，提工单给对应角色
   - 无差距 → 标记达标
5. 产出PM审查报告：
   - 哪些项目达标
   - 哪些项目有差距 + 差距是什么
   - 需要提哪些工单
6. 文件安全铁律：删除→回收站，覆盖=犯罪，先读后动
7. 汇报：审查结论+是否需要继续循环
```

**2.3 PM审查结果处理**

- PM提了新工单 → 回到Step 1（继续循环）
- PM确认全部达标且无工单 → Step 3

### Step 3: 终止判断

| 模式 | 条件 | 动作 |
|------|------|------|
| 匀速 | 任何 | 直接终止，等下次Calendar触发 |
| 加速/并行 | PM确认达标 + 无工单 | ✅ 终止，通知主人 |
| 加速/并行 | PM发现差距 | 回到Step 1 |
| 加速/并行 | 3轮空转 | 自动降速到匀速模式，终止本轮 |
| 聚焦+任意 | 聚焦项目达标 + 无工单 | 通知主人，等指令（解除聚焦/切换） |
| 聚焦+任意 | 主人说"解除聚焦" | 恢复所有⏸️工单，继续当前模式 |

**最终推送+报告：**
```bash
bash scripts/auto-collab.sh push "auto-collab: 协作循环完成"
```

产出报告到 `交接台/物料/`。

## 触发配置

### Calendar定时任务

| 配置 | 值 |
|------|---|
| 事件名 | 自动化协作引擎[模式] 或 自动化协作引擎[聚焦:项目名+模式] |
| 频率 | 匀速：每4小时；加速/并行：每2小时；聚焦：取决底层模式 |
| time_range | ±30min |
| 弹性调度 | time_range ±30min，spawn前调用 `auto-collab.sh jitter` 获取随机偏移 |

### Heartbeat轻量检查

```markdown
- 检查交接台工单：有⏳待领取→通知主人"有{N}个待领工单"
- 检查项目停滞：任何项目最近变更>5天→通知主人
```

| PM审查范围 | 全部项目 | 全部项目 | 全部项目 | 只审白名单项目 |

## 工单流转规则

| 完成角色 | 产生 | 流转给 |
|---------|------|--------|
| 系统开发者 | 代码完成 | 测试工程师 |
| 测试工程师(通过) | 需部署 | 本地运维/ECS运维 |
| 测试工程师(失败) | 需修复 | 系统开发者 |
| 运维 | 部署完成 | 测试工程师(回归) |
| 任何角色 | 体系问题 | AI调教师 |
| 产品经理 | 项目推动 | 对应开发角色 |
