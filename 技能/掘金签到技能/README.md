# 稀土掘金签到抽奖福利监控

> 自动执行稀土掘金的每日签到、幸运抽奖、福利兑换查询，并监控新增可兑换物品

## 功能特性

### 🎯 核心功能
| 功能 | 浏览器方案 | API方案 |
|------|-----------|---------|
| 每日签到 | ✅ 支持 | ✅ 支持 |
| 免费抽奖 | ✅ 支持 | ✅ 支持 |
| 福利查询 | ✅ 支持 | ❌ 不支持 |
| 自动兑换 | ✅ 支持 | ❌ 不支持 |
| 新增检测 | ✅ 支持 | ❌ 不支持 |

### 🔄 方案对比

| 特性 | 浏览器方案 | API方案 |
|------|-----------|---------|
| 稳定性 | 受网页改版影响 | 稳定，不受影响 |
| 风控风险 | 较高 | 较低 |
| 执行速度 | 较慢 | 快速 |
| 功能完整度 | 完整 | 仅签到+抽奖 |
| 配置复杂度 | 低（自动登录） | 中（需手动获取Cookie） |
| 推荐场景 | 日常使用 | 备用/风控严格时 |

### ⏰ 定时任务
- **执行时间**: 每天 07:35
- **重复规则**: 每日重复
- **时区**: 北京时间 (UTC+8)

## 安装与配置

### 1. 导入技能
将 `juejin-checkin` 目录复制到目标机器的 `skills/` 目录下。

### 2. 配置参数
复制配置文件模板并填入实际值：
```bash
cp config.env.template config.env
```

编辑 `config.env`:

#### 基础配置（必填）
```env
# 用户手机号（用于验证码登录）
PHONE_NUMBER=你的手机号

# 通知邮箱地址
EMAIL_ADDRESS=your_email@example.com
```

#### 方案选择（可选）
```env
# 签到方案：browser（默认）或 api
CHECKIN_METHOD=browser

# API方案需要配置Cookie
JUEJIN_COOKIE=csrf_session_id=xxx; sessionid=xxx
```

#### Cookie获取方法（API方案）
1. 浏览器打开 https://juejin.cn 并登录
2. 按 F12 打开开发者工具
3. 点击 **Application** → **Cookies** → **https://juejin.cn**
4. 找到并复制：
   - `csrf_session_id` 的值
   - `sessionid` 的值
5. 按格式填入：`JUEJIN_COOKIE=csrf_session_id=值; sessionid=值`

> ⚠️ Cookie有有效期（约1-2周），过期后需重新获取

### 3. 初始化历史记录目录
```bash
mkdir -p ~/.juejin-checkin/history
```

## 注意事项

1. **首次使用需登录**: 首次执行时需要手动输入验证码完成登录
2. **保持浏览器环境**: 定时任务依赖浏览器环境，确保云电脑正常运行
3. **历史记录保留**: 建议保留至少30天的历史记录以便对比
4. **邮件送达**: 检查垃圾邮件文件夹确保能收到通知

## 故障排除

| 问题 | 解决方案 |
|------|----------|
| 登录失败 | 检查手机号配置，尝试重新获取验证码 |
| 页面加载超时 | 增加等待时间或检查网络连接 |
| 抽奖按钮不可点击 | 可能今日抽奖次数已用完 |
| 邮件未收到 | 检查垃圾邮件文件夹，确认邮箱地址正确 |
