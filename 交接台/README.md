# 交接台 — 工单流转系统

## 使用说明

1. 创建工单：在 `工单/` 目录下创建 `WO-[编号]-[描述].md`
2. 提领工单：角色读取工单，更新状态为"🔧处理中"
3. 完成工单：更新状态为"✅已完成"，反哺知识体系
4. 失败工单：在 `BUG/` 目录下创建bug单

## 工单模板

```markdown
# WO-[编号]：[标题]

| 字段 | 内容 |
|------|------|
| 编号 | WO-[编号] |
| 提交者 | [提交者] |
| 接收角色 | [角色名] |
| 优先级 | P0/P1/P2 |
| 状态 | ⏳待领取 |

## 任务描述
[描述]

## 验收标准
- [标准1]
```

## 工单流转规则

| 完成角色 | 产生 | 流转给 |
|---------|------|--------|
| 开发 | 代码完成 | 测试 |
| 测试(通过) | 需部署 | 运维 |
| 测试(失败) | 需修复 | 开发 |
| 运维 | 部署完成 | 测试(回归) |
| 任何角色 | 体系问题 | 体系维护 |
