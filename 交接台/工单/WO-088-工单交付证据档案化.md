## 背景

OpenRath框架的核心贡献之一：**Session = 证据载体，而非聊天记录**。

一个任务跑完，理想的产物不该只停在"成功了"，而应是整份可回溯的证据卷宗：
```
原始需求 → 执行上下文 → 工具调用 → 产出物 → 验收结论
```

当前交接台工单交付=写文件/跑命令，没有结构化证据链记录。

---

## 任务目标

**改造六步门交付闭环**，让工单交付=证据档案。

### 具体要求

1. **修改六步门第⑤交付闭环**
   - 原：push.sh推送，闭环确认
   - 新：交付前必须生成证据档案，闭环确认时检查档案完整性

2. **设计证据档案模板**
   包含字段：
   - `原始需求`：工单/对话来源，链接
   - `执行过程`：关键节点、失败重试、分支决策（如有）
   - `产出物`：文件路径、变更摘要
   - `验收结论`：通过/有条件通过/不通过 + 理由

3. **对接交接台**
   - 新增物料类型：`证据档案物料`
   - 交付时在物料中附证据档案

4. **更新相关角色RULES**
   - AI调教师RULES（六步门执行门）
   - 系统开发者RULES（交付门）
   - 测试工程师RULES（验收门）

---

## 验收标准

1. 六步门第⑤文档中有证据档案要求
2. 证据档案模板存在于知识体系

4. 推送前执行1次证据档案demo验证
