# 掘金签到

> 自动执行稀土掘金的每日签到、幸运抽奖、福利兑换查询，并监控新增可兑换物品

## 执行步骤

### 混合方案（2026-04-28 更新）

| 功能 | 方案 | 原因 |
|------|------|------|
| 签到 | API | 稳定，只需Cookie，风控风险低 |
| 抽奖 | 浏览器 | API需要动态 `x-secsdk-csrf-token`，浏览器自动处理 |
| 福利兑换 | 浏览器 | 涉及表单填写和确认操作 |

### 签到流程（API方案）

1. 读取Cookie → 调用 `POST https://api.juejin.cn/growth_api/v1/check_in`
2. **签到前后必须记录矿石数**：签到前 `get_cur_point` → 签到 → 等待3-5秒 → 再查矿石数
3. 矿石增加（+66~200）→ 签到成功；矿石不变 → 签到失败，禁止报告成功

### 抽奖流程（浏览器方案）

1. 注入Cookie → 打开 `https://juejin.cn/user/center/lottery`
2. 点击"免费抽奖"按钮 → 获取结果
3. Cookie关键项：sessionid, uid_tt, passport_csrf_token, s_v_web_id, sid_guard

### 福利兑换流程

1. 进入福利兑换页面 → 查询所有可兑换物品
2. 对比历史记录，检测新增物品
3. 检查兑换目标是否满足（矿石足够 + 开放兑换）
4. 可兑换多样时，优先兑换多种（非高价单品）
5. 填写收货地址 → 完成兑换

## 注意事项

### 执行铁律

1. **签到和抽奖必须绑定执行**：签到成功后必须紧接着执行抽奖
2. 签到失败时，修复Cookie后也要补执行抽奖
3. Cookie失效时，自己用浏览器登录获取（手机号18630620063）
4. **矿石数验证**：只有矿石数确实增加才能报告"签到成功"

### Cookie相关

- `csrf_session_id` 已不存在，使用完整Cookie字符串即可
- Cookie有效期约1年，但可能提前失效
- 自动获取优先：agent-browser → 打开 https://juejin.cn/login → 手机号登录 → 抓取Cookie
- 手动获取备用：浏览器F12 → Application → Cookies → 复制完整Cookie

### 签到入口路径

- ✅ 直接访问：`https://juejin.cn/user/center/signin?avatar_menu`
- ✅ 抽奖页面：`https://juejin.cn/user/center/lottery`
- ❌ 错误路径：不要访问 /creator/task 或创作者中心

### 福利兑换规则

- 不同物品在不同工作日开放兑换（如空调毯=周三周四，筋膜枪=周五）
- 热门物品很快抢光，需要卡点（0点开放时立即兑换）
- 兑换前必须核对收货地址
