## Phase 1：核心备份（本地优先） ⏳ 待启动

**目标：本地文件 → 多后端备份，3-2-1规则可执行**

- [ ] 核心概念：BackupPolicy / FileInventory / ReplicaInfo
- [ ] 文件监听：检测文件变化（inotify/FSEvents）
- [ ] 增量备份：块级去重（类rsync算法）
- [ ] 存储后端：通过open-storage调用（本地/R2/WebDAV）
- [ ] CLI工具：`openvault backup/restore/status`
- [ ] 配置文件：策略定义 + 后端配置

**验收：** 配置一个备份策略，文件变化自动备份到3个位置，`openvault status`显示副本健康

---

## Phase 2：校验自愈 + 版本管理 📋 规划中

**目标：坏副本自动修复，多版本可回溯**

- [ ] 完整性校验：SHA256定时校验
- [ ] 自愈：检测到异常副本自动从健康副本修复
- [ ] 版本管理：多版本保留 + 增量链
- [ ] 快照：时间点快照（完整状态快照）
- [ ] 恢复：按版本/时间点恢复

**验收：** 手动损坏一个副本，自愈自动修复；恢复到任意历史版本

---

## Phase 3：OpenLink集成 + 远程管理 📋 规划中

**目标：通过OpenLink做运输，远程管理备份**

- [ ] OpenLink API集成：存储路由 + 传输路由
- [ ] API服务：远程查看/管理备份
- [ ] 多设备备份：ECS/云电脑/NAS统一管理
- [ ] 通知：备份异常/自愈/风险预警推送

**验收：** 通过API远程查看所有设备备份状态，跨设备恢复文件

---

## Phase 4：AI智能层 + 语义搜索 📋 规划中

**目标：AI管理备份 + 文件秒搜 + 知识整理 + 个性化**

- [ ] 文件索引扩展（ext-indexer）
  - 文本/代码/Markdown → 全文索引
  - 图片 → AI描述+标签
  - 音频 → 转文字+标签
  - PDF/Word → 文本提取
  - 向量化存储，语义搜索+关键词双路召回
- [ ] 搜索API
  - `openvault search "Rust选型讨论"` → 语义搜索返回相关文件
  - 支持跨项目搜索
- [ ] 知识整理智能体
  - 自动提取关键信息，按主题归类
  - 生成知识卡片存入知识沉淀目录
- [ ] 文件分类：AI自动识别重要度，推荐策略
- [ ] 语义级增量：文本/代码类文件语义差异备份
- [ ] AI恢复：自然语言意图 → 定位恢复
- [ ] 异常预测：健康趋势分析 + 风险预警
- [ ] 智能调度：根据网络状况/设备在线状态优化备份时间
- [ ] 个性化学习

**验收：** 自然语言搜索秒出结果；智能体自动整理知识；搜索结果个性化排序

## Phase 5：企业级 + 安全合规 + 物理智能体 📋 规划中

**目标：生产级可靠性 + 合规 + 人形机器人接入**

- [ ] 加密备份：端到端加密，服务端不可读
- [ ] 公私分治备份策略
  - 公开文件（visibility=public）：正常备份，可被安全扫描校验
  - 私密文件（visibility=private）：端到端加密备份，服务端零知识
- [ ] 人形机器人 / 物理智能体接入
  - 机器人作为OpenVault客户端：语音查询备份状态、触发恢复
  - 实时响应：机器人指令 <100ms 返回
  - 隐私隔离：机器人访问私密备份需用户授权+生物验证
- [ ] 审计日志：所有操作可追溯
- [ ] 合规策略：数据保留周期/地域合规（GDPR等）
- [ ] 多租户：团队/家庭共享备份空间
- [ ] Web管理面板

**验收：** 端到端加密备份+恢复；机器人语音查询备份状态；审计日志完整

## 设计铁律

1. **永不丢失** — 底线，任何决策优先保证数据安全
2. **自愈优先于告警** — 能自动修复的不通知人
3. **端到端加密** — 可选但默认开启
4. **最少依赖** — 核心备份不依赖OpenLink
5. **配置驱动** — 策略/后端/调度全部可配置
6. **渐进式** — 初期只做本地备份+云存储
7. **安全合规** — 内容扫描（复用OpenLink安全Hook）
