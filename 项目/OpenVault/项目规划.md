# OpenVault — 智能文件备份容灾系统

> 项目代号：OpenVault
> 创建时间：2026-05-09
> 状态：规划阶段
> 核心哲学：**狡兔三窟，AI守护，永不丢失**

---

## 一、项目定位

**OpenVault 不是网盘，不是同步工具，是智能时代的文件保险系统。**

- 网盘 = 存一份，靠服务商不倒
- 同步工具 = 多端一致，但不保证容灾
- OpenVault = 多地多副本 + 完整性校验 + 自愈 + AI智能管理

解决的核心痛点：**重要文件害怕丢失，需要多地备份容灾，靠人记不住也管不过来。**

---

## 二、与OpenLink的关系

**OpenVault是保险层，OpenLink是运输层。** 分层依赖，不合并。

```
OpenVault（保险层）          ← 复制策略/版本管理/完整性/恢复
       │ 调用
OpenLink（运输层）           ← 存储路由/传输路由/存储后端
       │ 依赖
open-storage（共享存储抽象）  ← StorageBackend trait/各后端实现
```

| 维度 | OpenLink | OpenVault |
|------|---------|-----------|
| 核心动作 | 路由（文件到哪去） | 复制（文件在多处都存在） |
| 问题域 | 怎么最快送到 | 怎么确保不丢 |
| 决策维度 | 网络拓扑、设备类型 | 副本数量、地理分布、版本历史 |
| 时间模型 | 实时/按需 | 周期性/增量 |

## 三、核心概念

### 3.1 备份策略（Backup Policy）

定义一个文件/目录需要几份副本、存在哪里、保留多久。

```rust
BackupPolicy {
    name:           String,
    replicas:       u32,              // 副本数（至少3）
    locations:      Vec<LocationSpec>,// 存储位置要求
    schedule:       Schedule,         // 备份频率
    retention:      Retention,        // 保留策略
    versioning:     Versioning,       // 版本管理
    priority:       Priority,         // 优先级（影响AI调度）
}

LocationSpec {
    backend_type:   BackendType,      // R2/OSS/本地/网盘/WebDAV/SFTP...
    region:         Option<String>,   // 地域要求（异地容灾）
    encryption:     bool,             // 是否加密
}

Schedule = 
    | Realtime          // 文件变化即时备份
    | Interval(Duration)// 定时备份
    | OnChange          // 检测到变化时备份

Retention =
    | KeepAll           // 永久保留
    | KeepVersions(u32) // 保留最近N个版本
    | KeepDuration(Duration) // 保留最近N天
    | Custom(Extension) // 自定义策略

Versioning =
    | Full              // 全量备份
    | Incremental       // 增量备份（块级去重）
    | Semantic          // 语义级增量（AI理解内容变化）
```

### 3.2 3-2-1黄金规则（默认策略）

- **3** 份副本
- **2** 种不同介质
- **1** 份异地

```toml
[policy.default]
replicas = 3
locations = [
    { type = "local", path = "/data/backup" },        # 本地SSD
    { type = "r2", bucket = "openvault-backup" },      # 云端异地
    { type = "webdav", url = "https://dav.example.com" } # 异地WebDAV
]
schedule = "on-change"
versioning = "incremental"
retention = { keep_versions = 10 }
```

### 3.3 文件清单（Inventory）

OpenVault维护一份完整的文件清单，记录每个文件存在哪里、什么版本、完整性状态：

```rust
FileInventory {
    file_id:        UUID,
    path:           String,            // 原始路径
    checksum:       String,            // SHA256
    size:           u64,
    replicas: Vec<ReplicaInfo>,        // 各副本信息
    last_verified:  Timestamp,         // 最后校验时间
    policy:         String,            // 所属策略
}

ReplicaInfo {
    location:       LocationSpec,      // 存在哪
    version:        u32,               // 版本号
    stored_at:      Timestamp,         // 存储时间
    verified_at:    Timestamp,         // 校验时间
    status:         ReplicaStatus,     // Healthy/Degraded/Corrupted/Missing
}
```

### 3.4 完整性校验与自愈

```
定时校验任务 → 计算checksum → 跟清单比对
    │
    ├─ 全部Healthy → 记录校验时间，下次
    ├─ Degraded(部分副本异常) → 从健康副本修复异常副本（自愈）
    ├─ Corrupted(全部异常) → 告警，人工介入
    └─ Missing(副本丢失) → 从其他副本重建
```

自愈是核心能力：**检测到损坏不需要人干预，自动从健康副本恢复。**

### 3.5 灾难恢复

```bash
# 一键恢复：按策略恢复所有文件
openvault restore --policy important

# 按意图恢复：AI理解自然语言
openvault restore "上周三的合同"

# 按时间点恢复：回到某个快照
openvault restore --snapshot 2026-05-01

# 按文件恢复：恢复特定文件
openvault restore /path/to/report.pdf
```

## 四、AI增强能力

### 4.1 智能文件识别

AI自动识别文件重要度，推荐备份策略：

```
代码仓库 → 高优先级，实时备份，5副本3地域
家庭照片 → 高优先级，定时备份，3副本
临时文件 → 低优先级，不备份
合同/证件 → 最高优先级，加密备份，5副本
系统配置 → 高优先级，变更时备份
```

### 4.2 语义级增量备份

传统增量备份是块级去重（rsync），OpenVault可以做语义级：

```
传统：检测到文件变了一个字节 → 传整个变化块
语义：理解文档修改了第3段 → 只传差异描述 + 变更内容
```

对文本/代码类文件效果显著，能大幅减少传输量。

### 4.3 AI恢复

```
"把上周改过的README找回来" 
  → AI理解：时间范围(上周) + 文件类型(README) + 操作类型(修改)
  → 定位版本 → 恢复到指定位置
```

### 4.4 异常预测

AI分析副本健康状况趋势，预测可能的风险：

```
R2存储的副本最近3次校验速度变慢 → 预警该后端可能有性能问题
移动硬盘上次连接是30天前 → 提醒接入备份
某类文件副本数低于策略要求 → 自动触发补备
```

## 七、共享基础设施

### open-storage 共享crate

从OpenLink中提取存储后端抽象，两个项目共用：

```
open-storage/
├── StorageBackend trait     # upload/download/presigned_url/delete
├── R2实现
├── OSS实现
├── WebDAV实现
├── SFTP实现
├── 本地实现
├── StorageRouter            # 存储路由
└── Presigned URL工具
```

### 共享模式

| 共享 | OpenLink用法 | OpenVault用法 |
|------|-------------|--------------|
| StorageBackend | 文件中转时选后端 | 备份时选目标后端 |
| StorageRouter | 按文件特征路由 | 按策略要求选位置 |
| Presigned URL | 直传下载 | 备份上传/恢复下载 |
| Extension Registry | 注册Action/Condition/Hook | 注册Policy/Verifier/Classifier |

## 八、MVP路线图

### Phase 1：核心备份（本地优先）

**目标：本地文件 → 多后端备份，3-2-1规则可执行**

- [ ] 核心概念：BackupPolicy / FileInventory / ReplicaInfo
- [ ] 文件监听：检测文件变化（inotify/FSEvents）
- [ ] 增量备份：块级去重（类rsync算法）
- [ ] 存储后端：通过open-storage调用（本地/R2/WebDAV）
- [ ] CLI工具：`openvault backup/restore/status`
- [ ] 配置文件：策略定义 + 后端配置

**验收：** 配置一个备份策略，文件变化自动备份到3个位置，`openvault status`显示副本健康

### Phase 2：校验自愈 + 版本管理

**目标：坏副本自动修复，多版本可回溯**

- [ ] 完整性校验：SHA256定时校验
- [ ] 自愈：检测到异常副本自动从健康副本修复
- [ ] 版本管理：多版本保留 + 增量链
- [ ] 快照：时间点快照（完整状态快照）
- [ ] 恢复：按版本/时间点恢复

**验收：** 手动损坏一个副本，自愈自动修复；恢复到任意历史版本

### Phase 3：OpenLink集成 + 远程管理

**目标：通过OpenLink做运输，远程管理备份**

- [ ] OpenLink API集成：存储路由 + 传输路由
- [ ] API服务：远程查看/管理备份
- [ ] 多设备备份：ECS/云电脑/NAS统一管理
- [ ] 通知：备份异常/自愈/风险预警推送

**验收：** 通过API远程查看所有设备备份状态，跨设备恢复文件

### Phase 4：AI智能层 + 语义搜索

**目标：AI管理备份 + 文件秒搜 + 知识整理 + 个性化**

- [ ] 文件索引扩展（ext-indexer）
  - 文本/代码/Markdown → 全文索引
  - 图片 → AI描述+标签
  - 音频 → 转文字+标签
  - PDF/Word → 文本提取
  - 向量化存储，语义搜索+关键词双路召回
- [ ] 搜索API
  - `openvault search "Rust选型讨论"` → 语义搜索返回相关文件和知识
  - `openvault search --tag daw --recent 7d` → 按标签+时间筛选
  - 支持跨项目搜索（OpenLink知识沉淀/OpenDAW文档/个人文件）
- [ ] 知识整理智能体
  - 自动提取关键信息，按主题归类
  - 生成知识卡片存入知识沉淀目录
  - 支持自然语言指令："整理最近一个月的DAW开发笔记"
- [ ] 文件分类：AI自动识别重要度，推荐策略
- [ ] 语义级增量：文本/代码类文件语义差异备份
- [ ] AI恢复：自然语言意图 → 定位恢复
- [ ] 异常预测：健康趋势分析 + 风险预警
- [ ] 智能调度：根据网络状况/设备在线状态优化备份时间
- [ ] 个性化学习
  - 基于搜索/访问历史学习偏好
  - 自动标注文件优先级
  - 定时推送相关文件摘要
  - 整理风格适配（用户偏好精简vs详尽）

**验收：** 自然语言搜索秒出结果；智能体自动整理知识到沉淀目录；搜索结果个性化排序

### Phase 5：企业级 + 安全合规 + 物理智能体

**目标：生产级可靠性 + 合规 + 人形机器人接入**

- [ ] 加密备份：端到端加密，服务端不可读
- [ ] 公私分治备份策略
  - 公开文件（visibility=public）：正常备份，可被安全扫描校验
  - 私密文件（visibility=private）：端到端加密备份，服务端零知识
  - 策略配置：不同visibility自动匹配不同加密级别和存储位置
- [ ] 人形机器人 / 物理智能体接入
  - 机器人作为OpenVault客户端：语音查询备份状态、触发恢复、确认副本健康
  - 实时响应：机器人指令 <100ms 返回
  - 隐私隔离：机器人访问私密备份需用户授权+生物验证
  - 物理上下文：机器人知道自己的位置，自动选择最近的副本恢复
  - 场景示例："帮我把昨天的会议录音恢复到客厅音箱" → 机器人定位文件 → 选最近副本 → 传输恢复
- [ ] 审计日志：所有操作可追溯
- [ ] 合规策略：数据保留周期/地域合规（GDPR等）
- [ ] 多租户：团队/家庭共享备份空间
- [ ] Web管理面板

## 十、设计铁律

1. **永不丢失** — 这是底线，任何设计决策优先保证数据安全
2. **自愈优先于告警** — 能自动修复的不通知人，只有修不了的才告警
3. **端到端加密** — 可选但默认开启，服务端不可读客户端加密数据
4. **最少依赖** — 核心备份能力不依赖OpenLink，OpenLink是增强不是必需
5. **配置驱动** — 策略/后端/调度全部可配置
6. **渐进式** — 初期只做本地备份+云存储，不急着接AI
7. **安全合规** — 内容扫描（复用OpenLink安全Hook），合规优先

## 十一、配置示例

```toml
[openvault]
data_dir = "/var/lib/openvault"

[policy.code]
paths = ["/root/code", "/root/projects"]
replicas = 3
schedule = "on-change"
versioning = "incremental"
retention = { keep_versions = 20 }
locations = [
    { type = "local", path = "/backup/code" },
    { type = "r2", bucket = "openvault-code" },
    { type = "webdav", url = "https://dav.example.com/code" }
]

[policy.documents]
paths = ["/root/documents", "/root/contracts"]
replicas = 5
schedule = "on-change"
versioning = "incremental"
retention = { keep_versions = 50 }
encryption = true
locations = [
    { type = "local", path = "/backup/docs" },
    { type = "r2", bucket = "openvault-docs", encryption = true },
    { type = "oss", bucket = "openvault-docs-cn", region = "cn-hangzhou" },
    { type = "webdav", url = "https://dav.example.com/docs" },
    { type = "sftp", host = "nas.local", path = "/backup/docs" }
]

[policy.media]
paths = ["/root/photos", "/root/videos"]
replicas = 3
schedule = "interval:6h"
versioning = "full"
retention = { keep_versions = 5 }
locations = [
    { type = "local", path = "/backup/media" },
    { type = "r2", bucket = "openvault-media" },
    { type = "ctdisk" }
]
```
