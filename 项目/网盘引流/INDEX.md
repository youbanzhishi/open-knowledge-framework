# 网盘引流

> 模式：资源收集 → 城通网盘上传 → 多平台文章发布 → 引流变现

## 执行步骤

### 数据统计流程

1. **收集各平台数据** → 云手机App查看文章数据（知乎、微博、小红书浏览/互动/评论等）
2. **收集网盘下载数据** → 城通网盘后台或API
3. **汇总分析** → 计算各项指标（互动率、转化率、平台表现分）
4. **生成周报** → 保存到 `docs/周报/` 目录
5. **策略调整** → 根据规则提出建议

### 发布流程

1. 收集合规资源（体积≤10MB，无版权风险）
2. 上传城通网盘，每个文件夹≤5个文件
3. 分享文件夹链接（必须用 `/d/` 格式，带密码参数 `?p=8069`）
4. 多平台发布文章，文案只用城通链接
5. 发布前严格去重（至少7天内无重复）

## 注意事项

### 关键经验

- 链接格式：必须用文件夹链接 `/d/`，不能用文件链接 `/f/`
- 城通链接必须带密码参数 `?p=8069`
- 文案只能用城通链接，不能混入夸克/百度链接
- 城通API认证：session参数必须在请求体JSON中传，不能用Bearer Token
- 去重检查：必须先创建发布记录，再执行去重
- 资源质量：宁缺毋滥，不发布低质量/空白资源
- 后期新增平台后需要补发历史文章确保各平台文章数量一致
- 厌恶重复发布相同内容，发布前必须严格去重

### 策略调整规则

| 条件 | 操作 | 优先级 |
|------|------|--------|
| 下载量 > 平均值2倍 | 该类资源优先级提高 | 高 |
| 下载量 < 平均值50% | 该类资源暂停/减少 | 中 |
| 单篇互动 > 100 | 考虑发布续篇或系列 | 中 |
| 某平台表现突出 | 加大该平台投入 | 高 |
| 某平台持续低迷 | 减少该平台发布 | 低 |

### 通知规则

- 🔴 重大变化（爆款/异常波动）：立即通知用户
- 🟡 中等变化（热度变化/表现对比）：写入周报
- 🟢 轻微波动：仅记录
