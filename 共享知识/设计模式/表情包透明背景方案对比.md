# 表情包透明背景方案对比

> 沉淀日期：2026-05-11
> 来源：扣子备份聊天记录（2026-04-27~04-30）

## 方案优先级

| 优先级 | 方案 | 适用场景 | 核心原理 |
|--------|------|----------|----------|
| 1 | 混合方案v3 | 白底原图通用 | rembg处理主体 + 洪水填充保护文字 + 合并掩码 |
| 2 | 三模型对比法 | 要求最高质量 | u2net/isnet-anime/isnet-general-use对比选最佳 |
| 3 | 坐标掩码涂抹法 | 混合方案覆盖不到的局部 | 精确定位坐标处理 |
| 4 | 内部填充法v3 | 深色背景图 | 从内部填充，封开口后灌水 |
| 5 | 内部洪水填充法 | 闭合区域 | 闭运算封开口再灌水 |
| 6 | 外部洪水灌水法 | 浅色主体 | 从外部灌水，水到之处透明化 |
| 7 | 轮廓线围栏法 ⭐ | 黑线轮廓清晰 | 用黑色轮廓线本身当围栏，只补底部开口 |
| 8 | 颜色灌水法 ⭐ | 彩色+白色混合 | 按颜色划分，非白色区域不透明化 |

## 关键铁律

1. **泛蓝修复**：PIL处理alpha通道后RGB偏移变蓝 → 只改alpha不改RGB，用原图RGB重拼
2. **方法保留原则**：新方法不能覆盖旧方法，每种都有用武之地
3. **isnet-anime**：对动漫/卡通形象抠图优于u2net（mIoU 0.94 vs 0.91）
4. **绿幕方案**：图生图生成的绿幕颜色深浅不一，需两套检测条件
5. **用户原创贡献**：轮廓线围栏法和颜色灌水法都是用户原创思路

## 方案选择决策树

```
白底原图 → 混合方案v3
黑底原图 → 先转绿幕 → u2net抠图
有黑线轮廓 → 轮廓线围栏法
彩色+白色混合 → 颜色灌水法
需要精修 → 坐标掩码涂抹法（在主力方案基础上补充）
```
