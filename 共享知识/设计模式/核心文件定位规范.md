# 核心文件定位规范

> 本文档定义知识体系中每个核心文件的定位边界：放什么、不放什么、怎么判。
> 来源：2026-05-12 主人与AI调教师的体系瘦身讨论（OPT-028~031）。

---

## 设计哲学

**铁律是安全培训不是参考书，记忆是看板不是百科全书。**

开100个session时，每个sub-agent被迫读的每一行都在消耗注意力和token。文件定位不清=什么都往里塞=文件膨胀=铁律贬值=没人看=形同虚设。

定位的本质是**减法**：每个文件只做一件事，同一信息只存一处。

---

## 五大核心文件定位

### TOOLS.md — 跨角色通用铁律

| 维度 | 规则 |
|------|------|
| **放什么** | 所有角色都必须遵守的铁律，违反=不可逆损害 |
| **不放什么** | 角色专属铁律、项目热规则、踩坑经验、操作手册 |
| **判断标准** | "开100个session，这条铁律每个都需要读吗？" 不需要→归角色RULES |
| **优先级** | P0数据安全 > P1记忆安全 > P2权限安全 > P3效率规范 |
| **膨胀信号** | 超过20条 → 检查是否有角色专属条目该移走 |

**反例**：CI修复流程、混音输出格式、JUCE编译参数 — 这些只有对应角色需要，放TOOLS=所有角色被迫读无关内容

### MEMORY.md — 当前状态看板

| 维度 | 规则 |
|------|------|
| **放什么** | 当前活跃项目、暂停项目（随时可重启）、待办事项、活跃定时任务 |
| **不放什么** | 已完成项目详情、角色知识/经验、历史记录、铁律 |
| **判断标准** | "这条信息现在还在推进吗？" 不在→归别处 |
| **理想行数** | ≤30行，一眼扫完 |
| **膨胀信号** | 超过50行 → 检查是否有已完成项目该归INDEX |

**归处速查**：
- 已完成项目 → 项目INDEX.md
- 经验/踩坑 → 角色knowledge/
- 体系设计决策 → 体系优化追踪.md
- 项目里程碑/开发记录 → 项目INDEX.md状态段
- 铁律 → TOOLS.md
- 用户偏好 → USER.md

### SOUL.md — 身份与性格

| 维度 | 规则 |
|------|------|
| **放什么** | 我是谁、性格设定、说话风格、项目使命、底线原则 |
| **不放什么** | 具体操作规则、项目进度、技术经验 |

### USER.md — 主人信息

| 维度 | 规则 |
|------|------|
| **放什么** | 主人的个人信息、偏好、联系方式、操作习惯 |
| **不放什么** | 任务进度、技术经验、铁律 |

### SECRET.md — 保密信息

| 维度 | 规则 |
|------|------|
| **放什么** | 密码、密钥、Token、账号等需要保密且重复使用的内容 |
| **不放什么** | 非保密信息、公开项目地址 |

## 信息归属决策树

收到一条新信息时，按以下顺序判断归属：

```
1. 是密码/密钥/Token？ → SECRET.md
2. 是主人的个人偏好/信息？ → USER.md
3. 违反=不可逆损害，所有角色都需要？ → TOOLS.md
4. 违反=不可逆损害，只有某个角色需要？ → 角色RULES.md
5. 换个项目还有用？ → 角色knowledge/
6. 只对当前项目有用？ → 项目docs/knowledge/
7. 正在推进中？ → MEMORY.md
8. 已经完成？ → 项目INDEX.md
9. 是关于体系为什么这样设计的决策？ → 体系优化追踪.md
```

---

## 反模式（不要这样做）

| 反模式 | 为什么错 | 正确做法 |
|--------|---------|---------|
| 什么都往TOOLS塞 | 铁律膨胀=铁律贬值，无关条目越多越不被重视 | 只放跨角色通用铁律 |
| MEMORY记已完成项目 | 项目越多越膨胀，搜索效率下降 | 已完成→INDEX |
| 同一信息存两处 | 更新时容易漏一处，变成矛盾信息 | 唯一真相源，另一处引用 |
| MEMORY记经验知识 | 经验不会过期但MEMORY会膨胀 | 经验→角色knowledge |
| 把通用历史塞进体系优化追踪 | 追踪记的是"体系为什么这样设计"，不是项目历史 | 项目历史→INDEX，体系决策→追踪 |
| TOOLS写操作手册 | 操作步骤不是铁律，是流程 | 流程→SKILL.md |

## 维护周期

| 时机 | 动作 |
|------|------|
| 项目完成 | MEMORY对应条目→项目INDEX，精简为1行指向 |
| 角色新增 | 检查TOOLS是否有该角色专属铁律该移走 |
| TOOLS超20条 | 触发瘦身检查 |
| MEMORY超50行 | 触发瘦身检查 |
| 月度 | 回顾是否有信息归属不合理需调整 |
