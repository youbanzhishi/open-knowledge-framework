> 提炼时间：2026-05-09

## 踩坑经验

| 坑 | 症状 | 解法 | 严重度 |
|----|------|------|--------|
| 依赖sessions_history导致复盘不可靠 | 拉取不到太久之前的对话 | 改为扫描文件：聊天记录+项目文档 | 高 |
| 只看summary丢失技术细节 | 沉淀的内容缺乏深度 | 必须阅读完整对话历史，不能只看summary | 高 |
| 等用户说"记录一下"导致遗漏 | 高价值知识未被沉淀 | 对话中出现高价值知识自动触发沉淀，不等提醒 | 中 |
| 流程性内容误沉淀 | 沉淀了测试报告、运行日志等无复用价值的内容 | 过滤流程性内容，保留有复用价值的技术规范和踩坑经验 | 中 |
| 重复沉淀同一内容 | 同一话题多次沉淀 | 使用已沉淀清单去重，相似度>0.7视为重复 | 中 |

## 最佳实践

### 自动触发信号（出现任一即触发）
| 信号 | 动作 |
|------|------|
| 技术概念解释 | 自动沉淀 |
| 多方案对比 | 自动沉淀 |
| 踩坑教训 | 自动沉淀（双写SKILL.md+文档） |
| 方法论/流程 | 自动沉淀 |
| 系统机制理解 | 自动沉淀 |
| 决策记录 | 自动沉淀 |

### 价值判断规则（权重评分）

**高价值特征（权重+2）**：
- 有对比表格
- 有架构图
- 有原理说明
- 用户明确说好
- 有方法论
- 有解决方案

**中价值特征（权重+1）**：
- 有示例代码
- 有选型建议
- 有注意事项
- 跨session复用

**低价值特征（权重-1）**：
- 纯操作步骤
- 临时信息
- 简单问答
- 上下文依赖强

**阈值**：权重≥3值得沉淀，1-2边界情况

### 过滤标准：流程性内容 vs 有价值知识
```
过滤标准 ≠ "是否由技能生成"
过滤标准 = "是否有复用价值"

✅ 保留：技术规范、踩坑经验、解决方案（即使来源是技能）
❌ 过滤：测试报告、运行日志、任务汇总表、进度报告
```

### 去重机制
- 相似度计算：交集关键词数/并集关键词数
- 相似度>0.7 → 跳过或追加
- 相似度0.4-0.7 → 检查是否互补
- 相似度<0.4 → 新话题创建新文件

## 设计决策

| 决策 | 选项 | 最终选择 | 理由 |
|------|------|----------|------|
| 数据源 | sessions历史 vs 扫描文件 | 扫描文件 | sessions无法拉取太久之前，文件不会丢失 |
| 触发方式 | 等用户提醒 vs 自动触发 | 自动触发优先 | 高价值知识在对话中出现，不应遗漏 |
| 沉淀深度 | 看summary vs 读完整对话 | 读完整对话 | 技术细节在对话过程中，不在summary |
| 过滤标准 | 按来源判断 vs 按价值判断 | 按价值判断 | 避免误删有价值的技术规范 |

## 可升级为热规则的警告

- ⚠️ **对话中出现高价值知识必须自动沉淀**：不等用户说"记录一下"
- ⚠️ **沉淀前必须读完整对话**：不能只看summary，很多技术细节藏在过程中
- ⚠️ **去重命中后跳过沉淀**：已沉淀清单中的内容不重复沉淀
