# Agent 对比与 API 选型指南

## 一、扣子虾 vs Hermes 对比

### 成长性对比

| 维度 | 扣子虾 | Hermes |
|------|--------|--------|
| **记忆系统** | MEMORY.md + USER.md + TOOLS.md，跨会话持久化 | 类似，MEMORY.md + USER.md |
| **技能学习** | 技能可扩展，但受平台限制 | 技能可自编写，完全开放 |
| **迭代速度** | 平台统一更新，被动接受 | 开源社区驱动，可自己改 |
| **跨平台能力** | 扣子、飞书、微信、邮件 | Telegram、Discord、Slack、WhatsApp、Signal、WeChat、iMessage 等 15+ 平台 |
| **数据主权** | 托管在扣子平台 | 可完全本地化 |

**结论**：
- **扣子虾的成长性**：体现在与用户的关系深度、记忆积累、任务执行的连续性。受平台限制，但"开箱即成长"
- **Hermes 的成长性**：体现在技术自由度、社区迭代、完全可控。但需要用户投入运维精力

### 省心程度对比

| 维度 | 扣子虾 | Hermes |
|------|--------|--------|
| **部署** | 零配置，开箱即用 | 需自己部署、配置 API Key |
| **运维** | 平台托管 | 自己维护、更新、监控 |
| **成本** | 扣子积分，平台计费 | 需管理各模型 API 账户 |
| **故障排查** | 平台处理 | 自己排查 |

**结论**：扣子虾更省心。用户只需要对话，不用操心技术细节。Hermes 需要用户当半个工程师。

## 三、Hermes 使用阿里百炼 API

**可以！** 阿里百炼支持 OpenAI 兼容接口，Hermes 可直接对接。

### 配置方法

```bash
# 阿里百炼 OpenAI 兼容接口
base_url: https://dashscope.aliyuncs.com/compatible-mode/v1

# 可用模型
- qwen-turbo（最便宜，¥0.3/M 输入）
- qwen-plus（主力推荐，¥0.8/M 输入）
- qwen3-max（旗舰，¥6/M 输入）
- qwen-long（长上下文）
```

### Hermes 配置示例

```yaml
# Hermes 配置文件
llm:
  provider: openai
  base_url: https://dashscope.aliyuncs.com/compatible-mode/v1
  api_key: sk-xxx  # 你的阿里百炼 API Key
  model: qwen-plus
```
