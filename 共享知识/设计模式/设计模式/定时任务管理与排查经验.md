# 定时任务管理与排查经验

## 一、日程断裂问题

### 问题现象

定时任务突然不执行了，心跳检查持续报告超时。

### 根本原因

**旧日程过期 + 新日程延后 = 执行真空期**

具体表现：
- 旧日程设置了 `until` 字段，到时间就过期
- 新日程的 `dtstart` 设置在未来
- 两个日程之间形成"真空期"，任务无法触发

### 案例：打包发送邮件

```
旧日程：until: 202604280859（今天 08:59 过期）
新日程：dtstart: 202604290300（明天 03:00 才开始）
                        ↓
              今天凌晨的任务没有执行！
```

### 解决方案

1. **更新日程时同步 dtstart**：让新日程立即生效
2. **移除 until 字段**：除非明确需要限定结束时间
3. **同步 tasks.json UID**：确保配置文件和日历一致

## 二、tasks.json 与 Calendar UID 同步

### 问题现象

tasks.json 中的 UID 和 calendar 中的实际 UID 不一致，导致：
- 心跳检查无法正确识别任务状态
- 执行记录混乱

### 同步规范

每次更新日程后，必须同步更新 tasks.json 中的 UID：

```json
// tasks.json 示例
{
  "name": "打包文件和记忆发送邮件",
  "task_id": "7633609582278885667",
  "uid": "8415389d-da5e-4495-9ce2-8ff967e774be_split_7633609425055695139",
  ...
}
```

### 检查方法

1. 使用 `calendar_query` 获取日程列表
2. 对比 tasks.json 中的 UID
3. 不一致时更新 tasks.json

## 五、定时任务执行通知原则

### 用户核心要求

> "我希望所有的定时任务结束后都给我一个通知，无论是完成或者失败甚至出错，都告知我一声"

### 通知规则

```
所有定时任务执行后必须通知用户：
- ✅ 成功：告知完成结果
- ✅ 失败：告知失败原因
- ✅ 出错：告知错误信息
- ✅ 跳过：告知跳过原因（消耗了积分但任务跳过≠正常执行）
```

### 实施规范

1. **创建定时任务时**：在任务描述中添加通知要求
2. **执行完成后**：无论成功/失败/跳过，都必须向用户汇报
3. **汇报内容**：
   - 成功：完成结果（如"已导出10篇文章"）
   - 失败：失败原因（如"登录失败，需要更新Cookie"）
   - 跳过：跳过原因（如"今日引流机会已处理完毕，跳过"）
   - 出错：错误信息（如"网络超时，请检查网络连接"）

### 关联规则：心跳跳过状态处理

**状态为"跳过"的任务必须提醒，不能忽略！**

原因：
- "跳过" ≠ 正常执行
- 用户需要知道为什么跳过
- 跳过可能意味着资源冲突或配置问题

## 六、子Agent情报同步规则

### 用户核心要求

> "情报要同步，子agent解决的问题主agent也要同步"

### 同步规则

1. **子agent完成任务后**：必须通过 sessions_send 向主 agent 汇报结果
2. **发现问题解决后**：必须同步解决方案到主对话
3. **执行结果变更时**：必须主动回传主对话，不能静默结束

### 违规后果

- 修好了不汇报 → 主 agent 一直以为是失败的
- 情报不同步 → 导致重复排查浪费时间

## 七、time_range 参数的重要性

### 作用

设置调度时间窗口，让系统在资源空闲时执行任务，避免整点高峰排队。

### 设置建议

| 场景 | time_range 范围 |
|------|-----------------|
| 用户未指定时间 | ±30min |
| 指定大致时段 | ±10min |
| 指定具体时间 | ±5min |
| 时间敏感任务 | 不设置 |

### 更新日程时的注意事项

**通过 calendar_update API 设置 time_range 时，必须同时提供**：
- dtstart
- dtend
- rrule
- time_range

缺少任何一个参数都可能导致更新失败。

## 八、任务执行优先级与资源冲突

### 问题场景

凌晨时段多个任务竞争云手机资源：
- 01:30 平台引流机会检查
- 02:00 导出元宝收藏
- 同时还有大文件下载任务

### 解决方案

1. **错峰安排**：不同任务的执行时间错开
2. **资源检查**：执行前检查资源占用
3. **超时提醒**：设置合理的超时阈值
4. **自动重试**：心跳检测到跳过时，下次执行窗口自动补执行
