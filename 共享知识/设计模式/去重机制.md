# 去重机制

## 已沉淀清单格式

文件路径：`./知识沉淀/.已沉淀清单.json`

```json
{
  "records": [
    {
      "id": "20260425-001",
      "date": "2026-04-25",
      "topic": "xdotool vs Playwright vs RPA 区别",
      "keywords": ["xdotool", "playwright", "rpa", "自动化", "浏览器"],
      "summary": "三种自动化工具的核心区别和选型建议",
      "file": "xdotool-vs-Playwright-vs-RPA区别.md",
      "source_session": "主对话",
      "source_context": "用户询问微信自动化方案"
    }
  ],
  "last_scan_date": "2026-04-27",
  "scanned_sessions": ["主对话", "cron-xxx", "sub-xxx"]
}
```

---

## 去重逻辑

### 步骤1：提取新话题关键词
```
新话题 = "云手机和云电脑有什么区别"
关键词 = ["云手机", "云电脑", "区别"]
```

### 步骤2：计算与已沉淀话题的相似度
```
相似度 = 交集关键词数 / 并集关键词数
```

### 步骤3：判断
```
if 相似度 > 0.7:
    视为重复 → 跳过或追加
elif 相似度 > 0.4:
    边界情况 → 检查内容是否互补
else:
    新话题 → 创建新文件
```

---

## 互补内容的处理

如果新话题与已有话题相似但有补充信息：

**方案A**：追加到现有文件
```
在现有文件的"补充说明"部分追加新内容
标注更新日期
```

**方案B**：创建关联文件
```
创建新文件，在开头注明"关联：xxx.md"
```

---

## 跨Session去重

当同一话题在多个Session中出现时：

1. **合并处理**：将多个Session的内容合并到一个沉淀文件
2. **标注来源**：在来源处列出所有涉及的Session
3. **保留最完整版本**：选择内容最完整的版本作为基础

---

## 定时扫描时的去重

每次定时扫描时：
1. 读取 `.已沉淀清单.json`
2. 只扫描 `last_scan_date` 之后的消息
3. 新沉淀的内容追加到清单
4. 更新 `last_scan_date`
5. 更新 `scanned_sessions` 列表
