# 文件归类规则

## 核心原则

1. **同类文件集中管理** - 方便查找和维护
2. **迁移必须更新引用** - 避免引用失效
3. **图片谨慎迁移** - 可能导致历史对话无法显示
4. **记录迁移历史** - 保留路径变更记录
5. **删除必须走回收站** - 禁止直接 rm 删除，必须先移到 ./回收站/YYYYMMDD/

## 迁移安全流程

### 迁移前检查

1. **搜索所有引用**
   - 使用 memory_search 或 read_file 搜索旧路径
   - 检查 SKILL.md、config.json、MEMORY.md 等文件

2. **识别文件类型**
   - 图片文件：谨慎迁移，可能影响历史对话
   - 配置文件：必须更新所有引用
   - 技能目录：必须更新技能路径引用

### 迁移步骤

```
1. 记录旧路径和新路径
2. 搜索所有引用旧路径的文件
3. 创建新目录（如不存在）
4. 移动文件到新位置
5. 更新所有引用文件
6. 更新 MEMORY.md 记录迁移历史
7. 验证引用是否全部更新
```

### 图片迁移特殊处理

- **原则**：尽量避免迁移已生成的图片
- **原因**：历史对话中引用的是旧路径，迁移后会显示失效
- **解决方案**：
  1. 新图片按规则放到正确位置
  2. 旧图片保留原位置，不迁移
  3. 或在迁移后创建软链接/记录映射关系

### 心跳检查时的图片判定规则

**心跳检查跳过以下历史图片（不报告异常）：**
- `cnblogs_*.png` - 博客园登录截图
- `csdn_*.png` - CSDN登录截图
- `card_*.png` - 帖子卡片图片
- `cover.png` - 封面图
- 其他根目录下的 `.png`、`.jpg` 图片 - 默认视为历史图片，保留原位置

**只有新图片（明确知道是最近生成的）才需要提醒归类**

## 心跳检查项

心跳检查时执行以下检查：

1. **扫描根目录**
   - 检查是否有技能目录直接放在根目录
   - 检查是否有心跳文件不在心跳目录

2. **扫描引用一致性**
   - 检查 tasks.json 中的技能路径是否正确
   - 检查 MEMORY.md 中的路径引用是否正确

3. **报告异常**
   - 发现错放文件 → 提醒主人需要迁移
   - 发现引用失效 → 提醒主人需要更新
