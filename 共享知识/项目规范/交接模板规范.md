# 交接模板规范

> 来源：agency-agents NEXUS handoff-templates → 体系融入(OPT-086)
> 核心问题：多角色交接时上下文丢失是协作失败的第一原因
> 解决：7种标准交接模板覆盖所有交接场景，格式统一=信息不丢

## 为什么需要标准模板

| 问题 | 示例 | 后果 |
|------|------|------|
| 交接信息不全 | "你继续做那个任务" | 接手者不知道做到哪了 |
| 格式不统一 | 每次交接内容结构不同 | 关键信息遗漏 |
| 丢失验证标准 | 只传代码不传验收条件 | 返工 |
| 无升级路径 | 反复失败没人管 | 卡死 |

## 7种交接模板

### 1. 标准交接（角色间任务传递）

```markdown
## 交接单

| 字段 | 内容 |
|------|------|
| 从 | [角色名] |
| 到 | [角色名] |
| 任务编号 | [WO-XXX] |
| 优先级 | P0/P1/P2/P3 |
| 日期 | YYYY-MM-DD |

### 当前状态
[具体完成到哪一步了]

### 相关文件
- [路径] — [内容说明]

### 交付要求
- [ ] [验收条件1]
- [ ] [验收条件2]

### 约束
[技术/时间/资源限制]
```

### 2. QA通过

```markdown
## QA判定：通过 ✅

| 字段 | 内容 |
|------|------|
| 任务编号 | [WO-XXX] |
| 开发角色 | [角色名] |
| QA角色 | [角色名] |
| 尝试次数 | [N] |

### 验证结果
- [x] [验收条件1] — 已验证
- [x] [验收条件2] — 已验证

### 下一步
→ 标记任务完成，推进下一任务
```

### 3. QA失败

```markdown
## QA判定：失败 ❌

| 字段 | 内容 |
|------|------|
| 任务编号 | [WO-XXX] |
| 开发角色 | [角色名] |
| QA角色 | [角色名] |
| 尝试次数 | [N]/3 |

### 问题列表

#### 问题1：[分类] — 严重度[Critical/High/Medium/Low]
- 预期：[应该怎样]
- 实际：[实际怎样]
- 修复指引：[具体怎么改]
- 涉及文件：[路径]

### 验收条件状态
- [x] [条件1] — 通过
- [ ] [条件2] — 失败（见问题1）

### 重试指引
1. 只修上面列出的问题，不要引入新变更
2. 修完重新提交QA
3. 这是第[N]次尝试，最多3次
```

### 4. 升级报告（3次重试失败后）

```markdown
## 升级报告 🚨

| 字段 | 内容 |
|------|------|
| 任务编号 | [WO-XXX] |
| 尝试次数 | 3/3 已耗尽 |
| 升级给 | [AI调教师/主对话] |

### 失败历史
- 尝试1：[问题] → [修了什么] → [仍失败原因]
- 尝试2：[问题] → [修了什么] → [仍失败原因]
- 尝试3：[问题] → [修了什么] → [仍失败原因]

### 根因分析
[为什么反复失败：任务拆分不当？角色不对？架构问题？]

### 建议处理
- [ ] 换角色重做
- [ ] 拆分成更小任务
- [ ] 修改方案/架构
- [ ] 接受当前状态+记录限制
- [ ] 延期到下个迭代
```

### 5. 阶段门控（项目阶段切换）

```markdown
## 阶段门控：Phase [N] → Phase [N+1]

| 门控条件 | 阈值 | 结果 | 证据 |
|----------|------|------|------|
| [条件1] | [阈值] | ✅/❌ | [引用] |
| [条件2] | [阈值] | ✅/❌ | [引用] |

### 遗留风险
| 风险 | 严重度 | 缓解方案 | 负责角色 |
|------|--------|----------|----------|

### 下一阶段激活角色
| 角色 | 职责 | 优先级 |
|------|------|--------|
```

### 6. Sprint交接

```markdown
## Sprint交接

| 字段 | 内容 |
|------|------|
| Sprint编号 | [N] |
| 周期 | [开始] → [结束] |
| 目标 | [Sprint目标] |

### 完成状态
| 任务 | 状态 | QA次数 | 备注 |
|------|------|--------|------|

### 质量指标
- QA一次通过率：[X]%
- 平均重试次数：[N]
- 完成率：[X/Y]

### 延期到下个Sprint
| 任务 | 原因 | 优先级 |
|------|------|--------|

### 回顾
- 做得好：[关键成功]
- 要改进：[关键问题]
- 行动项：[下Sprint具体改什么]
```

### 7. 事件交接（生产问题）

```markdown
## 事件交接

| 字段 | 内容 |
|------|------|
| 严重度 | P0/P1/P2/P3 |
| 发现时间 | [时间戳] |
| 当前状态 | 调查中/缓解中/已解决/复盘 |

### 事件描述
- 发生了什么：[描述]
- 影响范围：[谁/什么受影响]
- 时间线：
  - [HH:MM] — [事件]
  - [HH:MM] — [事件]

### 已采取行动
1. [行动+结果]

### 交接上下文
- 已尝试：[什么]
- 未尝试：[什么]
- 疑似根因：[猜测]
- 相关日志/指标：[位置]
```

## 使用规则

1. **交接必须用模板**：角色间传递任务时，从上面选对应模板填写
2. **QA循环≤3次**：失败3次必须升级，不允许无限重试
3. **模板写在哪**：
   - 工单系统内：交接单内容写进WO-XXX.md
   - 即时交接：sessions_send消息按模板格式
4. **模板是下限不是上限**：可以加字段，但不能少字段
