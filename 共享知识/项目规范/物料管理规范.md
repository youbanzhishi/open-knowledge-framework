# 物料管理规范

> 物料 = 角色交接的实体产物（表情包、音频、文档、配置、镜像等）
> 核心原则：物料跟着项目走，交接台写路径，接收者一眼定位

## 存放规则

| 物料类型 | 存放位置 | 示例 |
|---------|---------|------|
| 文档/配置 | `项目/xxx/docs/` | 部署文档、API文档 |
| 图片（<5MB） | `项目/xxx/assets/` | 架构图、截图 |
| 大文件/交付物（音频/视频/表情包） | `项目/xxx/output/` | MP3、表情包zip、视频、release包 |
| docker镜像/Release | GitHub Release | 镜像URL |

## 交接台写法

交接单的「物料位置」字段按以下格式写：

- 体系仓库内文件：`项目/xxx/docs/deploy.md`
- 项目产出物：`项目/xxx/output/文件名`
- 外部资源：`https://github.com/xxx/releases/tag/v1.0`
- 云端文件：`computer://项目/xxx/output/文件名`

## 项目目录结构（产出部分）

```
项目/xxx/
├── docs/          # 文档
├── assets/        # 小图片
├── output/          # 交付物、大文件 ← 物料统一放这里
│   ├── 表情包/
│   ├── 音频/
│   ├── 视频/
│   ├── release/
│   └── 交接单/    # 非即时交接的详细说明
└── ...

## 原则

1. **不往git仓库放大文件** — output/目录下的音频/视频/压缩包通过gitignore排除，git只存索引和文档
2. **物料位置写入交接台** — 交付者必须写明物料在哪，接收者不用猜
3. **统一放项目产出目录** — `项目/xxx/output/`，不跨目录，跟着项目走
4. **用完不删** — 历史物料保留，"这些都是来时的路"

## IP素材独立仓库规则（2026-05-13 新增）

> 一个IP一个私有GitHub仓库，素材跟着IP走，不放在知识体系仓库里。

### 规则
1. **一个IP = 一个私有GitHub仓库**：表情包/IP形象等膨胀型内容，独立建仓库存放
2. **仓库命名**：IP英文名或拼音（如 `aojiao-xiaotangyuan`），简洁可辨识
3. **必须私有**：IP素材涉及商业价值，严禁公开仓库
4. **目录结构**：按版本/主题分目录，版本内按素材类型分（原图/透明背景/衍生图）
5. **README必写**：版本清单+目录结构+形象核心要点+关联知识体系项目路径
6. **对比图/调试中间文件不入仓库**：开发过程的对比图、掩码预览等放本地回收站或忽略
7. **项目INDEX必须注明仓库地址**：关联项目的INDEX.md「当前文件位置」段必须写GitHub仓库URL
8. **交接时写仓库地址**：交接台的物料位置写 `https://github.com/youbanzhishi/xxx`（私有仓库，需凭据访问）
