# 项目规范（宪法层）

> 所有项目共享的规范，改一次全局生效。INDEX.md只引用不复制。

## 为什么需要这层？

扣子协作痛点：规范散落在每个项目 → 改一次要通知所有人 → 最后没人遵守

解决：**规范集中管，INDEX只引用**
- 这里改 = 所有项目自动生效
- 项目特例在INDEX.md的"本地偏差"段声明
- 偏差必须有理由，没有理由就是bug

## 使用方式

1. **新项目启动**：先读这里的规范，按规范建目录
2. **现有项目对齐**：逐步迁移，不强制一刀切
3. **规范变更**：改这里的文件 → 通知各项目session → 各项目按需同步
4. **项目特例**：在INDEX.md的"本地偏差"段声明，写明原因

## 规范变更流程

1. 在对应规范文件中修改
2. 更新本README的变更日志
3. 通知各项目主对话session（或等heartbeat自然发现）
4. 各项目按优先级逐步对齐
