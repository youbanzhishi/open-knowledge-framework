# 工单自动流转系统

> 版本：v1.0
> 日期：2026-05-19
> 作者：AI调教师
> 关联：WO-021、WO-028、OPT-077、OPT-080

## 概述

工单自动流转系统实现"心跳触发 → 自动扫描 → 切换角色处理 → 循环直到完成"的完全自动化流程。

## 核心理念

- **机器能做的，不让人做**：扫描、状态更新、队列管理全部自动化
- **五步门保证质量**：自动切换角色后按五步门执行，保证处理质量
- **无需外部依赖**：纯文件驱动，不依赖API调用、消息通知、cron定时

## 触发机制

### 心跳触发

```
心跳（每30分钟）
    ↓
自动扫描交接台工单WO-*.md
    ↓
发现有⏳待领取 → 切角色处理
    ↓
完成后再次扫描 → 循环直到无待办
```

### 为什么不需要cron

| 方案 | 问题 |
|------|------|
| cron+脚本扫描 | 需要额外配置，需要手动触发脚本 |
| 消息通知唤醒 | 需要知道会话ID，需要API调用 |
| **心跳自动扫描** | 每次心跳自动执行，无需额外配置 |

## 执行流程

### 1. 扫描工单

读取 `open-knowledge-system/交接台/工单/` 目录下所有 `WO-*.md` 文件，提取：
- 状态（是否为⏳待领取或🟡待领单）
- 接收角色
- 优先级
- 任务描述

### 2. 切换角色

使用 `act.sh` 切换到对应角色：
```bash
bash scripts/act.sh "处理WO-xxx工单" . "角色/xxx"
```

### 3. 五步门处理

按角色RULES中的五步门执行：
1. 查后定方案
2. 执行
3. 验证
4. 反哺沉淀
5. 交付闭环

### 4. 更新状态

处理完成后：
- 更新工单状态为 ✅已完成
- 记录处理记录
- 再次扫描剩余工单

### 5. 循环直到完成

重复步骤1-4，直到所有待处理工单都已完成。

## 配置方式

### HEARTBEAT.md 配置

```markdown
# HEARTBEAT.md

# 心跳任务：每次触发时自动执行

## 任务1：自动扫描并处理工单
- 扫描 `open-knowledge-system/交接台/工单/` 目录下所有WO-*.md
- 发现状态为"⏳待领取"或"🟡待领单"的工单
- 自动切换到对应角色
- 按五步门执行处理
- 处理完成后更新工单状态为已完成
- 再次扫描，直到无待办工单
```

## 工单状态流转

```
⏳待领取 → 🔧处理中 → ✅已完成
              ↓
         ⚠️ 卡住（记录问题）
```

## 工单格式要求

每个工单文件必须包含：

| 字段 | 格式 | 说明 |
|------|------|------|
| 接收角色 | `\| 接收角色 \| 角色名 \|` | 决定谁来处理 |
| 状态 | `## 状态` + `⏳待领取` | 扫描标识 |
| 优先级 | `优先级：P1/P2/P3` | 处理顺序 |

## 优势

1. **零配置**：只需要配置HEARTBEAT.md，无需cron、无需API
2. **自动轮询**：每次心跳自动扫描，无需手动触发
3. **五步门保证质量**：自动处理但不牺牲质量
4. **可追溯**：每个工单有完整处理记录
