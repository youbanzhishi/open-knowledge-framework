# 踩坑记录规范

> 踩坑记录是知识体系的核心防御机制。"踩2次升级hot-rules"不能靠记忆，靠计数。

## 记录模板

每条踩坑记录必须包含以下字段：

```markdown
### 踩坑标题
- **count**: 1（每次同类型错误+1）
- **last_date**: YYYY-MM-DD
- **严重度**: P0/P1/P2/P3
- **描述**: 问题描述，一句话说清
- **解决方案**: 怎么解决的，可操作的步骤
```

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| count | ✅ | 同类型错误累计次数。每次同类型再犯+1 |
| last_date | ✅ | 最近一次发生日期，格式YYYY-MM-DD |
| 严重度 | ✅ | P0=不可恢复/P1=阻塞任务/P2=绕路可解/P3=体验不佳 |
| 描述 | ✅ | 简明描述问题和现象 |
| 解决方案 | ✅ | 可操作步骤，不要写"注意避免"这种废话 |

## 计数升级机制

| count | 动作 |
|-------|------|
| 1 | 记knowledge（被动防御，搜到能用） |
| 2 | 升hot-rules（每次加载可见） |
| 3+ | 提工单给调教师，考虑升铁律或脚本化 |

## act.sh自动展示

act.sh搜索结果中，踩坑记录会自动显示count值：
```
📝 角色/xxx/knowledge/踩坑-xxx.md [踩3次] （命中5行）
```
agent看到"踩3次"→触发升级判断→该升hot-rules或提工单。

## 计数更新方式

1. 发现同类型错误时，找到原踩坑记录
2. count字段+1，更新last_date
3. 如果count>=2但还没升hot-rules→立即升级
4. 如果count>=3→提工单给调教师
