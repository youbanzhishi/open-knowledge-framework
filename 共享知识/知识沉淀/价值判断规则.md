# 价值判断规则

## 高价值特征（权重 +2）

1. **有对比表格**：对话中包含工具/方案对比
2. **有架构图**：包含流程图、架构图等可视化内容
3. **有原理说明**：解释"为什么是这样"
4. **用户明确说好**：用户说"这个不错"、"学到了"、"记一下"等
5. **有方法论**：可复用的思路或经验
6. **有解决方案**：有价值的问题排查和解决过程

## 中价值特征（权重 +1）

1. **有示例代码**：包含可参考的代码片段
2. **有选型建议**：给出明确的选择建议
3. **有注意事项**：列出需要注意的点
4. **跨session复用**：内容可以应用到多个场景

## 低价值特征（权重 -1）

1. **纯操作步骤**：只是"怎么做"，没有"为什么"
2. **临时信息**：时效性强的内容
3. **简单问答**：一问一答，没有展开
4. **上下文依赖强**：脱离上下文无法理解

## 判断阈值

- 权重 ≥ 3：值得沉淀
- 权重 1-2：边界情况，看内容长度决定
- 权重 ≤ 0：不沉淀

---

## 示例判断

| 对话内容 | 特征 | 权重 | 是否沉淀 |
|---------|------|------|---------|
| "xdotool vs Playwright区别" | 有对比表格 + 原理说明 + 用户说好 | 2+2+2=6 | ✅ 是 |
| "云电脑怎么开通" | 有步骤 + 有说明 | 1-1=0 | ❌ 否 |
| "Agent架构讲解" | 有架构图 + 原理说明 + 方法论 | 2+2+2=6 | ✅ 是 |
| "Chrome启动失败的解决" | 有解决方案 + 有原理说明 | 2+2=4 | ✅ 是 |
| "帮我查个快递" | 简单问答 | -1 | ❌ 否 |
