# NEXUS Pipeline - 多智能体开发流程

## 概述
NEXUS Pipeline是6阶段开发流程，用于协调多个AI智能体有序完成复杂项目。

## 阶段定义

### Phase 0: 规划
- 需求分析
- 任务拆分
- 依赖识别

### Phase 1: 设计
- 架构设计
- 接口定义
- 技术选型

### Phase 2: 实现
- 核心代码开发
- 单元测试
- 代码审查

### Phase 3: 集成
- 模块集成
- 集成测试
- 性能优化

### Phase 4: 部署
- 环境配置
- 部署脚本
- 监控告警

### Phase 5: 运维
- 故障响应
- 持续改进
- 文档更新

## 角色分工
- **产品经理**: Phase 0-1
- **系统开发者**: Phase 2-3
- **运维工程师**: Phase 4-5
- **测试工程师**: 全程质量保障

## 与知识体系结合
- 工单流转使用交接台机制
- 每个阶段产出需更新MEMORY.md
- 关键决策记录在ADR
